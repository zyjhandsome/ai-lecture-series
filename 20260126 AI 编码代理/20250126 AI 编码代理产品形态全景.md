## 1. 为什么关注？— 市场与趋势

### 1.1 技术演进阶段与形态逻辑

AI编码工具按**产品交付形态**可划分为**四大核心形态 + 一类新兴平台嵌入式形态（四+一）**。它们并非替代关系，而是**针对不同开发阶段、不同任务粒度**的专业化分工演进；同时，头部产品正在跨形态扩展，形成多入口一致体验，串联逻辑演进为**"从辅助到自主，从单点到全栈，从单形态到多形态融合"**。

```
2022-2023          2024              2025             2025末-2026
   │                 │                 │                 │
   ▼                 ▼                 ▼                 ▼
┌─────────┐    ┌──────────┐    ┌──────────────┐    ┌────────────────┐
│ 行内补全 │ →  │ 侧边栏对话 │ →  │ 自主代理执行  │ →  │ 多代理协作系统  │
│ Copilot │    │ Chat模式  │    │ Cursor/Claude│    │ Antigravity   │
│ 单行建议 │    │ 问答交互  │    │ 多文件编辑   │    │ Cursor Cloud  │
└─────────┘    └──────────┘    │ 终端执行     │    │ Claude Code   │
                               └──────────────┘    │ 并行任务处理   │
                                                   └────────────────┘
                                                            ▲    
                                                        【当前阶段】
```

**四大核心形态 + 平台嵌入式形态的演进逻辑：**

1.  **原子级辅助 (IDE插件)**：
    *   **逻辑**：最初也是最基础的需求。在不改变开发者习惯的前提下，解决"下一行代码怎么写"的问题。
    *   **定位**：外挂式大脑，随叫随到。
2.  **项目级协同 (IDE内置代理)**：
    *   **演进**：插件受限于API权限，无法理解整个项目。于是诞生了**AI原生IDE**（如Cursor），它能索引整个文件树。
    *   **逻辑**：解决"这个功能怎么实现"的问题，需要跨文件修改能力。
3.  **脚本化/批量化 (CLI终端代理)**：
    *   **演进**：GUI界面操作太慢，不适合批量任务。于是AI进入终端（如Claude Code），直接操作Shell和Git。
    *   **逻辑**：解决"帮我把所有组件重构一遍并跑通测试"的问题，适合无头（Headless）模式。
4.  **从零到一 (全栈应用生成器)**：
    *   **演进**：如果连IDE都不想打开呢？针对非技术人员或快速原型，直接在浏览器生成应用。
    *   **逻辑**：解决"我有一个想法，帮我做个Demo"的问题，**降低门槛**。
5.  **协作平台委托执行 (平台嵌入式代理)**：
    *   **演进**：当任务协同、评审、追踪主要发生在 Slack/GitHub 等平台时，代理需要直接进入协作流。
    *   **逻辑**：解决"不用打开IDE，也能发起、跟踪、验收编码任务"的问题，适合异步委托和团队协作。

**串联后的工作流：**
用**全栈生成器**验证想法 → 用**IDE内置代理**开发核心功能 → 用**IDE插件**补全细节 → 用**CLI代理**批量生成文档/测试和维护 → 用**平台嵌入式代理**在团队协作系统中异步委托与追踪交付。

**当前阶段**：行业已进入"多代理协作 + 多形态融合"阶段。Cursor支持并行代理（Best-of-N模式）和Cloud Agent（云端异步执行），Google Antigravity的Manager View可协调多代理并行工作，Claude Code从CLI扩展到Web/桌面/移动端/Slack/GitHub/编辑器扩展，体现出"同一代理能力多入口触达"趋势。

> **产品形态演进的深层洞察**：
> 
> 1. **专业化分工与跨形态融合并行**：多种产品形态的并存并非技术不成熟的表现，而是**专业化分工 + 统一代理能力多入口触达**的共同结果。就像传统开发中，IDE、CLI、CI/CD工具各司其职一样，AI编码工具也在形成类似生态分工，同时头部产品在做形态融合。
> 
> 2. **用户角色的分化**：不同形态对应不同的用户角色和使用场景。IDE插件面向"日常编码者"，IDE内置代理面向"架构重构者"，CLI代理面向"自动化工程师"，全栈生成器面向"产品验证者"，平台嵌入式代理面向"团队协同与任务委托者"。理解自己的角色，才能选择最合适的工具。
> 
> 3. **工作流的串联逻辑**：核心形态可串联成完整工作流——用全栈生成器验证想法 → 用IDE内置代理开发核心功能 → 用IDE插件补全细节 → 用CLI代理批量生成文档/测试 → 在平台嵌入式代理中做异步委托与团队协同。**组合使用**而非单一依赖，才能最大化效率。
> 
> 4. **技术栈的抽象层次**：从低到高，形态的抽象层次递增——行内补全（原子级）→ 侧边栏对话（函数级）→ 自主代理（文件级）→ 多代理协作（项目级）→ 平台嵌入协同（组织级）。这种抽象层次的提升，使得AI能够处理越来越复杂的任务，但同时也需要更强的上下文理解能力。

### 1.2 工具协议 (MCP) 与 技能规范 (Skills) 的双重演进

在2026年的语境下，代理能力的扩展已分化为两个关键维度：**连接能力 (MCP)** 与 **认知能力 (Skills)**。

*   **MCP (Model Context Protocol)**：**"连接器"**。开放标准协议，让 AI 能连接数据库、Slack、GitHub 等外部工具。
*   **Skills (技能规范)**：**"说明书"**。定义代理行为模式的标准（通常为 `SKILL.md`），让代理学习特定团队的 Coding Style 或工作流。

| 产品形态 | 代表产品 | MCP 支持 (连接工具) | Skills 支持 (定义行为) | 关键节点 |
| :--- | :--- | :--- | :--- | :--- |
| **CLI 终端代理** | **Claude Code** | **原生核心** (2025.6) | **首发定义** (2025.10) | 2025年10月推出 `SKILL.md` 规范，定义了代理如何学习新技能，引发行业效仿。 |
| **IDE 内置代理** | **Cursor** | **全面支持** (2025.6) | **正式跟进** (2026.1) | v1.0 (2025.6) 支持 MCP；**v2.4 (2026.1)** 正式引入 Skills 支持，允许用户自定义代理能力。 |
| **IDE 内置代理** | **Windsurf** | **深度集成** | **流式学习** | 通过 Cascade 引擎支持工具调用；更侧重于通过观察用户行为"隐式"学习技能。 |
| **IDE 插件** | **GitHub Copilot** | **扩展生态** | **规则配置** | 主要通过 Copilot Extensions (MCP) 连接工具；技能定义更多依赖 `.github/copilot-instructions.md`。 |

> **关键区别**：**MCP** 让 AI 手里有了"锤子"（工具），而 **Skills** 教会了 AI 怎么"盖房子"（方法论）。Claude Code 在 2025 年底带火了 Skills 概念，使代理的定制化从 Prompt 工程进化为工程化的技能包管理。

#### 1.2.1 Skills 演进：从 Prompt Engineering 到 Flow Engineering

把你工作中反复执行、有固定流程的任务，拆解成 AI 能理解、能稳定复用、能自动执行的一套流程——这不仅仅是"玩法"的升级，更是 AI 应用逻辑的一次质变。

我们正在经历从 **Prompt Engineering（提示词工程）** 向 **Flow Engineering（流程工程）** 的跨越。Claude Skills 的爆发，正是这一趋势的最佳注脚。哪怕是现在流行的 Vibe Coding（氛围编程），其尽头其实也是 Skills——将经验沉淀为流程，将直觉固化为逻辑。

> **从 Prompt 到 Skills 的深度洞察**：
> 
> 1. **工程化的必然趋势**：Prompt Engineering 的本质是"每次对话都要重新描述需求"，而 Skills 的本质是"将需求固化为可复用的流程"。这种转变使得 AI 应用从"手工作坊"走向"工业化生产"，是技术成熟的标志。
> 
> 2. **知识沉淀的机制**：Skills 不仅仅是代码规范，更是**团队知识沉淀的机制**。通过 `SKILL.md` 文件，团队可以将最佳实践、编码风格、工作流程等隐性知识显性化，让新成员快速上手，让 AI 代理理解团队文化。
> 
> 3. **跨平台互操作的意义**：2025年12月 Agent Skills 开放标准的发布，标志着 Skills 从"产品特性"升级为"行业标准"。这意味着不同平台的 Skills 可以互操作，团队可以在 Cursor 中开发的 Skills 迁移到 Claude Code 中使用，**降低了锁定风险，提高了投资回报**。
> 
> 4. **Vibe Coding 的归宿**：当前流行的 Vibe Coding（氛围编程，通过自然语言描述生成应用）看似与 Skills 无关，但其底层逻辑其实也是"将经验固化为流程"。当 Vibe Coding 的流程足够成熟时，自然会沉淀为 Skills，实现从"一次性生成"到"可复用模板"的转化。
> 
> 5. **企业级应用的关键**：对于企业而言，Skills 的价值在于**标准化和可审计性**。企业可以建立 Skills 库，确保所有 AI 代理遵循统一的编码规范和安全标准，这是 Prompt Engineering 无法实现的。这也是为什么 Skills 在企业级产品（如 GitHub Copilot Enterprise）中受到重视的原因。

### 4.2 成本与性能深度对比（2026版）

这几类产品的计费模式有本质区别：**订阅制 (SaaS)** vs **消耗制 (Pay-as-you-go)**。

#### A. 价格对比表

| 产品 | 计费模式 | 典型费用 | 适用人群 | 成本陷阱 |
| :--- | :--- | :--- | :--- | :--- |
| **Cursor / Windsurf** | **固定订阅制** | **$20/月** (Pro)<br>$40/月 (Business) | **高频交互用户**<br>每天写代码>2小时 | 几乎无陷阱，封顶费率，性价比最高。 |
| **GitHub Copilot** | **固定订阅制** | $10/月 (Pro)<br>$19/月 (Business)<br>$39/月 (Enterprise) | **企业用户**<br>合规要求高 | Business版适合团队，Enterprise版包含Copilot Agent模式，性价比不错。 |
| **Claude Code** | **按量付费 (Token)** | **~$15-30/月** (轻度)<br>**~$85-200/月** (重度) | **架构师/Tech Lead**<br>做复杂重构任务 | **极其昂贵**。如果是全自动循环任务，一天可能消耗$10+。 |
| **Devin** | **混合制** | 订阅费 + 算力单元(ACU) | **企业定制**<br>通常$300+/月起 | 适合代替外包，单次任务成本高，但比真人便宜。 |

#### B. 性能与效果对比 (ROI分析)

根据最新的行业评测和成本效率报告：

1.  **代币效率 (Token Efficiency)**：
    *   **Claude Code** 优于 Cursor。根据社区评测，Claude Code 在处理相同任务时，消耗的 Token 数约是 Cursor 的 **1/5.5**。这意味着虽然它单价贵，但它"想得更清楚，做得更少"，适合大型重构。（注：此为社区基准测试数据，实际消耗因任务类型而异）

2.  **基准测试 (SWE-bench Verified)**：
    *   **Claude Code (Opus 4.5/4.6)**: **80.8-80.9%** (最高水平)
    *   **GPT-5.1-Codex-Max**: **77.9%**；**GPT-5.2**: **80.0%**
    *   **Gemini 3 Pro/Flash**: **76.2-78.0%**
    *   **Devin**: 专注端到端成功率，虽然跑分不一定最高，但**自我修复能力 (Self-healing)** 最强，PR合并率达67%。
3.  **响应速度**：
    *   **Cursor (Fast Mode)**: 毫秒级，适合"边写边改"。
    *   **Devin / Claude Code**: 分钟级甚至小时级，适合"丢给它去跑，我去喝咖啡"。

**总结建议**：
*   **省钱且高效**：主力用 **Cursor ($20)** 处理90%日常工作。
*   **攻坚克难**：遇到极复杂的重构或文档生成，临时调用 **Claude Code** (按量付费)，虽然单次贵，但省下的时间（人力成本）远超 $5-10 的API费用。

- 需要**跨团队异步分发与验收任务** → 平台嵌入式代理
