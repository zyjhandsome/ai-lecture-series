## 1. OpenAI 全力押注 Codex：砍掉"支线任务"构建超级应用对抗 Claude

**核心观点**：OpenAI 正在进行一次重大战略转型——将 Codex（其编码/代理产品）融入 ChatGPT，构建一个对标 Anthropic Claude Cowork 的超级应用，同时砍掉 Sora 等未达预期的产品线。

**要点提炼**：
- Alex Heath 前一天在 OpenAI 总部与领导层深入交流，并在 OpenAI 创始办公室的图书馆与 Codex 团队及 Greg Brockman（联合创始人）共进晚餐
- **Peter Steinberger**（OpenClaw 创造者）加入 OpenAI 约三周，接受了加入后的首次采访
- OpenAI 砍掉 Sora、搁置"成人模式"，将这些称为"支线任务"（side quests），集中所有资源和 GPU 构建超级应用
- Fidji Simo（OpenAI 应用 CEO）和 Greg Brockman 共同主导这次产品大整合
- Codex 将成为 ChatGPT 的核心"执行引擎"（harness），类似 Claude Code 之于 Claude 的关系

**详细内容**：

Alex 透露他花了一整天在 OpenAI，与大量领导层交流，尤其是 Codex（即 OpenAI 的 Claude Code 竞品）团队。当晚在 OpenAI 最初的办公室——一个他们早期进行大量研究工作的图书馆里——与 Codex 团队和 Greg Brockman 共进寿司晚餐。

> **Alex**："Peter Steinberger，OpenClaw 的创造者，'Claude 之父'（Claude father），加入 OpenAI 大约三周了。他告诉了我他在做什么、在想什么——这是他的首次采访。"

OpenAI 后悔没有更早在编码领域做到更好。Anthropic 对编码的聚焦，尤其是 Claude Cowork 让非技术用户也能通过 AI 执行代码、爬取数据库等操作（你不需要知道底层发生了什么），这给了 Anthropic 一个"起飞时刻"。

> **Alex**："我认为 OpenAI 正在意识到，金钱、增长、热度和好感都在编码生产力领域，而不是其他地方。"

OpenAI 的策略是将 Codex 的能力构建进 ChatGPT 的聊天界面——这个界面已经触达近 **10 亿用户**。目前存在一个认知问题：用户不知道 Codex 具备 Claude Cowork 的能力。将编码执行引擎融入消费级聊天体验，是 OpenAI 对 Anthropic 的正面回应。

Ellis 追问这是"追随炒作周期"还是"真正的战略转向"。Alex 认为并非变心：

> **Alex**："这些公司确实相信编码是通向 AGI 的路径，这一直没变。但我认为他们看到了——哦，这东西在起飞。模型现在也能够可靠地（至少大多数时候可靠地）执行代理式任务了。所以时机就是现在。"

OpenAI 观察到 Claude Code 的增长后产生了紧迫感。Sora 在 2025 年 9 月末上线后初始爆发，登顶 App Store，但后续增长不如预期。与此形成鲜明对比的是，人们在"疯狂使用" AI 编码工具和 Claude Cowork。

[🕒 约 10:00–18:00]

## 2. 聊天机器人将进化为代理编排器：MCP 连接一切的 6-9 个月倒计时

**核心观点**：大型 AI 聊天助手（ChatGPT、Claude、Gemini）将在 6-9 个月内从"输入文本、获取文本"的基础聊天工具进化为"代理编排器"——拥有联系人、记忆、工具连接和执行能力的中枢平台。

**要点提炼**：
- MCP（Model Context Protocol）正在将 Replit、Figma、Granola 等工具连接到主要 AI 助手
- 大量 Vibe Coding 工具已通过 MCP 与主要助手打通
- 应用可能变成"提示的容器"（containers for prompts），以用户界面呈现
- 这一转变将比多数人预期的更快发生

**详细内容**：

Alex 认为，Replit 等 Vibe Coding 工具已经通过 MCP 连接到 ChatGPT 和 Claude 等主要助手。未来 ChatGPT、Claude、Gemini 将成为拥有用户联系人、记忆、以及各种数据和工具钩子的编排层。

> **Alex**："Figma 已通过 MCP 对 Claude 完全开放。你可以让 Replit 通过 Claude 直接构建东西——比如当你的 Granola 会议记录工具同步时，自动触发一个 Replit 应用。我认为我们会在未来几个月内开始以完全不同的方式思考应用。"

Ellis 提出了一个关键区分：Wabi 的 Eugenia 将许多迷你应用描述为"提示的容器"——某些功能作为用户界面呈现确实更好（比如 AI 图片生成的相册）。而 Dreamer 的底层本质是连接各种拥有 API 和 MCP 的服务。Ellis 认为，理论上一年后你可以直接让 Claude 做这些事情。

> **Alex**："宏观来看，我们不会再把这些聊天机器人看作基础的'输入文本、获取文本'工具。在 6 到 9 个月内，我们将把它们视为代理编排器。这将以非常快的速度向更多人引入这种能力。"

[🕒 约 18:00–22:00]

## 4. Stream 智能戒指的诞生：从城市探索工具到对话式 AI 伴侣的多代原型进化

**核心观点**：Sandbar 的 Stream 智能戒指经历了多代原型迭代，从最初的搜索/导航工具转变为以对话和笔记为核心的 AI 可穿戴设备，其转折点是当团队开始探索"对话"场景时，用例才真正"咔嗒"对上了。

**要点提炼**：
- 第一代原型形态庞大（"像验光师放在你眼前的仪器"或"居家监控脚环"），底部有缺口
- 初始用例是连接搜索 API 和导航，定位为城市探索产品
- **关键转折**：当团队开始探索对话和即时笔记功能时，才感觉到产品方向正确
- Alpha 测试发现多种自发用例：会后人际关系笔记、教授用来备课、开车时准备会议
- Mina 本人已佩戴原型约 2 年

**详细内容**：

Mina 展示了多代原型的实物。第一个原型非常大——Alex 形容为"像验光师测视力时放在你眼前的东西"，Ellis 则说像"居家监控脚环"。Mina 曾经戴着这个原型在曼哈顿西区的 High Line 步道行走，试图把它当作某种时尚声明。

> **Mina**："当我们有了这个原型并尝试各种用例时，它们看起来非常不同。我们把它连接到搜索 API，连接到导航，主要把它当作一个城市探索产品。只有当我们开始探索对话和即时笔记时，才感觉用例真正咔嗒对上了。"

经过更多迭代后，团队在"Alpha 测试"中将原型分发给外部用户。使用方式出人意料地多样化：有人将它用作个人关系管理工具——每次会议后记录关于某个人的笔记；有一位教授用它在开车途中准备课程内容；还有人用它准备会议。

> **Mina**："Stream 的很多魔力在于对话。它不仅仅捕捉你说的话，它会记住并以你自己的声音回话。所以你可以把它当作一个思考伙伴。"

这就是为什么像备课计划、会议准备、写作构思等"渐进式思考"的用例开始蓬勃发展——这些场景本质上是迭代性的，不是一次性输入就完成的。

近期团队更进一步探索代理能力：LLM 不仅能回应，还能执行动作。"如果你随时随地都能触发一个动作，让某件事开始运行，那意味着什么？"

[🕒 约 28:00–35:00]

> **Mina**："它是可以引导的。它不会用一大段话来轰炸你，而是问你问题来引导方向。"

## 6. "自我延伸"产品哲学：触觉反馈、无身份人格与用户完全控制权

**核心观点**：Sandbar 的整个产品设计由一个核心信念驱动——"自我延伸"（Self-Extension）：技术应当感觉像是你自己的延伸，而非伙伴、助手、替代品甚至工具。这个北极星指导了从触觉反馈到声音克隆到人格设计的每一个决策。

**要点提炼**：
- **北极星**：技术应感觉像人的延伸，消除目标不一致和过度解释的摩擦
- **触觉反馈**：在如此小的戒指中加入触觉引擎"并不容易"，但为了"闭环"感知体验而坚持实现——让用户感觉自己说的话已被"听到"
- **自我声音克隆**：联合创始人 Kirak（Hong）建议用用户自己的声音作为默认，第一天会感觉怪异，到第二天团队每个人都爱上了这个"内心声音"
- **无身份人格**：Stream 应有个性（personality）但没有身份（identity）——不应有观点、不应有目标。用户可自行赋予这些属性
- **用户自主权**：用户始终可以切换成其他声音；公司应尽可能少地替用户做决定
- 实现"无身份人格"在技术上很难——需要大量基础设施来创建合成用户进行测试，并用裁判模型（judges）评估交互是否符合人格目标

**详细内容**：

Mina 反复强调，拥有一个强烈的信念或北极星对产品设计至关重要。他经常与其他正在挣扎于决策的创始人交流，并发现如果你能"插下一面旗帜"说"这是我相信的"，就能让很多事情变得容易得多。

> **Mina**："当某样东西拥有你的上下文、按照你的目标行事、并且是你想要发生的事情的字面延伸时，我认为你就消除了与'非你之物'之间的大量摩擦——那种目标可能不一致、上下文不太对、你不得不过度解释自己的摩擦。"

关于触觉反馈，在如此小的戒指中实现触觉引擎在工程上极具挑战性。但团队知道，要让用户感觉这是"自己的延伸"，就需要利用身体的更多感官。触觉反馈的核心价值在于"闭环"——让你知道你说的话已经被"听到"了，未来还可以用于静默提醒（如到期提醒）或执行中需要确认的场景。

自我声音克隆的故事特别有趣：最初 Stream 使用的是一个现成的语音，感觉不太对。联合创始人 Kirak（Hong）对 Mina 说："如果你认真对待'自我延伸'这个理念，那它应该用你自己的声音说话。" Mina 当时觉得"这一定会很怪异"，但试了之后发现——第一天确实有点不可思议感（uncanny），但到第二天，团队里每个人都爱上了"用自己的声音作为内心声音"的体验。

> **Alex**："这有点吓到我了，老实说。当你让它像我一样说话的时候。"

关于无身份人格（personality without identity），Mina 解释了其中的技术难度：

> **Mina**："它不应该有观点，不应该有目标。从长远来看，用户可以按自己的意愿赋予它这些东西。但实际上，让模型开箱即用就以这种方式运行相当困难。所以我们不得不构建大量基础设施——创建合成用户让 Stream 与之交互，然后用裁判模型来评估交互是否符合我们对人格的目标。"

当被问及使用 Stream 后是否减少了 ChatGPT 或 Claude 的使用时，Mina 说在移动场景中，几乎所有 AI 和笔记应用的使用都转移到了 Stream 上；但在桌面端仍然使用传统 AI 工具。Sandbar 的目标并非与 ChatGPT 或 Claude 竞争，而是提供一个灵活的界面，从移动场景中那些"你在其他平台上不太容易做到的事情"开始。

近期 Sandbar 开放了私有 SDK Alpha 报名，寻找有兴趣在 Stream 之上进行开发的开发者。关于 MCP 集成，Mina 的回答是"简短答案是可以"，但很多东西会立即崩溃——上下文管理、对话管理、延迟、异步处理等问题是 MCP 无法开箱解决的。

## 7. 从神经接口到消费硬件：MIT、CTRL-Labs 与 Meta 的技术积淀

**核心观点**：Mina 的技术背景深深根植于神经接口领域——从 MIT 媒体实验室的神经假肢研究，到 CTRL-Labs 开发的外周神经接口（后被 Meta 收购并发展为 Meta 显示眼镜的神经臂带），这些经历塑造了他"技术应当赋予人自由"的核心信念。

**要点提炼**：
- MIT 媒体实验室研究聚焦于神经假肢——为失去肢体的个体恢复行动能力
- 在应用物理实验室（Applied Physics Lab）实习，专注手臂假肢
- 在 Kernel（Brian Johnson 创办）短暂实习，当时该公司在做脑植入研究
- 加入 CTRL-Labs：开发基于手腕肌肉电磁场的外周神经接口，可通过指尖动作实现"隔空手写"
- CTRL-Labs 被 Meta 收购后，其技术成为 Meta 显示眼镜的神经臂带（neural band）的基础
- Alex 拥有 Meta 显示眼镜和臂带，对各阶段原型印象深刻；Ellis 在 CES 上体验过

**详细内容**：

Mina 解释了外周神经接口的工作原理：当你做出动作时，大脑通过神经系统发送信号；当信号到达肌肉时，肌肉会显著放大信号，产生一个电磁场；手腕上的传感器读取这个电磁场，然后用大规模训练的深度学习模型来解读这些电磁场信号。

> **Mina**："你可以做一些像用指尖在桌面上写字这样细微的动作，我们就能重建你写了什么——这是一种更容易的移动书写方式。"

Alex 补充说 CTRL-Labs 的技术是 Meta 新款显示眼镜所配神经臂带的基础。Ellis 在 CES 期间在赌场地面上试过戴眼镜配合臂带使用的体验。

对于 Mina 来说，神经接口一直关乎"完整地表达自我"：我们脑海中有各种思绪、情感和体验，虽然有艺术等工具来分享完整体验，但仍有很多东西难以用语言表达。神经接口的美在于能让我们"更完整地表达"。

> **Mina**："通过这些经历，我形成了一个核心信念：最好的人机关系，是技术表现得像你的延伸，而不是伙伴、助手、替代品甚至工具。"

## 8. 语音与对话的本质区别：单向捕捉不够，双向交互才是关键

**核心观点**：Mina 观察到市场上很多人将"语音输入"等同于"对话"，但 Sandbar 认为这是本质不同的两件事——单向语音捕捉无法满足移动场景下对记忆召回、补充信息和想法迭代的需求，只有双向对话才能真正释放价值。

**要点提炼**：
- 很多人看到 Stream 就说"我想用它来捕捉"、"当语音输入设备"——但使用后迅速发现单向不够
- 有时需要召回某件事、需要更多信息、需要迭代一个想法（不是一次性完成）
- 现有 UI 几乎没有单向界面——Claude Code 在构建过程中也会请求澄清
- 向用户传递输出（output delivery）是最难的反馈、ML 和基础设施问题所在

**详细内容**：

> **Mina**："我还没有看到太多人在'语音'和'对话'之间做出区分。我们相信语音，但更相信对话。因为我们观察到，很多人看到我们做的东西就说'哦，我想用它来捕捉，我想把它当作语音输入设备。'但真正生活在其中后，你会很快发现——有时你需要回忆某件事，有时你需要更多信息（尤其在移动中），有时你捕捉的那件事不只是一锤子买卖，你实际上在迭代一个想法。"

"这就是为什么让设备能回话变得如此重要。你可以在今天大多数 UI 中看到这一点——几乎没有单向界面。Claude Code 在构建过程中也会请求澄清。所以你需要某种方式将输出传递回用户——而那正是最难的反馈、ML 和基础设施问题所在。"

Alex 对此深有同感。他提到自己从 Monologue 切换到了 WhisperFlow 进行语音听写，虽然有所帮助，但仍存在摩擦。他表示自从第一次看到 Stream 以来，这个产品就持续萦绕在他脑海中。

> **Alex**："直到我们有了眼镜或其他什么，或者不知道项链可穿戴设备会不会真正流行，在我们有某种更容易的可穿戴 AI 输入方式之前——还是有太多摩擦。而且老实说，它不仅仅关乎 AI 部分。对我来说，是记忆召回。太多次我在切换上下文、走出会议或遛狗时想记住某件事，你就是不想打字。"

## 9. 可穿戴设备形态之争：戒指、眼镜与消费硬件的"第三代定律"

**核心观点**：Sandbar 认为戒指解决了语音输入的问题（快速、私密、推拉式控制），但不能解决视觉和某些音频场景的问题，其他形态因素必然是需要的。消费硬件领域存在一个"第三代定律"——通常到第三代产品时，团队才真正对产品感到自豪。

**要点提炼**：
- 戒指擅长：语音输入（快速、私密、push-to-talk 控制）
- 戒指局限：不能解决视觉问题，也不能覆盖所有音频场景
- Sandbar 尝试过多种其他形态，但没有一个达到内部标准
- **第三代定律**：学习错误（第一代）→ 修正改进（第二代）→ 真正自豪（第三代）
- Quest VR 头显、iPhone、消费硬件普遍遵循这一规律
- Ellis 认为 Apple Vision Pro 违反了苹果过去对"就绪感"的标准
- Mina 持不同意见：发布产品（即使不完美）能获得宝贵的市场学习

**详细内容**：

当 Alex 问到 Mina 是否在考虑其他形态因素（眼镜、手环、项链、圆饼等）时，Mina 的回答非常诚实：

> **Mina**："坦白说，如果我有一个确信的答案，我会告诉你'是的，我们正在做 XYZ。'我们尝试了很多东西，但没有一个超越我们的标准。我只能说，我确实相信还需要其他形态因素来解决交互的其他部分。"

Ellis 提到了消费硬件的"第三代定律"：Quest VR 头显是到第三代才真正走向大众市场，iPhone 也是到 3G 版本才真正进入曲棍球棒增长曲线。Mina 对此表示认同——在消费硬件中，你在第一代学习所有错误，在第二代修正它们，到第三代才真正能构建出你引以为豪的产品。

Ellis 随后引出了对 Apple Vision Pro 的批评，认为苹果似乎违背了自己过去对"就绪感"的标准——发布了一个大多数消费者还没准备好的产品。一些商业广告甚至是"假的"（不切实际的使用场景），Apple Intelligence 也有类似问题。对比 OG iPhone——虽然没有 3G 和复制粘贴，但苹果非常清楚地知道路线图，并对产品的核心有足够信心。

Mina 对此持不同看法：

> **Mina**："我会站在另一边。我认为他们可能在正确的时间发布了。他们展示了很多世界首创的能力，他们看到了很多用例，他们分享了对未来的诠释。通常任何产品——尤其是消费硬件产品——都需要几次修订才能走向大众市场。"

Mina 还指出，Sandbar 的一个优势恰恰来自于"不重要"——作为一家小公司在纽约街头戴着各种疯狂的原型，没人在意，这反而给了他们更多自由度去测试和收集真实反馈。

关于可穿戴设备行业，Mina 提出了一个关键观察：历史上从未真正出现过"输入/界面型"可穿戴设备。Apple Watch 可以争论，但本质上是健康产品；耳机聚焦于音乐。在给用户带来新的计算体验方面，Meta 显示眼镜 + 神经臂带可能是第一步。

> **Mina**："任何新的可穿戴设备都需要在叙事、用户教育和可及性上投入大量时间。否则，它们只会停留在少数愿意跳过各种障碍的科技爱好者的玩具。"

Alex 则直率地表示他觉得健康可穿戴设备"很烂"——他已经停用了 Oura，买了 Whoop 一个月后也不戴了。

> **Alex**："这些健康可穿戴设备存在的意义就是让你对自己感觉更糟。哦，你醒来觉得累？你的睡眠评分是 43。让你更糟了。"

## 10. AI 改变一切：从供应链仪表盘到 AI 辅助写作的全面重塑

**核心观点**：AI 对 Sandbar 的影响远超编码——它正在重塑公司运营的每一个环节，从供应链管理到创意输出；同时 Alex 描述了 AI 如何彻底改变了他的写作流程，将编辑-记者关系中的创作模式应用于 AI 协作。

**要点提炼**：
- **Mina 的个人变化**：决策速度大幅加快（用 Stream 或 Claude 快速思考决策）；创作量激增（Claude Code 编程 + AI 图像生成，一度睡眠严重不足）
- **供应链仪表盘案例**：Sandbar 运营负责人独自用 Claude 构建了一个巨型可配置仪表盘——展示每个零件的交货周期、不同关税情景下的价格模拟。"在 Fitbit 和 Peloton，这需要一个团队花数周才能完成。"硬件负责人目睹后"震惊了"
- **Alex 的写作流程变革**：不再自己"写"——而是像编辑指导记者一样与 AI 对话（"这个过渡太敷衍了""用这个角度更新鲜""你的结尾太差了"），零到一的创作过程变得更快更有趣
- **Alex 的写作二分化理论**：未来写作将分化为两极——一极是极度人性化、反 AI 的美丽散文（越来越怪异和有创意）；另一极是功能性的信息媒介（如新闻报道、内容摘要）；中间地带将非常艰难

**详细内容**：

Mina 描述了 AI 对他个人工作方式的两大影响。第一是决策速度：他现在用 Stream 或 Claude 来思考那些以前需要"写下来、沉思、内化很久"的事情。"这些东西非常擅长问你那些能快速把你导向真实意图的问题。"第二是创作量：

> **Mina**："我能用 Claude Code 编程，我能用 AI 图像模型创作图像。有一段时间——和很多人一样——我几乎不睡觉，就一直在做东西。后来不得不告诫自己：'不行，Mina，你需要休息。'"

他举了 Instagram 上一些最近发布的黑白照片作为例子——戒指在身体各种奇特部位保持平衡的图片——都是用 AI 图像工具制作的。

供应链仪表盘案例尤为引人注目：

> **Mina**："几周前我们做了一次供应链审查。我们的运营负责人基本上做了一个巨型可配置仪表盘——展示每个零件的交货周期，让你在不同关税情景和不同价格点之间切换。我们必须做出一些非常复杂的决策，而这个工具让这些决策变得非常容易。我们的硬件负责人看到后目瞪口呆。他说，'在我以前的公司，在 Fitbit 和 Peloton，这需要一个团队花数周才能完成。'"

Mina 强调这个仪表盘的构建过程也"非常辛苦"——需要和 Claude 来回大量交互才能完成。但关键是，一个人就做到了以前需要整个团队的工作。

Alex 则描述了 AI 如何改变了他的写作流程：

> **Alex**："我不再真正'写'了。我跟 AI 对话。它有我所有的联系人，我像编辑指导记者撰写报道一样跟它说话。我说：'不，这不好。这个过渡太敷衍了。用这个角度开头更聪明，因为这个信息更新鲜。把这段移到这里。你的结尾太差了。'就像我在训练一个年轻记者——只不过它是我的 AI。"

他承认 Claude 和 OpenAI 在记忆功能上"还不太行"，但 AI 已经让零到一的创作过程变得更快更有趣，因为他最讨厌的就是面对空白页面和一堆采访记录、然后需要从头组装的过程。

> **Alex**："理想情况下，这释放了我的时间——因为写作过程花的时间更少了——让我能消化更多信息、与更多人交谈、建立起我所报道领域的鸟瞰图。你真正获得的是我对这个领域的策展——以及通过播客和新闻通讯接触到我交谈的那些人。那才是我的竞争优势，不是我的文笔。"

关于写作的未来分化：

> **Alex**："我一直在想这个问题。我认为写作总体上会朝两个方向走——一个是非常人性化的、反 AI 的美丽散文，坦白说会越来越怪异、越来越有创意，不断推动边界；另一个是纯功能性的——一种达成其他目的的媒介或手段，比如在我的情况下是报道，在其他情况下可能是新闻摘要之类的。我认为中间的所有东西都将面临非常艰难的处境。这是我目前的工作假设。"

Mina 对此表示认同，并将话题引向更深层的思考——在 AI 时代，人际互动的独特价值恰恰在于"他者性"（otherness）：

> **Mina**："我尤其喜欢把这项技术定义为'自我'而非'他者'——因为当 AI 是'他者'并与其他人类竞争时，它几乎混淆了与不同于你的人相处的真正好处是什么。我享受与你们交谈的很大原因是你们会挑战我，有着与我非常不同的生活经历。这些东西很难从机器那里获得。"

## 11. 硬件工程的现实：蓝牙连接、金属外壳传音与发货时间表

**核心观点**：Stream 戒指面临比普通健康可穿戴设备更严峻的硬件挑战——需要通过金属外壳传输音频数据（而非仅传心率），这对蓝牙连接和天线设计提出了极高要求。计划 2026 年 4 月开始 Beta 测试，夏季正式发货。

**要点提炼**：
- 蓝牙协议本身已经成熟很多，但要实现"始终连接"仍然非常困难
- Sandbar 有两名全职固件工程师 + 天线顾问 + 连接性顾问
- 独特挑战：与只传心率的健康戒指不同，Stream 需要通过金属外壳传输**音频**——数据量大得多
- 连接测试需考虑房间人数、干扰源等变量
- 这也是 Sandbar "不急着发货"的原因之一
- **发货时间线**：Beta 测试 2026 年 4 月启动 → 第一批正式发货 2026 年夏季 → 第二批也在夏季内
- 定价：银色 249 美元 / 金色 299 美元

**详细内容**：

Ellis 在播客尾声抛出了一个出人意料但非常实际的问题："蓝牙终于不烂了吗？"他指出多年来有太多设备试图做环境感知或语音笔记之类的事情，但蓝牙始终是瓶颈。

> **Mina**："它确实成熟了很多，但仍然相当困难。比如要实现'始终连接'就非常不简单。我们有两名全职固件工程师、天线顾问和连接性顾问。"

Mina 补充说，Sandbar 还面临一些独特的挑战：与只传输心率数据的健康可穿戴设备不同，Stream 需要传输音频——这意味着通过金属外壳发送大量数据，使连接问题更加复杂。测试也很有趣——需要检查不同房间人数和不同干扰水平下连接的变化。

> **Ellis**："蓝牙和 Wi-Fi 是我在科技行业待了这么久以来最讨厌的两项技术。所以我们必须以这个话题结尾。"

关于 Apple 的 U/W 芯片（Ultra Wideband），Mina 承认 Apple 确实在无线通信方面通过自建整个通信管线获得了关键优势——苹果可以在自己的设备之间做一些第三方外设无法立即获得的事情。但 Sandbar 对目前能获得的接口访问权限感到满意——"对我们的应用来说够用了"。

[🕒 约 62:00–66:00]
