## Irina Ghose 开场演讲：信任、AI与印度机遇

### 软件与信任基础设施

> **Irina Ghose**："你点击的每一下，你做的每一次互动，你做的每一个决定——这些由应用程序驱动的行为在表面上看起来非常简单。但作为公民，背后有一个无形的过滤器：我是否信任我正在做的一切？这一切背后都有信任。"

- 软件服务支撑了日常生活的方方面面——订购杂货、购买药品、买保险、叫出租车，每次互动背后都有一个服务链
- 构建者（Builder）已将信任注入了他们所做的一切，构建了整个信任基础设施
- **但如今情况正在发生转变**：软件不再只是执行被编写的代码——软件正在思考、解读和做出决定
- 当软件开始"思考"，信任就不再是理所当然的，必须在设计核心中**主动构建**，而不是事后补救

### 为什么选择 Anthropic

- Anthropic 致力于构建安全、有益的产品，将可解释性（Interpretability）和信任置于核心
- Anthropic 是一家**公益公司**（Public Benefits Corporation），其驱动的影响力与大型企业合作的方式都有良好的治理

### My Little Bit 基金会

- Irina Ghose 15年前创办了名为"My Little Bit"的基金会
- 起初只有一台电脑在一个实验室里，教弱势女孩学习 Excel 和 PowerPoint
- 如今已扩展到 STEM 奖学金、技能培训到就业，影响了数十万名女孩
- 核心感悟：**技术的力量不在于技术本身，而在于它向社区的扩散**

### 印度的 Claude 使用现状

- 印度已成为全球**第二大 Claude 用户基地**
- 自2025年10月宣布印度业务运营以来，**运营收入几乎翻倍**
- **6%的全球 Claude 对话来自印度**
- 50%的对话用于**计算和数学工作**
- 开发者在构建推动行业发展的代码、现代化大型数据库、大规模构建生产级软件
- GitHub 上通过 Claude 进行的提交（Commit）占总量的4%，且仅一个月内就翻了一倍
- 过去六个月，Claude 已支持**10种印度语言**：从印地语到孟加拉语，从马拉雅拉姆语到古吉拉特语
- **Claude Opus 4.6** 是最新的迭代版本

### 企业合作案例

| 企业 | 应用场景 |
|------|---------|
| **Cognizant** | 赋能35万员工，现代化数据库，为客户迁移遗留应用 |
| **Air India** | 优化代码构建，为客户服务构建智能体应用 |
| **Razorpay** | 风险管理和决策支持 |
| **Karya** | 构建多语言接口的核心数据，为明天的AI做准备 |
| **Collective Intelligence Project** | 确保数据质量，支撑多语言AI建设 |

### 印度的采用速度与全球影响力

> **Irina Ghose**："印度组织的采用速度远快于许多其他市场。当印度在最复杂的挑战中、在最复杂的网络基础设施可用性中、在多语言的复杂多样性中解决软件开发问题时，我们就准备好将这些经验推广到全世界。"

- 企业关注的问题已从"是否采用"转变为"以多快的速度大规模采用"
- 明天AI时代的架构师就坐在今天的会场中

## 炉边对话：Irina Ghose 与 Dario Amodei

### 印度开发者社区的技术深度

> **Dario Amodei**："印度最独特的一点可能是 Claude 使用的技术深度。我们一直都能看到休闲消费者使用和专业开发者使用的混合，Claude 一直偏向生产力和专业用途，但在印度这一倾向更加明显。"

- 印度拥有庞大的开发者社区，大量使用 API 和 Claude Code
- Anthropic 在印度的**运营收入在过去四个月内翻倍**
- **Claude Code 的增长可能比整体收入增长更快**
- 这种增长速度反映了 Claude 模型和编码模型的总体爆发趋势，但在印度比世界其他地方**更为极端**

### 印度的AI机遇与独特优势

> **Dario Amodei**："印度统计部门正在构建一种 MCP 统计服务器来查询经济数据和经济统计数据。一般来说，其他地方的政府机构不会这么快行动。"

- **创业活力和技术能力**在印度独一无二，大量人想要找到机会并构建产品
- **纯粹的规模优势**：可以在数亿人中进行实验，快速学习，快速试错
- **政府机构的快速采纳**：如统计部门正在建设 MCP（Model Context Protocol）统计服务器
- 印度拥有世界上**最大的劳动人口**，并有构建服务数十亿人的**数字公共基础设施**（Digital Public Infrastructure）的成熟经验

### 多语言与社会公益应用

- **多语言互操作**是最令人兴奋的应用方向之一——构建在印度多种语言之间互操作的工具
- Claude 正努力覆盖**长尾语言**，包括印度大量的地方语言
- **Adalat AI**：将大量法院案件信息公开可查询，服务普通民众和法律工作者
- **EkStep Foundation**：构建数字基础设施，帮助小农户获得提高生产力的建议
- 核心理念：将AI融入数字基础设施，让生产力和社会效益**扩散到全国各地**

### 企业建议：提前布局未来

> **Dario Amodei**："不要为十步工作流中的第四步构建。尝试做整个工作流，即使AI模型目前还不完全能胜任。两年前人们不理解技术进步的速度有多快，我的建议常遭到质疑，但随着每次模型发布，人们逐渐理解了。"

- **"滑向冰球将要到达的位置"**——不要为AI模型当前的能力构建，要为1-2年后的能力构建
- 如果是大型企业，部署和测试需要时间，应对技术走向**有想象力**
- **最成功的企业**是那些**端到端转型**、以最广泛方式使用AI的企业
- **实例**：在 Co-Work（Claude Code 的非技术版本）发布之前，Anthropic 内部的法律、财务和沟通部门就已经在使用 Claude Code 底层的**智能体引擎**——因为智能体执行任务的概念远超代码范畴
- 那些理解这一点的公司**走在了前面**，他们能够以尚未发布的产品对应的方式使用模型，**看到了产品路线图的前方**

### AI的宏观影响与责任

> **Dario Amodei**："这项技术将以前所未有的速度和程度改变世界。这听起来可能夸张，但如果我们看看改进和扩散采用的速度，这确实在现代历史上没有先例。"

- AI模型既可以被正当使用，也可能被滥用——例如帮助编写代码的模型也可能被用于网络攻击
- 需要客户的帮助来共同维护安全防线
- **为民主而构建**：印度是世界上最大的民主国家，帮助政府更高效地服务人民是技术的重要价值
- **经济公平**：这项技术即使创造巨大价值，也会带来经济动荡——必须确保每个人都能分享收益

### 医疗健康与持久商业价值

> **Dario Amodei**："真正会拥有最大护城河的是那些与物理世界相关的、涉及真实原子世界的、需要独特技能的领域。生物学和医学完全符合这些条件——它是专业领域，你需要大量知识，还要应对整个监管体系。这些艰苦的工作将带来该领域的持久商业价值。"

- **医疗和健康**是最令人兴奋的正面方向
- 纯软件应用容易被复制，但涉及**物理世界、专业知识和监管壁垒**的领域具有持久护城河
- 第二个建议：构建**真正改善人们生活**的产品——不是短期上瘾的，而是符合**长期利益**的
- 这是他在《Machines of Loving Grace》（《爱的恩典之机器》）中阐述的核心理念——构建能够为人类**长期利益**行事的AI智能体和模型

### 人机共存与情商

- 我们正在构建具有人类许多认知能力的系统
- 关键问题：什么需要**人类触感**？什么适合**AI系统**处理？AI最擅长什么？人类最擅长什么？
- 目标是实现一种**人类与AI的混合社会**

### 未来几个月的期待

> **Dario Amodei**："在软件方面，我们开始走向端到端。模型开始执行多阶段工作流，人类可以进入监督者的角色，将自身效率提升10倍到100倍。"

- **科学、健康和生物学**：模型在复杂技术领域的能力正在快速提升
- **软件端到端**：模型开始执行多阶段工作流，人类担任监督者角色，**效率提升10-100倍**
- **金融和法律**领域的模型能力也在快速增强
- 各行业都有**数十家创业公司的空间**，需要"百花齐放"，在不同地区和行业尝试不同方法

## Rahul Patil CTO 主题演讲：Claude 的未来与印度的角色

### 印度数字公共基础设施的成就

**Aadhaar（身份认证）**：
- 2009年，人们认为为一个十亿人口的国家构建生物识别解决方案是不可能的
- 印度建成了 Aadhaar，为**14亿人**提供了数字身份，解锁了大规模基本服务
- 不到十年完成，已成为全球**50多个国家**的模板

**UPI（统一支付接口）**：
- 2016年，印度的通用即时支付被认为不可能
- 印度建成了 UPI，如今每年处理约**2300亿笔交易**
- 这是美国、英国和欧盟所有实时交易总和的**12倍**
- 更便宜、更可靠、更快速——为街头小贩、商务人士、员工和组织**全面服务**

> **Rahul Patil**："印度一次又一次地拥抱新技术，然后找到如何将其扩展到全球级别的方法。"

### 个人背景与加入 Anthropic 的初衷

- 班加罗尔土生土长，父母住在会场2公里处
- 六年级爱上计算机科学，母亲是计算机科学教师
- 就读于 Baldwins、St. Joseph's 和 PES University，后赴美深造
- **职业经历**：参与构建了全球四大云平台中的三个（AWS、Microsoft、Oracle Cloud Infrastructure），之后在 Stripe 担任 CTO 负责支付和金融服务基础设施

> **Rahul Patil**："大约一年前，我在观察AI、使用AI时，看到了它以前所未有的速度改进。我从未见过任何技术改进得如此之快。我意识到这可能是未来一百年中最具变革性的事情。Anthropic 是使命召唤——因为它有正确的雄心、正确的愿景，还有以可扩展和负责任方式执行的正确基础。"

- 在 Anthropic 的职责：统合技术团队、全球规模基础设施和产品
- 目标：交付**改进最快的产品和平台**，使其极度可靠、安全，并与全球构建者合作

### Claude 是什么：模型与产品平台

**两大组成部分**：
1. **基础AI模型**——核心研究，Anthropic 最初成立的目标，仍是日常工作的驱动力。本质上仍是一个**充满好奇的研究实验室**
2. **平台与产品**——通过这些途径体验AI智能。目前仍处于消费这些智能的**极早期阶段**

### 核心理念：帮助人们做真正的工作

> **Rahul Patil**："我们构建模型时问的第一个问题是：这能帮助人们更好地完成真正的工作吗？这是在我们深入研究或产品化之前最重要的测试。"

- **真实案例**：一家非常年轻的创业公司 Sakit 专注于慢性病患者护理，他们使用 Claude 发现了一个能预防住院的明确标志——**这就是驱动力所在**
- 构建模型时，始终思考的是那些**用AI做真正工作、不仅为自己也为终端用户创造价值**的客户

### 专注生产力工作的哲学

- **超级专注**于生产力工作：编码、问题解决、金融、生命科学、生物学、化学、物理学、法律工作、知识工作
- **跨领域技能迁移**的关键洞察：
  - 擅长物理学的模型善于从第一性原理推理
  - 擅长编码的模型善于分解问题并找到解决方案
  - 擅长法律的模型善于逻辑推理
  - 将所有能力结合在一起，就能**解决真实世界的复杂问题**

> **Rahul Patil**："如果一个模型被设计为让你一直开心、从不反驳你，你真的会信任它来做代码审查、帮你推理医疗问题吗？这就是为什么我们对生产力工作的超级专注在构建模型时至关重要。"

### 产品与平台架构

**Anthropic 为构建者服务的理念**：
- 不可能思考到每个国家的每个用例——因此构建**面向最广泛用例的平台**
- 每一层抽象都要**可扩展**——虽不会完整，但提供扩展点让构建者自行适配

**三层需求与产品对应**：

| 构建者需求 | 对应产品/服务 | 说明 |
|-----------|-------------|------|
| "给我原始智能" | **API（第一方 + 多云原生）** | Token 输入、Token 输出；Anthropic 是唯一在所有主要云（AWS、Google Cloud 等）上原生可用、跨多芯片架构的公司 |
| "我要构建智能体，但不想自建基础设施" | **Agents SDK + 定制智能体** | 智能体循环、自主运行、上下文保持、重试、任务边界判断等；Claude Code 就构建在 Agents SDK 之上 |
| "给我现成的解决方案并可自定义" | **Co-Work + Co-Work 插件** | 开箱即用的解决方案，提供插件、技能和 MCP 连接器，便于接入组织上下文和工作流 |

**端到端案例——Adalat AI**：
- 定位为**法庭操作系统**
- 服务对象：使用22种官方语言或上千种方言的普通人
- 挑战：法律文档分散在多种语言、多种上下文和多种历史记录中
- 让普通人可以获取法律信息，让律师更高效，让法庭运行更顺畅
- 是**公私合作**（Public-Private Partnership）的典范，使用了 Anthropic 全栈技术

### 智能体（Agent）的未来

> **Rahul Patil**："什么是智能体？它是你最好的推理伙伴。协作不意味着总是同意你——协作意味着让你更有效。它会在你错误时反驳你，在有足够上下文时自主决策，同时持续学习和成长。"

**理想智能体的特性**：
- 使用第10天应该比第1天感觉更好，第100天比第10天更好
- 随时间推移能够执行更长的任务，拥有适当的自主权
- **现实挑战**：使用150天或200天后开始丢失记忆、以不可预测的方式压缩记忆

**三大核心基础设施组件**：

**1. 上下文智能（Contextual Intelligence）**：
- 不知道时会提问，但逐步学习个人偏好
- 第10天理解你的偏好 → 第20天理解团队偏好 → 第100天理解组织偏好
- 不仅是记忆，还包括**情景记忆和组织记忆**
- 背后有非常先进的算法，将通过 SDK 和 API 对所有构建者开放

**2. 长时间运行执行（Long-Running Execution）**：
- 真实世界中有价值的工作需要**多天甚至多周**的执行
- 科学方法论：提出假设 → 实验 → AB测试 → 学习 → 迭代 → 添加监控
- 智能体需要能够**长时间运行**，拥有适当自主权，同时知道何时寻求帮助
- 智能体应该在需要帮助时感到**心理安全**

**3. 真正的协作（Genuine Collaboration）**：
- **透明推理**：清楚说明为什么这样思考
- 在不合理时**提出反对意见**
- 不仅完成你要求的任务，还帮助你**拓展视野**，了解什么是可能的

### 印度为何准备就绪

**四大理由**：

1. **最乐观的国家**：反复调查显示印度是对AI最乐观的国家，**75%的印度人相信AI会帮助他们取得更多成就**
2. **最具全球视角**：已与世界上几乎所有国家合作
3. **最大的开发者社区**：**2000万 GitHub 开发者**，每三个新 GitHub 开发者中**有一个来自印度**
4. **压缩式创新**：印度的故事不是渐进式采用，而是**压缩式跃迁**——知识开放性和大规模适应力使其**跨越式发展**

> **Rahul Patil**："对我来说，印度的故事不是关于渐进式采用，而是关于压缩。事情发生得非常快速。正是印度的知识开放性和大规模适应力，真正实现了大多数技术变革的跨越式发展。"

**典型案例**：

- 硅谷的任何实验室都无法想到所有可能的用例——这取决于印度本地的构建者
