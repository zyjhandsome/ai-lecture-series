## 任何人都能做软件：十年路径与 Vibe Coding

**核心观点**：Replit 的北极星是**极低门槛的真实软件交付**：部署、托管、获客、扩展，用户不必被技术细节绑架；2024 年 9 月起进一步把「写代码」抽象为自然语言与 Agent。

**要点提炼**：

- **十年三阶段**：先解决**开发环境**（最难的是环境搭建）→ 再解决**部署环境**→ 最后攻克「编码仍令人畏惧」——用 **AI Agent** 在后台生成代码，用户侧主要是自然语言交互。
- **Vibe Coding**：**Amjad Masad** 称 Replit 在 2024 年 9 月成为首批把代码整体抽象掉的产品之一；背后仍有编码 Agent，但界面重心是自然语言。
- **Agent 4 之后**：加入**设计交互**：可写评论、在**画布**里拖拽；并强调未来交互未必永远是文本，可能走向**多模态**（甚至音视频），目标是让人以自然方式表达想法并「近乎魔法般」变成**真实、安全、可扩展**的软件，而非玩具。

**详细内容**：

**Amjad Masad** 用一句话概括产品野心：只要具备**读写**能力的人，就能带着想法进来，离开时拥有**已部署、已托管、有流量、能扩展**的应用，而不必操心构建过程中的技术琐事。他把这条路线放在 **10 年**的时间尺度上：从消除环境摩擦，到消除部署摩擦，再到用 Agent 消除「面对代码」的心理与技术摩擦。

关于 **Agent 4**，他补充：**内置设计能力**与画布式协作，让用户可以用多种模态与 Agent 互动；并预判行业不会长期停留在纯文本。

## AI 原生建造者：为什么不是「只服务工程师」

**核心观点**：许多传统开发者**享受折腾工具链**（像工匠自制工具），而 Replit 的最大价值更多落在**技术相邻角色**与**领域专家**身上；因此 **2023** 年起公司把目标明确为**不主攻传统开发者**——尽管公司外表仍像 devtool 公司，但服务的是**创作者**与新一代 **AI 原生开发者**。

**要点提炼**：

- **个人动机**：**Amjad Masad** 自述从小编程，但真正兴奋点是**创造**与**创业**；他认为开发者工具的长期趋势是**越来越重**，从 **BASIC** 时代到大学毕业搭建 Web 应用变成噩梦，促使他做「更有愉悦感、更围绕创造」的工具。
- **谁最受益**：**产品经理**（写过代码但不想配环境/部署）、**设计师**（有想法但常被工程资源卡住）、以及**创业者**——**Amjad Masad** 说他们让他想起年轻时的自己：**有想法、有热情，却被「必须先变技术」卡住**。
- **代际变化**：新一代用户是 **AI 原生**，可以在不必理解系统每个组件的情况下创造软件。
- **历史类比**：**Andrew Miklas** 提 **VB6** 降低门槛；**Amjad Masad** 回应：**VB6 比折腾 React 与 Webpack 更友好**，行业「变难了」，他想把编程体验拉回到更直觉的状态（他半开玩笑地说类似「让编程再次伟大」）。

**详细内容**：

**Amjad Masad** 解释：并非贬低热爱配置的传统开发者，而是观察到一个分化——**最从产品里榨出价值的人**，往往不是最沉迷工具链的人。公司使命也曾从「让编程可及」演进到「创造十亿新开发者」，但随着 AI 与产品成熟，他们更明确地把重心转向**创始人与领域专家**。

## 人们在 Replit 上真实构建什么

**核心观点**：实际用例横跨**个人软件**、**企业软件**与**创业者产品**；关键杠杆是**离问题最近的人能亲自把产品做出来**，从而释放大量被忽视的垂直需求与效率。

**要点提炼**：

- **医疗 / 理疗垂直案例**：一位**物理治疗师**在**筋膜与筋膜释放**子领域积累深厚方法论，需要复杂应用（扫描身体、活动度测量、在 **3D 身体模型**上追踪进展）。他们曾**花数十万美元外包**全球开发却受挫，最终在 Replit 自建；**Amjad Masad** 评价为其见过最好的健康科技应用之一，并强调**领域专家亲自构建**的力量。
- **垂直 SaaS 与「世界很大」**：例如维护泳池的家族生意背景创始人做泳池业 SaaS；另一位给体育俱乐部做软件，现场展示俱乐部仍在用 **MS-DOS** 级系统——说明硅谷视角外的**盲区**巨大；当任何人都能做软件，**生产力与财富创造**可能在广泛经济部门发生。
- **个人 / 家庭软件**：例如韩国家长为孩子罕见病做管理软件；穿戴设备数据汇总；家庭场景里**家务排行**类 iPad 墙应用等。
- **企业侧两类需求**：
  1. **产品开发加速**：AI 时代「必须更快」的压力不只在工程师——产品、设计也能构建；**Whoop** 客户故事称**可试验的想法数量提升一个数量级**（从约 100 个想法只能试 **5** 个，到能试 **50** 个）。
  2. **内部工具与业务流程应用**：例如 **RevOps** 汇聚 CRM、数据团队、**Gong** 等多源数据，传统上被工程资源或采购 **SaaS** 卡住；而不断 onboard 新 SaaS 会形成**无法编程打通**的数据孤岛。用户自建**报价配置器（CPQ）**等，可**省下数十万到数百万美元** SaaS 支出（**Amjad Masad** 口述量级）。

**详细内容**：

这一段用大量「来自真实世界边缘」的案例，支撑 **Amjad Masad** 的判断：**软件需求远未被满足**，一旦建造门槛下降，**最懂问题的人**会把质量与迭代速度带到传统外包难以匹配的水平。

## 增长与进入企业：PLG、内部冠军与「可能性教育」

**核心观点**：非开发者市场的扩张，类似当年 **Stripe**、**PagerDuty** 等让**开发者能在组织内引入工具**的路径——如今 **PM、设计、运营**也能成为采购与扩散节点；增长上仍信 **PLG（产品驱动增长）**，企业侧大量是**销售辅助 + 内部冠军**，以及面向领导层的**教育 / 布道**。

**要点提炼**：

- **周末玩出来的工作场景**：个人场景与工作场景重叠——人们在周末做个人 App，一旦意识到「这也能用代码解决」，会产生**认知切换**：看世界的方式变成「我能修这个、我能做这个」。**Andrew Miklas** 共鸣自己学编程时的体验。
- **PLG**：把产品做到极致、值得推荐给朋友；配合推荐机制等经典手段。
- **企业销售**：很多人把工具带进公司后成为**冠军（champion）**；销售团队帮助其说服上级，组织 **hackathon** 制造更多冠军，并对管理层做 **AI** 相关教育——**Amjad Masad** 形容这更像**布道与教育**，而非旧式推销独白。
- **自上而下入口**：也有公司主动评估各类 **Vibe Coding** 工具，Replit 以**安全与合规**等「企业基本功」作为对比优势之一。

**详细内容**：

**Amjad Masad** 强调：关键转折是让人理解「我能用软件解决问题」。一旦跨过门槛，扩散会沿着**人类网络**发生，而不是只靠传统广告投放。

## 能构建什么、还不能构建什么；集成与「技能 / MCP」

**核心观点**：对大多数创业者，**SaaS、消费者产品、自动化产品**可以「舒适地在 Replit 上构建」；但**新云平台、全新机器学习系统**并非当下聚焦；同时通过 **MCP** 与「技能」生态把集成做成 Agent 可检索、可安全接入的能力包。

**要点提炼**：

- **边界**：**Amjad Masad** 明确：想做**全新云平台**或**全新 ML 系统**不是 Replit 今日主战场；但工具底层给 **VM** 与通用 Agent，有技术的人仍可构建复杂事物。**完全 Vibe Coding、不关心技术细节**的用户，也有大量官网与营销中的成功案例可参考。
- **典型可成范围**：消费者 App、垂直 SaaS、内部工具、自动化；以及「**Replit 原生代理/外包机构**」——有人反馈比传统外包**便宜 60–70%** 且更高效（口述）。
- **集成机制**：与伙伴共建集成；同时进入「**skills / MCP** 革命」——各公司发布 **skills** 与 **MCP**，Replit **审核后**接入产品。用户说「我要接 **Stripe**」时，系统检索已沉淀的技能与代码并注入上下文。**Amjad Masad** 用《黑客帝国》里 **Neo 下载技能**比喻：Agent 发现「我会开直升机了」。
- **安全**：投入大量精力确保第三方技能**安全、可信**。

**详细内容**：

这一段把 Replit 的「能力边界」说得更可验证：**不是万能工业母机**，但在**应用层软件与自动化**上，**Amjad Masad** 对「零技术背景的创业者也能成」表达很强信心。

## YC、三个月密集迭代与复利增长

**核心观点**：YC 带来的首要心智是**三个月极高强度能完成多少**；以及新产品线用**每周复利**（例如 **7% 周环比增长**）自举；个人层面则打开了融资与人脉网络。

**要点提炼**：

- **三个月专注**：**Sam Altman**（当时仍负责 YC）在批次开场说：接下来三个月告诉朋友你会「消失」——不能帮人搬家，但极致专注就能做成很多事。**Amjad Masad** 团队在 YC 期间用**白板 + Demo Day 倒计时 + 每日更新清单**。
- **入营前后对比**：进 YC 时 Replit 还像「命令行里写点代码跑跑」；结业时已具备 **Web 开发**、**早期托管**、**代码智能提示**等大量 IDE 能力——**三个月**内显著变「更像完整产品」。
- **内化到 Replit 节奏**：如今每次 **Agent 大版本**更像 **4 周**冲刺：全员到办公室，**三餐 + 24/7 咖啡**，打极高野心目标——**Amjad Masad** 称之为 YC 心态延续。
- **增长公式**：新产品/新计划回到 **Paul Graham** 常说的约 **7% 周环比增长**作为 bootstrap 框架（**Amjad Masad** 表示不确定 **PG** 为何选 **7** 这个数字，但认为好用）。
- **融资与人脉**：进 YC 前约只融到 **50 万美元**，VC 门难敲；Demo Day 后接触面大开。他们曾被 YC **拒绝三四次**，后因 **Paul Graham** 与 **Sam Altman** 在 **Hacker News** 上看到而入选。**Amjad Masad** 主动求介绍见 **Mark Andreessen**，在对方家里早餐路演，**a16z** 领投种子轮；他认为若无 YC，网络与资金可能撑不住，**公司未必活到今天**。

**详细内容**：

**Andrew Miklas** 以「外人被接纳」共鸣这段经历；**Amjad Masad** 强调 YC 教会的不是「鸡汤」，而是可重复的**强度、节奏与复利指标**。

## 从 Vibe Coding 到自主 Agent：Agent 3、Agent 4 与路线图节奏

**核心观点**：自 **2024** 起，**Amjad Masad** 团队以「**AI 能力大约每年两次台阶式跃迁**」来对齐路线图，因此大约**每六个月**发布一代 Agent；**Agent 3** 押注长时自主运行，**Agent 4** 押注并行、异步设计与协作。

**要点提炼**：

- **2024 时间线**：年中 **Claude** 与 **Claude Sonnet** 让「生成大量代码」变得可行（相对 **GPT-3.5** 的「偷懒」）；年末看到实验室在**长程推理**上的早期信号。
- **Agent 3**：目标是当时市场上**最自主**的 Agent；平台为此重写后端，支持用户不在电脑前仍**长时间跑容器**完成任务——**Amjad Masad** 认为即便「真·自主」到 **11–12 月**才更成熟，Replit 在 **9 月**已展示世界前进方向。
- **Agent 4 的四个支柱（口述结构）**：
  1. **并行 Agent**：自主模式的痛点是「大提示扔进去后你只能坐着看」；需要回答 **下一步做什么**——并行与异步让用户同时推进设计、规划与其他工作。
  2. **异步设计 Agent + 画布**：一边构建一边设计下一页/下一功能；准备好再**分叉到新线程**启动实现。
  3. **团队并行**：并行架构自然延伸到协作——他人加入会话可**fork 新 VM**并行工作；编排器也能在同一聊天窗口拆分任务。
  4. **跨资产统一上下文**：网站、移动 App、演示文稿、视频等都应共享项目上下文；例如 Web App 做好后一句「做个移动 App」可生成并铺在画布上；部署时 **Web** 与 **TestFlight / Android** 等渠道联动——**Amjad Masad** 总结为「**在 Replit 上运营整家公司**」。

**详细内容**：

他还提到 **2025**：上半年 **Vibe Coding** 浪潮，下半年 **Opus 4.6** 等触发更偏**自主**的浪潮，并点名 **OpenClaw** 等现象——用于解释「为什么他们的版本节奏跟着模型能力走」。

## 未来技能：后提示词世界、可能性嗅觉与韧性

**核心观点**：行业正走向**后提示词（post-prompting）**——更多是给 Agent **高层目标**（例如「优化我的营销漏斗」）；对人来说，关键是**知道什么可能**、保持试用与学习、以及**想法生成与创造力**。

**要点提炼**：

- **高层指令**：未来或许能对 Replit 说「每天给我做一个 SaaS 公司、做营销实验、直到产生收入」——**Amjad Masad** 认为**几乎接近**可用（时间表上指向 **Agent 5** 或更早）。
- **仍重要的能力**：
  - **理解可能性**：多玩工具、保持「玩耍心态」、持续跟进行业变化；曾经「总在网上」可能是缺点，现在反而重要。
  - **别放弃**：今天做不成，**一个月后再试**；他也建议用户隔几周重试一次需求。
  - **想法生成**：AI 会更擅长帮你想点子，但人仍要**持续提出问题**、判断世界需要什么；小创业者常经历产品周期：**某段时间很赚钱，随后过气**，需要持续创造力（他举 **Pieter Levels** 式路径作为例子口径）。

**详细内容**：

这段把「技能」从**写提示词**推进到**定义目标、迭代假设、保持输入与韧性**，与他对 Agent 路线图的判断一致。

## 若今天重来会怎么做；仍在等待的技术解锁

**核心观点**：复盘创业，**文化**与对 **PMF（产品市场契合）** 的自欺是最大的坑之一；技术上，他长期期待 **Computer Use** 模型更好，并对**持续学习（continual learning）**仍处早期有清晰遗憾。

**要点提炼**：

- **文化与组织**：曾**搞砸文化**并经历**裁员与重置**（口述）；若重来希望少犯类似错误。
- **PMF 诚实**：获得任何用户与收入都值得庆祝，但**真 PMF**是「爆炸性」的不同；要在「看起来还行」时更早纠偏。
- **Computer Use**：他等待这类模型变强，但对进展**失望**；困惑于数据获取理应不难（对比自动驾驶），可能与**语言更易压缩**、**视频流更难**有关。现实 workaround 是：**Coding Agent** 能调用 API、写脚本，间接完成许多「本需鼠标键盘」的事——例如表格 Agent 变强与编码变强联动。
- **对 Replit 的意义**：作为 Vibe Coding 平台，需要能**测试用户应用**并反馈「功能对但体验差」等问题——**品味（taste）**很重要；因此他们大量 **prompt 与增强 Computer Use** 来服务内部测试 Agent。
- **持续学习**：现在常见 hack 是把学到的东西写进 **skill 的 Markdown 文件**等；但「在组织里部署后越用越懂该组织」的真·持续学习仍远。

**详细内容**：

**Andrew Miklas** 追问「未来公司若 Agent 一路到底，人还剩什么」，引出下一章 **Amjad Masad** 的「两类工作」框架。

## 未来公司：两类工作与「全员创始人心态」

**核心观点**：未来公司大体由**建造者**与**销售**构成；销售会更像**转型顾问式**的**教育者 / 布道者**，且因人类信任机制而**更难被完全替代**；组织内部会出现更多**业务通才型内部创业者**，带着模糊使命「让公司更好」，用 Agent 解决一线问题。

**要点提炼**：

- **销售为何还在**：别的公司仍希望与人交流、向人学习；**Amjad Masad** 认为销售因此成为更**可防御**的岗位之一（措辞为「更 defensible」）。
- **建造者层级上移**：自动化永无止境；人的工作会像从「人工计算员」到「计算机操作员」再到「软件」一样**不断抬高抽象层**。
- **业务通才 + Agent 副官**：抽象未来组织——几乎人人像创始人：早上思考如何提升公司成功与收入，巡视公司发现问题，**创建或委派 Agent** 解决。
- **内部案例**：Replit 内部有「**Vibe Coding 驻场团队**」，使命很宽泛——不是传统产品/销售那样边界清晰，而是**到处让公司更好**：例如去支持团队，围绕 **Zendesk** 等工具梳理痛点，做队列优先级可视化，**CSAT**（客户满意度指标口径）上升；再到 HR 做 onboarding 与福利信息平台等。

**详细内容**：

这与 **Agent 4** 的「并行 + 协作 + 画布」叙事形成闭环：**人**负责目标、品味、客户理解与组织动员；**Agent**负责长时与并行的工程实现。
