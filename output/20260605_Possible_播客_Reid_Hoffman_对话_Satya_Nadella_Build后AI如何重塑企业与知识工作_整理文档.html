Build 后对话 Satya Nadella：AI 不是技术栈，而是企业的未来

Satya Nadella 认为 AI 已非「买几个模型」的技术项目，而是与人力资本绑定的代币资本与隐性知识——企业须自建可控的爬坡机器（hill-climbing machine）。

Possible Podcast — Reid Hoffman with Satya Nadella (post–Microsoft Build 2026) 2026-06-05 ~19 分钟阅读

核心导读

Satya Nadella 认为 AI 已非「买几个模型」的技术项目，而是与人力资本绑定的代币资本与隐性知识——企业须自建可控的爬坡机器（hill-climbing machine）；今日软件开发的 Agent 化只是全体知识工作的预演，社会许可最终靠可触摸的成果而非口号赢得。

Build 与「新 BASIC」。 Build 2026 的核心叙事不是又一款前沿模型，而是帮每家公司造自己的爬坡机器：明确目标、评测（eval）与奖励，把轨迹训练成 IP，而非泄露给外部模型厂。

企业运营与 CEO 职责。 Microsoft 设想 2 万 员工与 2000 万 Agent 同环运转；Agent 365、沙箱身份与 Foundry 的 asserts 把可观测、可审计、可治理做成平台层。非科技 CEO 也需回答：昨夜的工作如何沉淀为你拥有、你控制的代币资本。

科学、主权与社会契约。 Reid Hoffman 借 Manas AI 的「化学 Move 37」说明 AI 推理模式可发现人类未触及的分子；国家「主权 AI」应学企业——在比较优势上爬坡，而非防火墙式自闭或单模型依赖。教宗通谕与行业话术形成对照：人文尊严、全球南方收敛与可验证益处，是赢得公众信任的唯一路径。

诗歌、压缩与人类经验：Satya Nadella 的人文底色

核心洞察

诗歌是「人类经验的压缩形式」——与代码同源，塑造 Satya Nadella 看待 AI 时代的人文视角。

深度解析

Reid Hoffman 以 Greylock 旧地重聚开场，先问 Satya Nadella 家中大量诗集的来源。Satya Nadella 回忆中学英文课的标准诗集（已绝版），从 Shelley、Wordsworth 到印度诗人 Sarojini Naidu；他自认并非深研诗歌的学者，「懂诗的名声远超真懂」，但仍把诗歌当作 bored 时的读物。

核心比喻：诗歌像代码，是压缩（compression）的最佳形式——文学捕捉人类经验，诗歌是其极限压缩。Satya Nadella 最共鸣的是成长于印度 Hyderabad 的 Urdu 诗，尤其 Ghalib、Faiz，亦受 Rumi 影响；他就读的高中同时教 English、Hindi、Sanskrit（「像这里的拉丁语」）、Telugu、Urdu、Persian，第二语言选 Sanskrit，同学则分流各语种——Reid Hoffman 自嘲「只会一种语言的是美国人」。

Build 2026：新 BASIC 是「爬坡机器」

核心洞察

Microsoft 五十年前的 BASIC 解释器问「别人能建什么」；AI 时代答案是每家企业自有的爬坡机器——目标、评测与奖励才是新 IP。

深度解析

Reid Hoffman 将 Microsoft 首产品 BASIC 解释器与 Build 2026 对照：当年框定「他人能在此基础上构建什么」，如今 AI 时代的新「解释器」是什么？

Satya Nadella 称这仍是公司 DNA 的两问：别人能建什么？因此我们该建什么？ AI 时代的答案是爬坡机器（hill-climbing machine）：给定目标、结果与 eval，用数据与强化奖励学习如何达成。Build 2026 整场大会并非「又一款前沿模型」，而是帮助每位开发者、每家 AI 原生创业公司与企业，构建自己的爬坡机器以在前沿运营。

新 IP 在于：你真正在意的目标集合、如何评测、如何奖励——其余「相当机械」。知道用什么数据训练、如何奖励，是下一层 IP 诞生之处。

Reid Hoffman

2

Satya Nadella

「新 BASIC 是爬坡机器——把目标、结果、eval 拿来，用数据和强化奖励学习如何达成。」

3

Satya Nadella

「知道该用什么数据训练、如何奖励——下一层 IP 就在那儿诞生。」

代币资本、隐性知识与企业数据主权

核心洞察

未来经济由人力资本与代币资本共同塑形；企业隐性知识正经人类轨迹编码进模型权重——泄露是单向门。

深度解析

Reid Hoffman 追问企业如何在拥有前沿智能的同时守住数据。Satya Nadella 的框架是：无论 Microsoft、初创还是百年银行，都需处理人力资本与代币资本（token capital）的交织与复利。

关键不在「数据聚合」，而在企业隐性知识（tacit knowledge）：独特运作方式、判断力、品味——今日主要在人力与其数字遗存中。AI 模型可通过人类工作轨迹提取并编码进权重。企业须格外留意：人类与数字资产如何共同训练作为 IP 保留的模型，而非泄露——泄露是单向门，百年积累的独特性可能一夜蒸发；资产负债表上甚至没有「隐性知识」科目，但我们默认有人力在就有。

Satya Nadella 指模型公司建「健身房」式奖励环境，雇佣曾在你们公司工作过的员工——这本身说明不该发生的事。范式应变为：欢迎模型进来，但在你控制的机器里爬坡；数据是喂给模型的 context，收集人机与 Agent 协作轨迹，持续闭环且不泄露。

1

Reid Hoffman

「企业需要自己的前沿智能，又要控制数据——该怎么想？」

2

Satya Nadella

「经济将由人力资本和代币资本共同塑形……隐性知识是独特运作、判断、品味——今天主要在人力里。」

3

Satya Nadella

「若泄露，是单向门——你可能花了一百年才有的东西，我们甚至说不清它是什么。」

4

Satya Nadella

「模型应在你控制的机器里爬坡；收集轨迹，持续闭环，不让它泄露。」

AI 员工与未来工作：从 IDE 到 ADE 的演进预览

核心洞察

编程 Agent 从补全→聊天→任务→长时自主的链条，将复刻到全体知识工作；新瓶颈是百个 CLI Agent 的认知负载。

深度解析

Reid Hoffman 延续微软董事会讨论：AI 员工——不只工具放大人类，还配置「专家级员工」——企业担心员工所承载的隐性知识是否外流。

Satya Nadella 用 80 年代类比：若有人说「40 亿打字员每天醒来打字」，当时会觉得荒谬——我们有打字员池、幻灯片池；但我们发明了知识工作，人人打字产出数字制品。我们尚未在概念上对齐的新问题是：Microsoft 约 2 万员工对 200 万乃至 2000 万 Agent，全在同一闭环——产生什么隐性知识？人机与 Agent 间传递什么制品？

编码是最佳观察场：IDE 内代码补全（像拼写检查、IntelliSense）→ 聊天引入 Stack Overflow 知识 → 推理模型与 Agent 模式小任务 → 长时自主「发射后不管」完成整份 PR。该过渡将发生在所有工作。Copilot 已有聊天、co-work；Build 宣布 Scout 与 Autopilot。

新痛点：人人开 100 个 CLI Agent 会话，线性聊天导致管理认知负载爆炸——于是回到类 IDE 的 ADE（Agentic Development Environment，智能体开发环境）：新 GitHub 应用像 Agent 收件箱，跨仓库微操宏观委托；Canvas 功能可用看板可视化 PR，便于人与 Agent 交互。

研究方向：模型需更擅长保持航向与可引导性（steerability）——不只指令跟随，还要真正可引导，才有信心。未来工作=人力资本与 AI 资本共创隐性知识，如同知识工作由数字制品与人力共创。

Agent 365：可观测、安全、治理与长时 Agent 断言

核心洞察

企业 AI 第三层是可管理性与安全：Agent 清单、身份、沙箱、策略，加长时运行时的断言（asserts）防运行脱轨。

深度解析

Reid Hoffman 问除体验层、爬坡机器外，企业还缺什么结构——例如「企业里的安全是什么？」

Satya Nadella 列三层：体验层（人类 Canvas、ADE 等）、爬坡机器、可管理性与安全——从可观测性始。Agent 365：Microsoft 可能有 2000 万 Agent，须先建立清单——它们是谁、在做什么、推理轨迹须完全可检视、可审计。Agent 还能生成并执行代码，需受策略约束的执行环境（文件系统、网络访问）→ 给 Agent 身份、沙箱、策略。

Agent 365 延伸 Entra（身份）、Defender（安全）、Purview（自动标注数据以保护）。长时 Agent 方面，编程语言早有运行时可验证性模型；Foundry 新增 asserts：在运行中断言边界，而非仅靠 guardrails 分类器——让执行路径不偏离轨道。平台、运行时、安全、管理、可观测层正在快速工程化。

CEO 必须领导：AI 是企业的未来而非 IT 项目

核心洞察

Fortune 称 Satya Nadella 像 AI 团队里的创业 CEO；更广义的教训是：非科技 CEO 也必须能点名自己的代币资本。

深度解析

Reid Hoffman 引用 Fortune 称 Satya Nadella 在 Microsoft AI 团队内像创业 CEO，并问各业 CEO 在 AI「再创业时刻」该做什么。

Satya Nadella：科技 CEO 必须深入技术栈，对未来有根本世界观，在成为主流共识之前就下判断——行业转型相当二元，CEO 须身先士卒、射门，知转型之严酷。非科技 CEO 过去可专注银行/医疗等主业，靠科技顾问——他正在修正先验：行业与 CEO 社群广泛尚未清醒，仍满足于发新闻稿、秀 8 个 Agent 就宣称有 AI 策略。

AI 不是一类技术，而是企业的未来——如同不能说「我不知道公司的人力资本」。CEO 须深入理解：你的代币资本是什么？ 能否具体说：基于昨夜/全公司运作，我们如何把劳动翻译成你拥有、你控制、你创造并设系统让其复利增长的知识——可以是上下文、技能、模型权重，不限形式，但必须可识别。

这与 mobile/PC/cloud 不同——那时 IT 部门懂供应商即可；现在是：当 AI 基本掌握你们行业所需的一切，行业结构如何变？从此理解这不仅是技术，而是企业的根本变革。

科学发现前沿：Manas AI 与 Reid Hoffman 回归创始人模式

核心洞察

深度解析

Reid Hoffman 问 LLM 下一前沿。Satya Nadella 转向 Reid Hoffman 投入的科学：Manas AI（与 Siddhartha Mukherjee 等共创）——若编码/知识工作的进展发生在科学上，社会许可会高得多；他希望顺序反过来——先做有益社会的科学发现，公众会更把 AI 当助手。

Reid Hoffman 透露：近月 Manas AI 进展令其需回归创始人模式——内部称化学领域的 Move 37（借 AlphaGo 神之一手），已出现可能针对多种疑难癌症的化学结构，顶尖计算化学家称「从未见过，可能有效」；仍处早期。Sid、Uzair 与 Reid Hoffman 讨论后，决定 10 年微软董事会任期至年底过渡——与 LinkedIn、OpenAI、GitHub 等合作荣幸巨大，但须重新投入公司；仍将与 Satya Nadella 密切合作。

Satya Nadella 称董事会上 Reid Hoffman 会被想念，但对其回归创始人模式感到兴奋——此刻社会影响更被广泛感受，正是所需。

异质智能、认知覆盖与代币效率

核心洞察

AI 推理模式与人类不同却可放大能力；认知覆盖（cognitive coverage）与代币智能是知识工作者的新核心技能。

深度解析

Reid Hoffman 用「外星智能」框架：高度模仿人类却推理模式不同——Manas AI 的化学人类未发现。Satya Nadella 谈归纳/演绎与 AI 的认知闭环：同事在 GitHub Copilot 写 cognitive coverage 技能——像测试覆盖率，Agent 做完活后出测验，让人类从 Agent 行为形成演绎理解；模仿游戏在一侧，认知覆盖在另一侧——「我们已认知覆盖 AI 做了什么」。

Reid Hoffman 接代币资本：现代知识工作者须谋划 AI Canvas 编排，且在资本配置边界内——代币可像「百万美元马桶盖」无效挥霍。Satya Nadella 称代币丰裕时代更值钱的是如何用代币：谁更高效达成重要结果谁领先——靠 eval/评分量表 捕捉只有你能定义的高品味。

Build 案例：零售商/快消处理贸易促销索赔——可用 Agent 自动化，但别用前沿模型解非前沿问题（他的铁律）。这是可重复、确定性的工作流，非材料科学发现——可用 MAI 5B 等小模型，用轨迹做 RL 爬坡，性能可超过前沿模型提示。Satya Nadella 强调对系统边界与特性的理解是溢价能力。

2

Satya Nadella

3

Satya Nadella

「一条铁律：别用前沿模型做非前沿问题。」

主权 AI 隐喻：国家应学企业，别落前沿之外

核心洞察

国家「主权」最好体现在本国企业能在代币经济中繁荣；闭关或单模型依赖都会失去比较优势。

深度解析
