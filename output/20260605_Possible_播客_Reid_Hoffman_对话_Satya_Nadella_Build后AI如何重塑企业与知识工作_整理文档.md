## 核心导读

> **全文论点**：**Satya Nadella** 认为 AI 已非「买几个模型」的技术项目，而是与人力资本绑定的**代币资本**与**隐性知识**——企业须自建可控的**爬坡机器（hill-climbing machine）**；今日软件开发的 Agent 化只是全体知识工作的预演，社会许可最终靠可触摸的成果而非口号赢得。

**Build 与「新 BASIC」。** **Build 2026** 的核心叙事不是又一款前沿模型，而是帮每家公司造自己的**爬坡机器**：明确目标、评测（eval）与奖励，把轨迹训练成 IP，而非泄露给外部模型厂。

**企业运营与 CEO 职责。** **Microsoft** 设想 **2 万** 员工与 **2000 万** Agent 同环运转；**Agent 365**、沙箱身份与 **Foundry** 的 **asserts** 把可观测、可审计、可治理做成平台层。非科技 CEO 也需回答：昨夜的工作如何沉淀为**你拥有、你控制**的代币资本。

**科学、主权与社会契约。** **Reid Hoffman** 借 **Manas AI** 的「化学 Move 37」说明 AI 推理模式可发现人类未触及的分子；国家「主权 AI」应学企业——在比较优势上爬坡，而非防火墙式自闭或单模型依赖。**教宗**通谕与行业话术形成对照：人文尊严、全球南方收敛与**可验证益处**，是赢得公众信任的唯一路径。

## 诗歌、压缩与人类经验：Satya Nadella 的人文底色

### 核心洞察

> 诗歌是「人类经验的压缩形式」——与代码同源，塑造 **Satya Nadella** 看待 AI 时代的人文视角。

### 深度解析

**Reid Hoffman** 以 **Greylock** 旧地重聚开场，先问 **Satya Nadella** 家中大量诗集的来源。**Satya Nadella** 回忆中学英文课的标准诗集（已绝版），从 **Shelley**、**Wordsworth** 到印度诗人 **Sarojini Naidu**；他自认并非深研诗歌的学者，「懂诗的名声远超真懂」，但仍把诗歌当作 bored 时的读物。

核心比喻：**诗歌像代码，是压缩（compression）的最佳形式**——文学捕捉人类经验，诗歌是其极限压缩。**Satya Nadella** 最共鸣的是成长于印度 **Hyderabad** 的 **Urdu** 诗，尤其 **Ghalib**、**Faiz**，亦受 **Rumi** 影响；他就读的高中同时教 **English**、**Hindi**、**Sanskrit**（「像这里的拉丁语」）、**Telugu**、**Urdu**、**Persian**，第二语言选 **Sanskrit**，同学则分流各语种——**Reid Hoffman** 自嘲「只会一种语言的是美国人」。

### 对谈实录

**Reid Hoffman**：「你家里有多少诗集，很多人不知道——能说说你为什么被诗歌吸引吗？」

**Satya Nadella**：「诗歌也许是人类经验最好的表达……广义看，它比历史更能捕捉人类经验，而诗歌是压缩形式。」

**Satya Nadella**：「我把它甚至比作代码——压缩的最佳形式。」

## Build 2026：新 BASIC 是「爬坡机器」

### 核心洞察

> **Microsoft** 五十年前的 **BASIC 解释器**问「别人能建什么」；AI 时代答案是每家企业自有的**爬坡机器**——目标、评测与奖励才是新 IP。

### 深度解析

**Reid Hoffman** 将 **Microsoft** 首产品 **BASIC 解释器**与 **Build 2026** 对照：当年框定「他人能在此基础上构建什么」，如今 AI 时代的新「解释器」是什么？

**Satya Nadella** 称这仍是公司 DNA 的两问：**别人能建什么？因此我们该建什么？** AI 时代的答案是**爬坡机器（hill-climbing machine）**：给定目标、结果与 **eval**，用数据与强化奖励学习如何达成。**Build 2026** 整场大会并非「又一款前沿模型」，而是帮助每位开发者、每家 AI 原生创业公司与企业，构建自己的爬坡机器以在**前沿运营**。

新 IP 在于：你真正在意的目标集合、如何评测、如何奖励——其余「相当机械」。**知道用什么数据训练、如何奖励**，是下一层 IP 诞生之处。

### 对谈实录

**Reid Hoffman**：「**Microsoft** 第一个产品是 **BASIC 解释器**；**Build** 上你们把未来框定在『别人用 AI 能建什么』——AI 时代的新 **BASIC** 是什么？」

**Satya Nadella**：「新 **BASIC** 是爬坡机器——把目标、结果、**eval** 拿来，用数据和强化奖励学习如何达成。」

**Satya Nadella**：「知道该用什么数据训练、如何奖励——下一层 IP 就在那儿诞生。」

## 代币资本、隐性知识与企业数据主权

### 核心洞察

> 未来经济由**人力资本**与**代币资本**共同塑形；企业隐性知识正经人类轨迹编码进模型权重——泄露是单向门。

### 深度解析

**Reid Hoffman** 追问企业如何在拥有前沿智能的同时守住数据。**Satya Nadella** 的框架是：无论 **Microsoft**、初创还是百年银行，都需处理人力资本与**代币资本（token capital）**的交织与复利。

关键不在「数据聚合」，而在**企业隐性知识（tacit knowledge）**：独特运作方式、判断力、品味——今日主要在人力与其数字遗存中。**AI 模型**可通过人类工作轨迹提取并编码进权重。企业须格外留意：人类与数字资产如何共同训练**作为 IP 保留的模型**，而非**泄露**——泄露是**单向门**，百年积累的独特性可能一夜蒸发；资产负债表上甚至没有「隐性知识」科目，但我们默认有人力在就有。

**Satya Nadella** 指模型公司建「健身房」式奖励环境，雇佣**曾在你们公司工作过的员工**——这本身说明不该发生的事。范式应变为：欢迎模型进来，但在**你控制的机器里爬坡**；数据是喂给模型的 context，收集人机与 Agent 协作轨迹，**持续闭环且不泄露**。

### 对谈实录

**Reid Hoffman**：「企业需要自己的前沿智能，又要控制数据——该怎么想？」

**Satya Nadella**：「经济将由人力资本和代币资本共同塑形……隐性知识是独特运作、判断、品味——今天主要在人力里。」

**Satya Nadella**：「若泄露，是单向门——你可能花了一百年才有的东西，我们甚至说不清它是什么。」

**Satya Nadella**：「模型应在**你控制的机器里**爬坡；收集轨迹，持续闭环，不让它泄露。」

## AI 员工与未来工作：从 IDE 到 ADE 的演进预览

### 核心洞察

> 编程 Agent 从补全→聊天→任务→长时自主的链条，将复刻到全体知识工作；新瓶颈是百个 CLI Agent 的认知负载。

### 深度解析

**Reid Hoffman** 延续微软董事会讨论：**AI 员工**——不只工具放大人类，还配置「专家级员工」——企业担心员工所承载的隐性知识是否外流。

**Satya Nadella** 用 **80 年代**类比：若有人说「40 亿打字员每天醒来打字」，当时会觉得荒谬——我们有打字员池、幻灯片池；但我们发明了**知识工作**，人人打字产出数字制品。我们尚未在概念上对齐的新问题是：**Microsoft** 约 **2 万**员工对 **200 万**乃至 **2000 万** Agent，全在同一闭环——产生什么隐性知识？人机与 Agent 间传递什么制品？

**编码**是最佳观察场：**IDE** 内代码补全（像拼写检查、**IntelliSense**）→ 聊天引入 **Stack Overflow** 知识 → 推理模型与 **Agent 模式**小任务 → **长时自主**「发射后不管」完成整份 **PR**。该过渡将发生在**所有工作**。**Copilot** 已有聊天、**co-work**；**Build** 宣布 **Scout** 与 **Autopilot**。

新痛点：人人开 **100** 个 **CLI** Agent 会话，线性聊天导致**管理认知负载**爆炸——于是回到类 **IDE** 的 **ADE（Agentic Development Environment，智能体开发环境）**：新 **GitHub** 应用像 **Agent 收件箱**，跨仓库微操宏观委托；**Canvas** 功能可用看板可视化 **PR**，便于人与 Agent 交互。

研究方向：模型需更擅长**保持航向**与**可引导性（steerability）**——不只指令跟随，还要真正可引导，才有信心。未来工作=人力资本与 **AI 资本**共创隐性知识，如同知识工作由数字制品与人力共创。

### 对谈实录

**Reid Hoffman**：「**AI 员工**——专家级 Agent 被配置到别处，企业所承载的隐性知识怎么办？」

**Satya Nadella**：「我们还没概念对齐：**2 万**人对 **2000 万** Agent 在同一闭环里会发生什么。」

**Satya Nadella**：「从补全→聊天→小任务→长时自主完成 **PR**——这会在所有工作发生。」

**Satya Nadella**：「我现在开 **100** 个 **CLI** Agent——认知负载太高；所以我们回到 **ADE**，像 Agent 收件箱。」

## Agent 365：可观测、安全、治理与长时 Agent 断言

### 核心洞察

> 企业 AI 第三层是**可管理性与安全**：Agent 清单、身份、沙箱、策略，加长时运行时的**断言（asserts）**防运行脱轨。

### 深度解析

**Reid Hoffman** 问除体验层、爬坡机器外，企业还缺什么结构——例如「企业里的安全是什么？」

**Satya Nadella** 列三层：**体验层**（人类 Canvas、**ADE** 等）、**爬坡机器**、**可管理性与安全**——从**可观测性**始。**Agent 365**：**Microsoft** 可能有 **2000 万** Agent，须先建立清单——它们是谁、在做什么、推理轨迹须**完全可检视、可审计**。Agent 还能**生成并执行代码**，需受策略约束的执行环境（文件系统、网络访问）→ 给 Agent **身份**、**沙箱**、**策略**。

**Agent 365** 延伸 **Entra**（身份）、**Defender**（安全）、**Purview**（自动标注数据以保护）。长时 Agent 方面，编程语言早有运行时**可验证性**模型；**Foundry** 新增 **asserts**：在运行中断言边界，而非仅靠 guardrails 分类器——让执行路径**不偏离轨道**。平台、运行时、安全、管理、可观测层正在快速工程化。

### 对谈实录

**Reid Hoffman**：「除 Canvas、**ADE**，企业还缺什么结构？安全在企业里意味着什么？」

**Satya Nadella**：「**Agent 365**——我要知道 **2000 万** Agent 是谁、在做什么；推理轨迹须完全可审计。」

**Satya Nadella**：「它们能写代码并执行——需要身份、沙箱、策略。」

**Satya Nadella**：「长时 Agent 需要 **asserts**——在运行中断言边界，不是只靠 guardrails。」

## CEO 必须领导：AI 是企业的未来而非 IT 项目

### 核心洞察

> **Fortune** 称 **Satya Nadella** 像 AI 团队里的创业 CEO；更广义的教训是：非科技 CEO 也必须能点名自己的**代币资本**。

### 深度解析

**Reid Hoffman** 引用 **Fortune** 称 **Satya Nadella** 在 **Microsoft** AI 团队内像**创业 CEO**，并问各业 CEO 在 AI「再创业时刻」该做什么。

**Satya Nadella**：科技 CEO 必须深入**技术栈**，对未来有根本世界观，在成为主流共识之前就下判断——行业转型相当二元，CEO 须**身先士卒、射门**，知转型之严酷。**非科技 CEO** 过去可专注银行/医疗等主业，靠科技顾问——他正在**修正先验**：行业与 CEO 社群**广泛**尚未清醒，仍满足于发新闻稿、秀 **8 个 Agent** 就宣称有 AI 策略。

**AI 不是一类技术，而是企业的未来**——如同不能说「我不知道公司的人力资本」。CEO 须深入理解：**你的代币资本是什么？** 能否具体说：基于昨夜/全公司运作，我们如何把劳动翻译成**你拥有、你控制、你创造**并设系统让其复利增长的知识——可以是上下文、技能、模型权重，不限形式，但必须可识别。

这与 mobile/PC/cloud 不同——那时 IT 部门懂供应商即可；现在是：**当 AI 基本掌握你们行业所需的一切**，行业结构如何变？从此理解这不仅是技术，而是**企业的根本变革**。

### 对谈实录

**Reid Hoffman**：「**Fortune** 说你像 **Microsoft** AI 里的创业 CEO——各业 CEO 该怎么参与？」

**Satya Nadella**：「若你是科技 CEO，必须在技术栈深处——否则没希望。」

**Satya Nadella**：「AI 不是技术，是企业的未来——就像不能说不知道人力资本。」

**Satya Nadella**：「你能回答吗：你的代币资本是什么？昨夜的工作如何变成你拥有、你控制、能复利增长的东西？」

## 平台信任与 AI 供应链：一年后的世界将彻底转向

### 核心洞察

> **Microsoft** 长期稳定来自「平台之上价值远超平台自身」；**Reid Hoffman** 一年后或将坐在「我的 AI 供应链长什么样」已彻底改变的世界里。

### 深度解析

**Reid Hoffman** 问 **Microsoft** 在 **LinkedIn**、**GitHub**、**OpenAI** 等收购与伙伴中如何建立信任——AI 时代更甚。

**Satya Nadella**：长期稳定的是做**工具与平台公司**，定义于**平台之上创造的价值应远超平台捕获的价值**——否则无稳定性，也无信任；客户与伙伴知非零和，非「补贴两三年再吃你 lunch」的博弈。**AI 时代这将是一切**——**Satya Nadella** 赌 **Reid Hoffman** 一年后坐在这里，世界将**完全转向**「我的 **AI 供应链** 长什么样」：如何帮企业**复利**其可独特点名的 AI 回报。学习系统按定义无边界，但市场与社会结构要求边界。

### 对谈实录

**Reid Hoffman**：「**Microsoft** 全球领先于如何在生态里建立信任——AI 时代有哪些教训？」

**Satya Nadella**：「我们是平台公司——平台上创造的价值应远超我们捕获的，才有信任。」

**Satya Nadella**：「**Reid**，我赌一年后你坐在这里，世界已完全转向：我的 AI 供应链长什么样。」

## 科学发现前沿：Manas AI 与 Reid Hoffman 回归创始人模式

### 核心洞察

> **Reid Hoffman** 宣布年底离任微软董事会，全情 **Manas AI**；若 AI 先破科学再破代码，社会许可会高得多。

### 深度解析

**Reid Hoffman** 问 **LLM** 下一前沿。**Satya Nadella** 转向 **Reid Hoffman** 投入的**科学**：**Manas AI**（与 **Siddhartha Mukherjee** 等共创）——若编码/知识工作的进展发生在**科学**上，社会许可会高得多；他希望顺序反过来——先做有益社会的科学发现，公众会更把 AI 当助手。

**Reid Hoffman** 透露：近月 **Manas AI** 进展令其需回归**创始人模式**——内部称化学领域的 **Move 37**（借 **AlphaGo** 神之一手），已出现可能针对多种疑难癌症的化学结构，顶尖计算化学家称「从未见过，可能有效」；仍处早期。**Sid**、**Uzair** 与 **Reid Hoffman** 讨论后，决定 **10** 年微软董事会任期至年底过渡——与 **LinkedIn**、**OpenAI**、**GitHub** 等合作荣幸巨大，但须重新投入公司；仍将与 **Satya Nadella** 密切合作。

## 异质智能、认知覆盖与代币效率

### 核心洞察

> AI 推理模式与人类不同却可放大能力；**认知覆盖（cognitive coverage）**与**代币智能**是知识工作者的新核心技能。

### 深度解析

**Reid Hoffman** 用「外星智能」框架：高度模仿人类却推理模式不同——**Manas AI** 的化学人类未发现。**Satya Nadella** 谈归纳/演绎与 AI 的**认知闭环**：同事在 **GitHub Copilot** 写 **cognitive coverage** 技能——像测试覆盖率，Agent 做完活后出测验，让人类**从 Agent 行为形成演绎理解**；模仿游戏在一侧，**认知覆盖**在另一侧——「我们已认知覆盖 AI 做了什么」。

**Reid Hoffman** 接**代币资本**：现代知识工作者须谋划 **AI Canvas** 编排，且在**资本配置边界**内——代币可像「百万美元马桶盖」无效挥霍。**Satya Nadella** 称**代币丰裕**时代更值钱的是**如何用代币**：谁更高效达成重要结果谁领先——靠 **eval/评分量表** 捕捉只有你能定义的高品味。

**Build** 案例：零售商/快消处理**贸易促销**索赔——可用 Agent 自动化，但**别用前沿模型解非前沿问题**（他的铁律）。这是可重复、确定性的工作流，非材料科学发现——可用 **MAI 5B** 等小模型，用轨迹做 **RL** 爬坡，**性能可超过**前沿模型提示。**Satya Nadella** 强调对系统边界与特性的理解是溢价能力。

### 对谈实录

**Reid Hoffman**：「像外星智能——**Manas** 的化学人类没发现；还有哪些推理模式放大人类？」

**Satya Nadella**：「**认知覆盖**——Agent 干活后出测验，让人类演绎理解它做了什么。」

**Satya Nadella**：「一条铁律：**别用前沿模型做非前沿问题**。」

**Satya Nadella**：「用 **MAI 5B** 和轨迹做 **RL**，可以超过前沿提示——这才是代币效率。」

## 主权 AI 隐喻：国家应学企业，别落前沿之外

### 核心洞察

> 国家「主权」最好体现在本国企业能在代币经济中繁荣；闭关或单模型依赖都会失去比较优势。

### 深度解析

**Reid Hoffman** 问企业数据主权与国家**主权 AI** 的平行关系。**Satya Nadella** 转向企业——因企业遍布全球——国家若抽象谈主权可能大错，侵蚀本国商业活动所承载的**比较优势（Ricardo）**；保住它才是最好主权。防火墙/数据驻留可考虑，但核心是**有经济→有繁荣企业→能在代币经济中建 IP**；与境外伙伴合作往往比突然落前沿更重要。

政策者须想**基础设施**：代币一端是电子，一端是 token——要做最便宜、最好、最环保的**电子—代币**转换数据中心；半导体等亦正当。最重要：用 AI **放大本国比较优势**——大小企业、跨国、公共部门都应在前沿运营；最糟是以主权之名脱离前沿；同时**依赖单一前沿模型**也非主权。解法是：**用模型在自家企业上爬坡**——经济体汇总即方程。

### 对谈实录

**Reid Hoffman**：「各国担心主权 AI——与公司信息、员工主权有何平行？」

**Satya Nadella**：「保住商业活动里的比较优势，才是最好主权。」

**Satya Nadella**：「最糟：以主权之名脱离前沿；也最糟：依赖单一前沿模型。」

**Satya Nadella**：「解法：用模型在自己企业上爬坡——经济体汇总如此。」

## Maya 与 Cobalt：微软自研芯片与系统级优化

### 核心洞察

> **Microsoft** 自研 **Maya**/**Cobalt** 与 **Nvidia**/**AMD** 通用 GPU 并用——优化大规模新负载，而非「一切自研」。

### 深度解析

**Reid Hoffman** 提 **Microsoft** 造 **Maya**、**Cobalt** 且与 **Nvidia**、**AMD** 伙伴并行——战略任务是什么？

**Satya Nadella**：**Nvidia**/**AMD**/**Intel** 建**通用**技术；**Microsoft** 用 **GPU** 做训练推理，也加速其他负载——**Build** 宣布用 **GPU** 加速 **Fabric** 数据仓库；Agent 负载需全方位性能。**CUDA** 等可加速多样负载；旧 **Nvidia** 芯片集群在生命周期内复用亦经济。

新负载：**数据并行同步**训练推理 + **Agent 运行时**（调用模式极不同，**3** 年前尚无规模）——从数据中心土建、冷却、**DC-AC** 最小损耗直送硅片，到 **Maya** 与 **MAI**/**OpenAI** 模型协同设计，**Cobalt**（**ARM** 核心）按 **GitHub** Agent 轨迹的编码调用模式优化——延迟/性能大增。**Microsoft** 将是持续优化大规模负载的**系统公司**，同时最大化利用伙伴通用技术。国家亦应：**开放外部创新 + 内部创新 + 全面基准测试**，为更好经济性——非「一切 **Maya**/**Cobalt**」，亦非完全外购。

### 对谈实录

**Reid Hoffman**：「**Maya**、**Cobalt** 与 **Nvidia**/**AMD** 并行的战略任务？」

**Satya Nadella**：「**Maya** 与 **MAI**/**OpenAI** 模型协同设计；**Cobalt** 按编码 Agent 调用模式优化。」

**Satya Nadella**：「我们是系统公司——优化大规模负载，同时最大化利用伙伴通用技术。」
