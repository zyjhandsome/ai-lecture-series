## 核心导读

> **全文论点**：**Aravind Srinivas** 把 **Perplexity** 定位为 AI 时代的**编排者**而非模型转售商：答案引擎只是引流，真正付费在前沿「替你干活」的代理；**Google** 首页已被倒逼成 **Perplexity** 式体验，但钱不在被商品化的消费搜索，而在**代理编排层**把模型智力转成高价值输出 token——核心指标是**每用户每瓦 token 价值**。长期赢家是不绑单一模型栈、能在本地与云端之间智能路由的公司；短期与中期真正卡脖子的是**电力、许可与公众阻力**，而非模型本身；出口管制短期护住美国前沿，却可能逼出中国在物理栈上的更强对手。

**进攻心态与产品叙事。** 出身印度中下阶层，**Aravind Srinivas** 选「赢的快感」：相对「进 **Google** 就算成功」的旧目标，已远超预期，故应**全线进攻**，而非防守避败。早期在社媒高调对比 **Perplexity** 与 **Google** 并不后悔，只是叙事陈旧；搜索战役已写入行业：**Google AI Mode** 在字体、引用、内联加粗与追问上「几乎就是 **Perplexity**」，只是质量仍差一截——这既是验证也是提醒：前沿已转向代理、深度研究与 **Computer**。

**模型不是产品，编排才是。** **Greg Brockman** 称「模型不再是产品」——**Codex**、**Claude Code**、**Perplexity Computer** 本质是模型 + **代理编排层**（代理循环规则、技能、子代理、连接器）。只转售模型 token 没有生意；**Perplexity** 差异化在于**跨模型编排**（**GPT** 与 **Claude** 可同处一套编排层，而 **OpenAI**/**Anthropic** 互不通融）。高价值用户跑**定时任务**式持续代理，而非一次性深度研究；这类高用量用户月耗可达 **1 万美元** 且物有所值——代理收入总量**将超过 **Google**/**Meta** 广告**（非广告订阅/用量维度）。

**24/7 代理、基础设施与地缘。** 编排要在**准确、智能、隐私、成本**四维找平衡：全云端前沿 **24/7** 无人负担得起，需本地模型 + 芯片 + 编排层持续学习。**Micron**（高带宽内存 HBM 供应商）**6–12 个月内市值可能超过 **Meta****——瓶颈定价权在内存、CPU（代理循环吃 CPU）与电力；约 **40%** 数据中心项目因公众阻力停滞。**DeepSeek** 在华为栈与内存效率上的垂直整合，使出口管制有 **20%–30%** 概率触发「过度建设后遭效率颠覆」；长期仍须严肃对待中国物理栈优势。就业叙事应转向「小团队也能建十亿美元公司」与 **能动性**，而非 **Anthropic** 式「工作末日」营销。

**公司路径与快问快答。** **Perplexity** 自训模型降前沿 token 成本；IPO 目标 **2028**（希望更早），当前 ARR 已「远超 **5 亿**」且增速快。曾被旧金山聚会票选「最可能失败」，此后收入翻三倍、烧钱降超 **50%**。十年持仓首选 **SpaceX**（太空基建独一份）；给弟妹建议：**保持好奇**；**Elon Musk** 的极限聚焦与 **Jensen Huang** 的「每天觉得自己很烂」值得学——**永不退休**。

## 倒逼 Google 重写搜索：答案引擎只是引流

### 核心洞察

> Perplexity 改写了 Google 路线图；AI Mode 几乎复刻其 UI，但前沿已转向代理。

### 深度解析

**Perplexity** 从搜索起家，仍是公众心智里的「答案引擎」；此后陆续推出浏览器代理、深度研究、**Computer** 等，但第一印象未变。 **Aravind Srinivas** 的大胆论断：**Perplexity**（或他个人）对 **Google.com** 的改变，**超过 Google 历史上任何一位产品经理**——因为 Google 内部从未有人愿意动每年 **2500** 亿美元广告生意所依赖的界面。

2024 年底他已预判 Google 会跟进；今天 **Google AI Mode** 在字体、引用格式、内联加粗、超链接、建议追问等体验上「**简直就像 Perplexity**」，只是质量仍不够好——他定期测所有竞品，对 Google 质量滞后略感意外。这对 **Perplexity** **好坏参半**：Google 把搜索做成「本该有的样子」是好事，但**前沿与钱**已不在「回答问题」，而在「**替你去干活**」。

公司仍拥有全球领先的深度研究能力；用户订阅 **Pro/Max** 主要为复杂研究报告与能执行任务的代理，而非传统 Q&A。**答案引擎从来都是前沿产品的引流**——AI 领域没人能舒服地认为「搜索搞定了就躺平」，包括 **Anthropic**：若以为 **Claude Code** 已经赢了，**6–12 个月**后可能连牌桌都坐不住。

### 对谈实录

**Harry Stebbings**：「帮我把『你们比 Google 任何 PM 都更改变 Google.com』这个论点讲透。」

**Aravind Srinivas**：「Google 从来没人想上线答案引擎——没人敢动那个每年赚 **2500** 亿的界面。现在你看 AI Mode，跟 **Perplexity** 几乎没区别：字体、引用、内联加粗、内链、建议追问——整个体验就是 **Perplexity**，只是还没我们好。」

「2024 年底我就料到会这样，从不意外。我反而惊讶质量还不到位——我什么产品都测。说实话，他们终于把 Google 改成该有的样子，我挺高兴。但**前沿才是赚钱的地方**——前沿不再是回答问题，而是**替你去工作**。我们仍有世界最强深度研究；用户付 **Pro/Max** 不是为传统答案，是为复杂研究报告和能办事的代理。」

「答案引擎一直是我们前沿产品的引流。AI 里没人能舒服——**Anthropic** 若觉得 **Claude Code** 已经赢了，6 或 12 个月后可能都不在了。这就是这行令人不安的真相。」

## OpenAI 仍主导消费搜索，但钱在代理前沿

### 核心洞察

> OpenAI 在消费搜索仍占主导，但该层已商品化；Codex、Computer、Claude Code 才是收钱前沿。

### 深度解析

录前 **Aravind Srinivas** 对 **Harry Stebbings** 表示 **OpenAI** **尚未准备好 IPO**——**Harry** 追问：两年前 ChatGPT 一家独大时，能否想象今日 **Perplexity CEO** 敢如此评价？**Aravind** 将「财务就绪」与「是否仍像王者」拆开：财务上 **OpenAI** 可能尚未就绪，但**感知上仍是行业龙头**——在**消费搜索**维度。

然而消费搜索**没钱**：已商品化，只是引流。**OpenAI** 全力 **Codex**、**Perplexity** 押 **Computer**、**Anthropic** 押 **Claude Code**、**Google** 尚未有对标产品、**Meta** 推 **Hatch**（**200** 美元/月）——大家都在抢「前沿」。**Harry** 问非广告领域是否只有 coding 有钱；**Aravind** 强调**不是关于 code 本身**，而是非广告订阅/用量收入总在**当前前沿能力**上——今天是「走出去替用户做事」。

**OpenAI** 能否有 **1000–2000** 亿广告生意？**Aravind** **看跌**：Google 最大广告主是 **Amazon**，其次 **Booking.com**（年 Google 投放约 **160** 亿量级）、**Expedia** 等——订酒店航班仍发生在 **Google** 而非 ChatGPT，因为决策需要**浏览选项、探索式界面**，而非对话式客观答案。直销消费品、时尚广告主要在 **Meta/Instagram** 的「刷信息流」行为里；聊天界面**抓不到**这类意图，还会**腐蚀信任**（问最好蛋白粉却插广告）。**Meta** 等在消息应用里插广告历来失败；**微信** 在中国能成因生态不同，美国用户行为不支撑。**Harry** 再质疑「前沿=前沿模型」是否被高估；**Aravind** 澄清：**前沿指当前 AI 能交付的最佳结果**，不等于前沿模型。

**Harry Stebbings**：「录前你说 **OpenAI** 没准备好 IPO——两年前你敢这么说吗？」

**Aravind Srinivas**：「想想这行：再大的优势也可能沦为追赶者。这跟 **Perplexity**/**Anthropic**/**OpenAI** 有没有 modes 无关——是场的状态。」

**Harry Stebbings**：「你现在仍视他们为行业龙头吗？在哪？」

**Aravind Srinivas**：「是。在**消费搜索**。但那里没钱——已商品化，只是引流。**OpenAI** 为何全力押 **Codex**？因为钱在那。我们押 **Computer**，**Anthropic** 押 **Claude Code**，**Google** 还没有这类产品，**Meta** 推 **Hatch** **200** 美元/月——你看发生了什么。」

**Harry Stebbings**：「**OpenAI** 会有 **1000–2000** 亿广告生意吗？」

**Aravind Srinivas**：「尚未证明。Google 最大广告主是 **Amazon**，**Booking.com** 年花 **Google** 约 **160** 亿——你今天在 ChatGPT 还是 **Google** 订酒店？」

**Harry Stebbings**：「**Google**。」

## 模型不是产品：编排层与 token 经济学

### 核心洞察

> 价值在编排层把模型智力转成输出 token；Perplexity 跨模型编排是 Anthropic/OpenAI 做不到的。

### 深度解析

**Greg Brockman** 推文「模型不再是产品」——作为前沿实验室领袖，他本有动机说相反的话；**Google** 也有人坚持「模型即产品」。**Aravind Srinivas** 认同 **Greg**：**Codex**、**Perplexity Computer**、**Claude Code** 都是**编排系统**——模型 + **代理编排层（agent harness）**（代理循环怎么跑、技能、子代理、连接器、工具）。没有编排层，模型内在智力**转化不成**有价值的输出 token。

若只是模型 token 转售商，**没有生意**——模型会商品化；模型厂商若没有界面产生高价值 token，同样难存续；基础设施层 serving token 有一点生意，应用层必须会** grounding（落地上下文）**、**编排**、接对个人/企业连接器，在统一系统里交付体验。

**Perplexity** 差异化：**不只跨工具/文件/连接器编排，还跨模型编排**——**Anthropic**/**OpenAI** 不能把 **GPT-5** 放进 **Claude Code** harness，也不能把 **Claude Opus** 放进 **Codex** harness；**Perplexity Computer** 可以两者兼有，从而提升**每瓦每用户 token 价值**。若假设定价底层是**电力（瓦特）**，谁能用最少电力产出最有价值 token，谁有定价权——**AI 最重要指标：token value per watt per user**。

人人以为在比模型层，其实在比**界面层谁产生最有价值 token**；建模型甚至是「留在前沿」的手段，但必须**拥有**产生高价值 token 的界面。旧思维「十亿用户才成功」必须改——少数 **高用量用户** 正驱动 token 经济：传闻某工程师让 **Amazon** 月烧 **5** 亿（**Claude Code** 代理循环配置不当）；**Meta** 等公司工程师年耗 **1000** 万级编程工具不罕见；**Perplexity Computer** 有用户月耗 **1** 万+ 且非浪费——业务跑在编排层里的代理循环上。

公司内部有人搭**多代理层级**，像独立软件架构；**Aravind** 常请他们给全公司演示。**高用量 vs 低用量**最大分水岭：是否跑**重复性定时任务**——一次性委派（深度研究）vs **AI 持续监控、事件触发、工作流常开**（每封 inbound 邮件分拣、延迟尖峰时根因分析并找对口工程师）。

### 对谈实录

**Harry Stebbings**：「『前沿』是否被高估？很多基础工作不需要前沿模型。」

**Aravind Srinivas**：「前沿不是前沿模型，是**当前 AI 能给出的最好结果**。**Greg Brockman** 说『模型不再是产品』——他作为实验室领袖本可说相反话。」

「**Codex**、**Computer**、**Claude Code** 是什么？**编排系统**：模型 + 代理编排层——代理循环规则、技能、子代理、连接器。没有编排层，模型智力变不成有价值输出 token。只做 token 转售没生意；模型会商品化。」

「我们差异化：**跨模型编排**。**GPT-5** 不会出现在 **Claude Code** 编排层里，**Claude Opus** 也不会在 **Codex** 编排层里——它们互相竞争。但两者都可以在 **Computer** 里，提高**每瓦每用户 token 价值**。AI 最重要指标就是这个。」

「别想着十亿用户——少数高用量用户在驱动经济。有人月花 **1** 万+ 在 **Computer** 上，不是浪费，是生意跑在这些编排层里的代理循环上。最大差别：你是单次任务，还是**定时任务式持续跑**。」

## 代理收入能否超越 Google 广告

### 核心洞察

> 非广告维度上，前沿代理产品用户虽少，总收入将超过 Google/Meta 广告规模。

### 深度解析

## 24/7 代理：编排四维与本地计算

### 核心洞察

> 24/7 前沿 AI 的瓶颈是成本与编排；需本地模型 + 云端按需，而非永远烧服务端 token。

### 深度解析

编排四目标相互竞争：**准确、智能、隐私、成本**。全押巨型数据中心 → 智能/准确拉满，隐私与成本差；全本地 → 隐私/成本好，可能非前沿智能。解法是**最佳平衡点**：必要时本地、必要时服务端，跨本地与服务端模型编排，落地个人上下文；有时智力够但编排层没接对工具——要世界级编排层让「还行的模型」显得很强，**总编排路由器**选对模型、子任务、子代理，并利用设备端算力。

**Harry Stebbings** 自嘲伦敦 VC 没什么可偷；**Aravind** 称每人都有敏感层级——**24/7 常开代理** 将由**编排者**而非模型厂商实现。

**Harry Stebbings**：「**Sam** 说会有 **24/7 AI**——持续运行的代理会来吗？」

**Harry Stebbings**：「你说的编排问题具体指什么？」

**Aravind Srinivas**：「四个目标在打架：**准确、智能、隐私、成本**。全云端巨型数据中心：智能准确拉满，隐私成本差。全本地：隐私成本好，可能不够前沿。解法：**本地与服务端模型按需编排**，落地个人上下文，世界级编排层让普通模型也显得强——**总编排路由器**选对模型、子代理、还用你设备上的算力。」
