放大镜下的第二次创业：Webflow 联合创始人 Bryant Chou 与 Ploy

AI 把「无限智能」摊在所有人面前，但知道拿它做什么仍依赖十年级领域经验——Bryant Chou 用 Ploy 把 Webflow 时代的设计匠心与 CMO 级增长诀窍灌进专用编排层，让中小公司与创始人不必再雇整支营销队，也能在竞争里靠品牌一致、数据闭环与反 Slop 审美一夜间追上对手。

Bryant Chou on Ploy: Anti-Slop AI Marketing After Webflow 2026-06-19 ~17 分钟阅读

核心导读

AI 把「无限智能」摊在所有人面前，但知道拿它做什么仍依赖十年级领域经验——Bryant Chou 用 Ploy 把 Webflow 时代的设计匠心与 CMO 级增长诀窍灌进专用编排层，让中小公司与创始人不必再雇整支营销队，也能在竞争里靠品牌一致、数据闭环与反 Slop 审美一夜间追上对手。

专用编排层胜过裸模型。 通用氛围编程工具能出页，却难保设计系统一致、难接 Search Console 做 SEO 报告；Ploy 用 Design Slurper（约 75 万美元 token 投入）、3,500 条精选设计提示词与 50+ 集成，把「三五个工程师干一周」压到约 75 秒——经验者把模型当工厂，而非抽奖机。

从首页到营销大脑。 楔子逻辑类似 Rippling 从录用通知书切入：首页是公司对外的脸与叙事源点；接上分析、邮件草稿与夜间「梦境循环」式建议后，网站升级为营销操作系统——YC 批次约 12% 已在用，反馈集中在「终于讲清自己做什么」。

经验创始人的回归。 D&D 式技能点分布曾逼超级个体全栈；AI 让「200 IQ 几乎不说话的码农」也能补营销短板。二次创业者则凭思想迷宫里的坐标直插金矿，再用 Agent 克隆自己——Gary Tan 称这是「40 岁独立创始人 时代」：Bryant Chou 的收束意象是烈日下的放大镜，把二十年经验聚于一点，点燃下一团火。

AI 时代为何需要经验：Ploy 的定位

核心洞察

模型自带无边智能，但做成世界级产品仍靠十年行业经验来「驾驭」它。

深度解析

Gary Tan 开场定位嘉宾：Bryant Chou 联合创立 Webflow，今日支撑约 1% 全网网站；现以 Ploy 重返 YC 当前批次，把建站能力再推一层——不止生成页面，还连接分析、CRM、搜索控制台，让营销在睡眠中自动运转。

Bryant Chou 定义 Ploy：可做出定制、获奖级网站；但更关键是网站已有流量与数据，团队在其上搭建完整营销平台——跑广告、找客户、写文案，并让企业在 ChatGPT、Perplexity、Claude 等答案引擎里被找到，实现营销自动运转。

Gary Tan 概括：像雇到「完美 CMO + 设计师 + 工程师」三合一。Bryant Chou 回应这恰是其履历——Webflow CTO 之外还带过营销与销售，现把这些诀窍烘焙进 Ploy。

界面看起来像众多氛围编程工具，但开箱即得普通工具没有的：集成栈、流量洞察，以及从旧站 URL 一键理解业务语境的能力（后文演示）。

时光机演示：四款 YC 老站点的 2026 重设计

核心洞察

Ploy 重做的不仅是皮相，而是用当代叙事与素材让人第一次看懂公司做什么——这才是反 Slop。

深度解析

为展示设计能力，Bryant Chou 从 Wayback Machine 抓取四位主持人的旧创业网站，短提示词即可「复活到 2026」。

Posterous（Gary Tan，2008）——「用邮件写简单博客」，当年 Gmail 按钮式 UI 今日显陈旧。Ploy 输出被赞「极美」；Gary Tan 笑称可能被诱回去写百万行代码把它再做真。Bryant Chou 指出站内 Google Veo 风格视频与提示词知识已嵌入产品——图像/视频生成需要大量策展与正确提示，起步就有上千例子的上下文是优势。

Scribd（Jared Friedman，2007）——「文档界的 YouTube」。Jared Friedman 回忆在 AWS 之前用物理服务器，最初放在宿舍衣柜里，从而知道「服务器很吵」。Ploy 版被评价为更好、更现代；流程是访问 Wayback Machine 链接 → 理解内容与业务 → 「我们要把这个生意带回现代」。

Auctomatic（Harj Taggar，2007）——帮小商家在线销售、管理 eBay 拍卖。Harj Taggar 惊讶 Ploy 竟生成像真实产品的仪表盘（热门 listing、渠道图等控件），「比当年官网更能让人理解 Auctomatic 做什么」——他自己当年都不太懂自家网站在说什么。

Escher Reality（Diana Hu，2017）——手机 AR API，当年请设计师做素材约 2 个月（Jared Friedman 曾是其批次合伙人）。Ploy 用 3–4 条提示做出带 AR 愿景视频的站点——不是随机视频，而是贴合「把虚拟形象放进真实世界」的产品叙事。Diana Hu：「现在我总算理解自己公司做什么了。」

Gary Tan 与 Harj Taggar 共同点出：AI Slop 若成立，这些站点不会有效——视觉更好，文案与产品理解也更好。

对谈实录

1

Jared Friedman

「你基本上只给了旧站，加一句短提示，就拉到 2026 了？」

2

Bryant Chou

「对。它去了 Wayback Machine 的 URL，理解内容，再理解你的生意，然后说：『好，我们要把这个生意带回现代。』」

3

Harj Taggar

「我以前根本不知道它是干什么的。」

4

Gary Tan

「不只是更好看——内容也更好。要是 AI Slop，这套不会管用。」

12% 批次渗透与营销增长民主化

核心洞察

官网首要任务是讲清故事；Ploy 帮创始人把叙事写得更连贯，而非堆 SEO 垃圾页。

深度解析

Bryant Chou 披露：约 12% 的 YC 当前批次在使用 Ploy；最大反馈是「终于能更连贯、简洁地讲自己的故事」——而这正是网站最重要的部分。

Gary Tan 回忆早年办公时间常帮创始人逐页走读官网解释产品；后来做得少了。Webflow 曾民主化网页开发与设计；Bryant Chou 希望 Ploy 民主化营销、揭开增长的神秘面纱——许多 YC 创始人产品极强，开始学会表达，但 SEO 等机械、繁琐的工作仍让人纠结「要不要雇人」。

Gary Tan 延伸社会图景：裁员潮里被裁的人里，有些能力不亚于创始人、本该自己创业；年纪稍大者难像 Bryant Chou 那样「三重威胁」，但 AI 可能是伟大的均衡器——Ploy 希望成为 AI 乐观叙事的一部分：更多市场入口、更多替代路径。

1

Bryant Chou

「约 12% 的 YC 批次在用 Ploy……反馈就是：『哇，我终于能把故事讲得更连贯、更简洁。』」

2

Gary Tan

「早期办公时间常是：『这是你们的网站，我们带你们走一遍解释它是干什么的。』」

3

Bryant Chou

「我想民主化营销、揭开增长的神秘面纱……这些有点机械、繁琐的 SEO 营销事，他们应该做，但又想：『天哪，我要不要雇人？』这就是我想带给 YC 和所有创始人的。」

D&D 创始人技能与 AI 平权

核心洞察

创始人像 D&D 角色卡各点不同技能；AI 让「单点满级」的极客也能补齐获客短板。

深度解析

Gary Tan 用与儿子玩龙与地下城的类比：重掷骰子时有人生命值高、有人是法师、有人是野蛮人——创始人能力分布同理。过去在编码上「满级」的人往往在表达、增长上弱；Ploy 时代，近乎不善言辞的 200 IQ 码农也能上线产品并靠工具在市场赢。

这与 末日论 叙事对照：Gary Tan 的乐观叙事是资本主义里更多准入、更多替代路径——被裁的资深从业者或许该直接下场。

对谈实录

1

Gary Tan

「以前你在某一属性上满级，往往做不了别的；现在那个人可以上线产品，市场上有太多东西能帮他赢。」

2

Gary Tan

「这就是我希望 Ploy 成为其中很大一块的 AI 乐观叙事。」

Design Slurper 现场演示与品牌一致性

核心洞察

Design Slurper 用确定性流程抽取整站设计系统，解决氛围编程工具「每次混搭、忘记品牌」的顽疾。

深度解析

Bryant Chou 现场注册 Ploy，假装来自 Cursor 团队。团队投入约 75 万美元 token 造 Design Slurper（设计抽取器）：纯确定性方法抓取现有网站，不只设计系统，还生成全部组件——后续生成按钮、标题字体一致，不会十个 header 变体，对 scaling 业务至关重要。

演示中 Slurper 运行并询问目标；Bryant Chou 选择「被搜索与 AI 找到」「把访客变成客户」。约 75 秒内完成抽取、组件重构，产出可响应式站点——字体、悬停 CSS 等最后一公里细节保留。Gary Tan 估等价 3–5 名工程师加前端至少一周；别人可能开 Claude Code 或 Cursor 做「差得多的版本」。

Bryant Chou 强调：要对网站应如何被制作有强主张才能做到；但重点不是克隆——网站建好后它能为你在后台做什么才是 Ploy 真正发力之处。

Gary Tan 补主持引子：其他氛围编程工具难以保持一致、常混搭设计——Ploy 的指令遵循令人印象深刻。

Bryant Chou 回到开篇论点：AI 时代需要一定专业度才知道如何处理模型里的无限智能；在行业十年以上的人，才能调用底层能力做出世界级产物。

对谈实录

1

Bryant Chou

「我们花了大约 75 万美元 token 做 Design Slurper……纯确定性方法，不只设计系统，还有网站应有的所有组件。」

2

Gary Tan

「实时在跑……大概等于三五个工程师加前端，至少一周。」

3

Bryant Chou

「你得对网站怎么做有主见……但重点不是复刻。重点是：网站建好后，它能为你工作什么——Ploy 在这里发光。」

4

Gary Tan

「用别的氛围编程工具做这种一致性很难——它混搭又忘掉设计系统。」

营销大脑：集成、睡眠循环与 SEO

核心洞察

从首页楔子到「公司大脑」：Ploy 夜间读流量与搜索数据，像 OpenClaw 梦境循环，但面向所有中小企业。

深度解析

Bryant Chou 描述集成：50+ 工具——代码库、Figma、分析、CRM、表格；可按访客身份草拟邮件。起点是网站，迅速成为描述产品、展示产品的公司大脑；Gary Tan 类比 Rippling 早期 Parker Conrad 从录用通知书生成器起步——人人有录用通知书，然后 HR、身份认证，再到公司 App Store OS；商业旅程里首页是第一步，Stripe Atlas 之后就得做主页，上线时不能仍语焉不详。

Ploy 接 GitHub 与全部记录系统；更酷的是睡眠时仍在思考：每晚看流量、Google Search Console、销售管线，主动建议——活跃目标账户、有人与营销活动互动等；开箱基础分析，还能告诉你是谁从哪家公司点了哪个行动按钮。

Gary Tan 称这像把 OpenClaw 超级长尾用户才懂的梦境循环（夜间迭代技能）做成人人可用的商业版。

Gary Tan 自述把 Ploy 接到 YC 的 Google Analytics 与 Search Console，OAuth 点几下即得完整 SEO 报告与建议；若无 Ploy，或许能逼 Claude Code 做，但后者不知如何开箱连这些 API、也不懂 SEO optimization，prompt 与工程量巨大。

Bryant Chou：最好 YC 公司站在模型开发「正确一侧」；给模型一点引导 + 大量数据，就有巨大超额收益——喂入结构化与非结构化数据，让它自行推导。

「像 Rippling 早期——Parker 做的第一件事是录用通知书生成器。」

2

Bryant Chou

「它不只连 GitHub，而是你的全部记录系统……更酷的是，它在你睡觉时还在想该做什么。」

3

Gary Tan

「你在做 OpenClaw 用户才知道的梦境循环——但给所有其他人。」

Gary Tan

「没有 Ploy 的话，也许能逼 Claude Code 做，但它开箱不会连这些 API，也不懂 SEO 优化。」

反 Slop 引擎：3500 提示词与 Andy Warhol 工厂论

核心洞察

Ploy 用 3,500 条策展提示词「淹没」模型的默认 AI 审美，像人类设计师翻灵感手册取气质。

深度解析

Bryant Chou 展示后台灵感手册（lookbook）：团队策展的网页设计前沿，别处难获取；用多模型（含 OpenAI ChatGPT 图像）生成 3,500 条网页设计提示词供 Ploy 取灵感——不会复制某一站，而是吸收气质，模仿真人设计师工作流。

Gary Tan 总结：Ploy 是 anti-slop——语料与上下文压过模型内置偏好（例如左对齐圆角等 AI 痕迹）。Bryant Chou 承认无法 100% 消除所有痕迹，但尽力反映个人与品牌的定制表达。

Andy Warhol 类比：Warhol 亲自创作，后期进工厂用机器复制版画——仍是 Warhol；模型是人类创意的工厂， Bryant Chou 想为数字营销交付这种产物。

Gary Tan 问为何选数字营销而非别的「工厂」。Bryant Chou：网页仍是最具变革性的技术之一；互联网让信息传播更民主化，空间仍令人兴奋；AI 时代小企业会更多，需要被找到、讲故事、代表品牌——创业可能比过去更重要。

对谈实录

1

Bryant Chou

「我们做了 3,500 条网页设计提示词……你不会得到一模一样，而是一些气质——这其实是人类设计师怎么工作的。」

2

Gary Tan

「Ploy 基本上是 anti-slop……你用它有的语料和上下文，压过模型内置偏好。」

3

Bryant Chou

「Andy Warhol 画画，后来进工厂用机器复制——还是 Warhol。模型 essentially 是人类创意的工厂。」

Webflow 往事与「煮海」式市场雄心

核心洞察

Webflow 服务约 5 万自由网页设计师的窄用户画像；Ploy 面向数千万人，是 AI 时代吞下整片海洋式的雄心。

深度解析

距 Webflow 约 13 年。Diana Hu 回忆 Bryant Chou 进 YC 时带来直接基于 CSS 的最佳可视化建站 GUI——在 HTML/CSS/JS 上的 visual interface，早期 no-code 标杆，今日仍骄傲其规模。

Gary Tan 问 2013 建站市场已极竞争激烈（记得当时约 8 家，历史更多）。Bryant Chou：联合创始人 Sergie Magdalin 与 Vlad Magdalin 是完美主义者，为「专业感、匠心感扑面而来」的产品死磕每一分钟细节——当时让人抓狂，事后证明关键。

Webflow 与 Ploy 策略几乎相反：Webflow 聚焦单一用户画像——自由网页设计师，诚实估计全球约 5 万；Ploy 解决数千万人的问题——AI 使吞下整片海洋可行，且能到获奖级程度。

Bryant Chou 2006 入行，见证云计算新原语；数年前「智能」新原语令人难以抗拒。

Gary Tan 追问竞争激烈市场如何建立信心。Bryant Chou：这次 TAM 相反、AI 杠杆不同；经验带来「知道往哪挖」的边际。

对谈实录

1

Diana Hu

「你带着直接基于 CSS 建网站的最佳 GUI 进来……」

2

Bryant Chou

「Webflow 我们聚焦一个用户画像——自由网页设计师，老实说全球可能就 5 万。Ploy 我们在解决数千万人……这是非常吞下整片海洋的事。」

2013 Webflow vs 2025 Ploy：竞争与头三个月

核心洞察

AI 改变产出量，不改变聚焦什么、如何塑形——后者仍偏有经验的构建者。

深度解析

Gary Tan 问头三个月 Ploy vs Webflow 2013。Bryant Chou：2013 YC 三个月内手工编码覆盖大量功能——「我还得打字？」；现在人人能 code，模型擅 infrastructure，产出远超以往：更多测试、代码覆盖、功能。

不变的是聚焦与塑形：什么该做、如何塑形仍有利于有经验的构建者——强愿景 + 履历背书时 AI 才真正帮忙。

经验例：Bryant Chou 曾坚持可视化构建面板（拖拽、缩放、流式控制），团队纠结如何把可视化工具带进 Ploy——一再推迟；现判断是：给模型足够上下文、截图、标注（点选「重写这段文案，加粗」）即可——产品内置网页设计师吸收意图产出。

Gary Tan 问模型更强后 Ploy 何去何从。Bryant Chou 无法预测未来，但大量企业仍要有主张的解决方案解决真问题；底层模型是通用型，需要为场景定制的产品帮客户达成结果——纯 SaaS 仍有权利调用模型能力造福终端用户。

Gary Tan 的竞争动态：Ploy 构建营销/SEO/GEO 闭环——建内容、说清产品、迭代、代理也要用；2026 不用 Claude Code/Codex/Cursor 写代码已不可接受；营销侧或同理。
