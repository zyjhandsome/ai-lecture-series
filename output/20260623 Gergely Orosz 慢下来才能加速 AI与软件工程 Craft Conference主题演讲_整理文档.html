慢下来才能加速：AI 代理遍地写码，为何多数团队仍不见生产力跃升？

AI 代理已在全行业铺开写码，表面上是「人人 5 倍产出」，但 Meta Instagram 漏洞 暴露的却是 代币刷量（token maxing）、裁员焦虑与 AI 极端化 如何掏空工程文化。

Slow down to speed up: AI and software engineering 2026-06-23（YouTube 上架；Craft Conference 主题演讲于 2026-06-04，匈牙利布达佩斯） ~15 分钟阅读

核心导读

AI 代理已在全行业铺开写码，表面上是「人人 5 倍产出」，但 Meta Instagram 漏洞 暴露的却是 代币刷量（token maxing）、裁员焦虑与 AI 极端化 如何掏空工程文化；真正拉开差距的公司，把 AI 嵌进团队级代理系统而非个人加速器，同时正视质量滑坡、成本失控与中层管理被掏空——工程师的未来在于验证而非盲读、领域专长与亲手构建 AI 系统，而非把思考外包给模型。

Meta 警钟：速度换安全。 史上最「滑稽」的 Instagram 账户接管只需两步：伪造美国 VPN 位置，向 Meta AI 谎称账户被黑并要求把验证码发到自己的邮箱——没有第二步。涉事代码据称由 AI 生成、AI 审查、无人工把关；约 100 人规模的信任与安全团队里，工程师在刷 AI 代币 leaderboard、担心 5 月 20 日 约 8,000 人（10%） 裁员，40% 核心成员被调去 Alexander Wang 旗下 xAI 做人工数据标注——Meta 正用 22 年积累的工程文化赌一个比 Opus 4.8 更强的自研模型。

行业分野：实验室狂飙 vs 系统投资。 Anthropic 的 Boris Cherny 同时跑 5 个并行代理、日合并 20–30 个拉取请求（PR）；Claude Code 已 100% 自举写 Claude Code。Cursor 开发者年产出代码行数从约 4,000 增至 8,000+，PR 体积 3 倍，合起来约 6 倍 代码量。Uber 约 20 人 平台团队为 3,000 工程师与 2 万 非技术人员自建 MCP 网关、Minion 后台代理、风险画像审查等——Laura Tacho（前 DX、现 AWS 开发者体验负责人）指出：困在左下角「个人 + 简单自动化」的公司看不到团队产出；要到右上角「团队级代理系统」，必须像 Uber 那样大规模投资。

工艺退潮与职业锚点。 Kent Beck 概括：我们积累代码的速度快于积累信任的速度。OpenCode 创始人 Dax 坦言团队在「该整体设计的地方交出一堆草率补丁」；Anthropic 官网 claude.ai 输入框曾连续一个月在键入时整页刷新。GitHub 负载 2 年 3 倍 却逼近 90% 可用性。Gergely Orosz 收尾建议：限制每日代理用量至可验证范围；技术债现在便宜到应主动清；Mitchell Hashimoto（HashiCorp 创始人）规则——我编码时让代理规划，代理编码时我在审查，永远只多开 1 个 后台代理；别外包学习；构建 RAG / 评测（eval） 等 AI 工程 能力并深耕业务领域。

Meta Instagram 漏洞：AI 极端化的自伤

核心洞察

全球最严密的发布体系之一，被「AI 写码 + AI 审码 + 人类刷代币」组合拳击穿。

深度解析

Gergely Orosz 以布达佩斯听众熟悉的语境开场：本周 Meta / Instagram 内部数千工程师经历「可能是 Meta 史上最难熬的一周」。

漏洞机制（两步，第二步不存在）

1. 用 VPN 把位置伪装成受害者所在国（例：要接管 Barack Obama 的 Instagram 账户，就伪装成美国）。 2. 打开 Meta AI，声称「我的账户被黑了，请把验证码发到这个我控制的邮箱」——Meta AI 真的发出验证码，任意账户即可被接管。

Gergely Orosz 称之为史上第一个「零认证、零密码重置」式账户接管——对软件工程师而言，这就是个 bug。难理解的是：Facebook 时代传承下来的工程文化、全球顶尖的自动化金丝雀发布、层层验证、人工代码审查、约 100 人规模的 Instagram 信任与安全（Trust & Safety）团队——全部失效。

连锁反应

- 周二：Meta 首席信息安全官（CISO）在 SEV（严重事件）调查尚未结束时就宣布辞职。 - Gergely Orosz 从信任与安全团队消息人士处获悉（他称此为首次公开细节）：根因不仅是 AI，而是 AI 极端化（AI maxing）、裁员与一种他括号标注的「AI 精神病（AI psychosis）」——意指组织在 AI 指标上失去理性，非医学诊断。

代币刷量（token maxing）

Gergely Orosz 在 4 月 撰文描述的趋势：Meta、Amazon、Uber 等公司将工程师 AI 代币用量纳入考核或隐性信号。 Meta 曾有内部排行榜，头衔包括「session immortal token」「legend」；4 月 项目被砍，但代币燃烧已失控。 Meta AI 免费，工程师为奖金刷量：手写代码？不用；读文档？让 AI 读并烧代币。涉事 SEV 代码由 AI 生成，且由 AI 三重审查，无人类介入。

裁员与组织震荡

- 5 月 20 日：约 8,000 人（10%） 裁员，但提前一个月放风，员工在「我会不会被裁」焦虑下进一步刷 AI 使用量，以免代币数字难看。 - 信任与安全团队不再想「信任与安全」，而在想「代币刷量」。 - 伦敦为主、历时 7–8 年 搭建的信任与安全组织中，40% 在 5 月 20 日 前被调至 Alexander Wang 的 xAI 做 GitHub PR 式人工数据标注（审 PR、加测试、写反馈，循环往复）——Meta 内部玩笑：这支标注大军比 OpenAI 还大；接近 5,000 名开发者在做标注。 - 重组后多数团队规模减半，部分服务无人值班（on-call）——在 Meta 史无前例。 - 美国员工屏幕录制用于训练 AI；士气低于 2022–2023 大裁员；大量留任奖金与持续面试并行——工程师担心被派去「当标注工」，Meta 用 22 年建立的工程组织赌自研模型。

Gergely Orosz 判断：这是 Mark Zuckerberg 与 Alexander Wang 自上而下的选择——模型比业务安全更重要；要么几个月内拿出超越 Opus 4.8 的模型，要么 Meta 工程文化将长期士气低迷。极端，但正在发生。

过去六个月：编码范式为何说变就变

核心洞察

2025 夏仍拒绝 AI 写码的人，到 2026 年 2 月 已翻转——模型跨过「够用」阈值。

深度解析

DHH（David Heinemeier Hansson，Ruby on Rails 创造者）2026 年 2 月上 Gergely Orosz 播客时称：2025 年 10 月他对 Lex Fridman 说 AI 还不直接写他的代码；部分因模型不够好，2026 年 2 月已翻转——大部分代码由 AI 撰写。他是软件工艺派，不受任何实验室资助，但认定「模型写的代码比我能写的更好」。

Simon Willison（Hacker News 高频引用者、独立工程师、Datasette 作者）称 2025 年 11 月发布的 Opus 4.6 与 GPT-5.4 让代理「真正有用」；行业已用六个月消化这一事实，公司开始重金投入。

Linear（Gergely Orosz 合作伙伴，数据首次公开）对比：使用 AI 代理的团队 vs 未使用团队——前者代码交付量约为后者 5 倍（质量另论）。Cursor 数据：

| 指标 | 变化 | |------|------| | 开发者年产出代码行数（仅 Cursor） | 约 4,000 → 8,000+（约 2.5 倍） | | PR 体积 | 3 倍 | | 合效应 | 约 6 倍 代码量 | | 无人工审查即接受 AI 改动的开发者比例 | 2026 年 1 月（Opus 4.7、GPT-5.5 发布后）大幅上升 |

Gergely Orosz 串联 Meta 无人工审查合并上线与这一行业曲线——「六倍代码，六倍 bug」的玩笑在会场引发共鸣。

Anthropic 与 OpenAI：实验室内的代理日常

核心洞察

模型公司把「代理并行 + 自举 + 原型替 PRD」写进日常；非模型公司很难原样复制。

深度解析

Anthropic

- Gergely Orosz 2025 年秋访总部，2026 年 2 月与 Claude Code 创造者 Boris Cherny 深谈。 - Boris Cherny 笔记本上常开 5 个并行代理，日合并 20–30 个 PR，同时领导 Claude Code 全线。 - PRD 式长文档规划「已死」，用原型替代。 - Claude Code 代码 100% 由 Claude Code 生成；全公司 AI 写码比例约 70–90%，无硬性 KPI。 - Claude Cowork 10 天建成，商业上大获成功；Microsoft 内部试图复制（适配 Excel / Windows 本地场景），Satya Nadella 给团队 1 个月 deadline，2.5 个月仍无对标产品——公司差异真实存在。

OpenAI

- Pragmatic Summit 2026 年 2 月旧金山，Gergely Orosz 与 Codex 团队同台。 - 内部 ChatGPT 版有 Fix it 按钮：截图 + 「修这个 bug」→ Codex 生成 PR → 工程师甚至非工程师可合并（有安全网）。 - 多层 AI 代码审查，分级：部分代码仅 AI 审查即可上线，关键路径仍须人工。 - 工程师常同时跑多个代理；访谈中工程师称「没有 1 个代理在跑，是 5 个」——开会时笔记本盖留缝让本地代理继续跑。 - 2025 年 10 月转折：此前仍有开发者 30% 手写、70% AI；Codex 团队已 100% Codex。OpenAI 内部多数人已不再手写代码。 - Codex 自举改进：旧金山时区夜间跑全套测试，早晨产出改进建议；会议/debug 时发语音笔记给 Codex，会议结束前返回结果。 - 内部共识：品味（taste）——知道该做什么——越来越重要。

Cursor 与 Google：产品公司与巨舰的不同节奏

核心洞察

Cursor 全押代理与自研编码模型；Google 体验闭环强，但 Gemini coding 能力拖后腿。

深度解析

Cursor

- Gergely Orosz 2025 年 10 月访旧金山办公室（脱鞋文化、时而凌乱时而整齐——「隐形排序算法」）。 - 2026 年 1 月起从「Tab 补全编辑器」转向代理；自研 Composer 编码模型（除 OpenAI、Anthropic 外少数高质量 coding 模型之一），成本低；租赁 Azure / AWS 上数万 NVIDIA GPU，推理曾为最大成本，现已兼做训练，趋近「迷你 AI lab」。 - SpaceX 收购 Cursor 传闻在场中被提及。 - 全员技术背景：Lee Robinson（开发者关系）用 Cursor 迁移全站 CMS，非传统软件工程师岗位亦深度使用。

Google

- 一切自建：内部 IDE Cider（由 Web 工具演进为 Visual Studio fork）、Jet Ski（对外 Anti-Gravity 的内部版，接入 Piper monorepo 等）、Critique 代码审查（不用 GitHub）。 - Gemini 深度集成；Code Search 被 Sourcegraph 等外界借鉴，但不对外开放。 - 内部系统生态：Borg（Kubernetes 前身）、Monarch（类 Datadog）、Piper monorepo 等，AI 贯穿。 - 痛点：工程师能偷偷用 Claude Code 时就会用——但政策上只能走 Gemini 通道，AI 采纳率低于 Anthropic / OpenAI 系公司；CEO 已公开承认，正在追模型。

Meta 内部工具与 Uber 的自建 AI 栈

核心洞察

Meta 押注自研模型与公开 prompt 轨迹；Uber 用约 20 人团队服务 3 万人级组织。

深度解析

Meta（工具与文化）

- 内部编码助手 Metamate；Trajectories：GitHub 提交会公开展示完整 prompt——2025 年 12 月部署引发反弹（员工未被告知公开），出现「请帮我写 for 循环」类令人尴尬提交；有工程师改用波兰语写 Meta AI 对话以降低可读性。 - 当前优先级：强制重组做模型、全量行为追踪；Mark Zuckerberg 目标——模型超越 Opus 4.8，否则组织士气与能力将面临长期损伤。

Uber（Gergely Orosz 旧东家，详见 Pragmatic Engineer 深度文）

约 3,000 工程师；开发者体验团队约 20 人 已转型 AI 体验团队，自建包括：

| 组件 | 作用 | |------|------| | MCP 网关 | 发现、注册内部工具 | | Uber Agent Builder / Studio | 无代码搭代理；2 万非技术人员使用 | | AI FX CLI | 类 Claude Code，接 monorepo 与多模型 | | Uber Minion | 后台代理规模化；比通用 Claude Code 更懂 Morphus 实验系统；会建议「这样改 prompt 更快更省」 | | Code Inbox | AI 审查 PR 洪水中的优先级收件箱 | | Smart Assignments | 类值班 SLA 的审查分派 | | Risk Profiles | 标记高风险变更 | | U Review | 内部版 CodeRabbit / Sonar |

Stripe（Minions、Tool Shed、Blueprints）、Ramp（Inspect、Glass、Dojo、Sensei）、Shopify（Sidekick、LM Proxy、Dev MCP）、Airbnb（One Everything、Catalyst）等大厂同样在自建——「你把 Slack 接个 AI 代理很酷，但这是下一层级」。

初创公司与传统企业：同一浪潮，不同水位

核心洞察

初创敢「一句话修全库 bug」；传统巨头并不落后，只是没有 Uber 级平台团队。

深度解析

初创

- 代理写码、AI 审查、Slack 集成创意层出不穷。 - 案例：某 B 轮 7,000 万美元 初创在 Slack 对代理说「哈哈，把代码库所有 bug 都修了」——代理回报 4 个严重认证漏洞，「后门大开」；团队此前不知。 - 「氛围编程 SaaS」多见于初创工程师自嗨，Gergely Orosz 不认为是严肃商业模式。

传统企业

- Cisco 2026 年 1 月向 18,000 工程师推广 Codex（当时 Codex 仍较早期），推进复杂迁移。 - JP Morgan Chase 建多代理框架：专用代理标注客户交互数据，配 eval、法官式聚合——「老银行里也有酷东西」。

行业横切趋势：从个人加速到成本悬崖

核心洞察

个人刷代币与工具成瘾退潮；中层管理被掏空；AI 预算突然变成 C 级议题。

深度解析

Laura Tacho 框架（Gergely Orosz 前一晚发信，Laura Tacho 清晨回复）

- 许多组织看到个人很强、团队产出不动：因把 AI 当个人「加速果汁」——邮件摘要、Slack 自动化、甚至代码生成。 - 领先者从业务结果出发：更快上线、同质量下更多功能、或维持质量（Spotify CTO 约 1.5 个月前与 Gergely Orosz 午餐：用 AI 的底线是质量不变；产出未暴增，但大量内部质检工具，AI 推广节奏比 Meta 慢——符合其目标）。 - Laura Tacho 心智模型二维：横轴个人 vs 团队，纵轴简单自动化 vs 代理系统——多数公司困在左下（个人 + 简单自动化），目标在右上（团队级代理系统），路径是 Uber 式系统集成，无法单靠买 Claude Code / Cursor。

代币刷量与工具成瘾

- 在退潮，但 Microsoft 仍有内部 leaderboard（Gergely Orosz 吐槽应关停）。 - 定价结构像成瘾：10/20 美元 个人版 → 触顶 → 100/200 美元 → guilt 不用完额度 → API 按量；早期像赌博「再来一条 prompt」；睡眠变差、醒来想代理。

中层管理被裁或改 IC

- 经理被裁员或改回个人贡献者（IC），「亲自动手」= 少管人多写码。 - Gergely Orosz 辩护优质中层：技术底子厚、能听风声、在故障频发时拉专项小组改文化——砍掉中层，工程文化必降（他的判断）。

CEO / CTO 重返写码

- Vercel CEO Guillermo Rauch 2026 年 2 月：大量 CEO/CTO 狂热情怀编程；上市公司 CEO 私信「我们在用 Vercel / Claude Code，我又写上了」——同时中层减少保护工程师，高层「氛围编程」以为完工。

成本悬崖（极新趋势）

- Sam Altman 当天早晨发帖：AI 预算突然成公司大问题（Gergely Orosz 一周前 Newsletter 已写，OpenAI 内部 Slack 有人调侃 Sam 是否读了）。 - Reddit 梗：共享账户一夜少 15,000 美元。 - Anthropic 对企业客户开启 API 原价（非 startup/个人折扣）；GitHub Copilot 6 月 1 日起类似，有人 3 天烧完原 1 个月 200 美元 额度。 - Uber CTO 3 月称全年 AI 预算已烧光；现设工程师每月 1,500 美元 上限，超额用免费模型——许多公司在 200–1,500 美元 区间试验；成本可逼近工程师薪酬，无人愿长期接受。

软件工艺：质量、脆弱性与仍较真的工程师

核心洞察

代码涨速超过信任涨速；旗舰产品也不内部试用自家界面；认真审查的人被 PR 海淹没。

深度解析

质量滑坡案例

- Anthropic claude.ai 主站：Gergely Orosz 付费用户连续约 1 个月遇 bug——输入「How can I…」时 React 生命周期触发整页刷新，已输入内容丢失；发推后产品经理称「感谢反馈，我们会修」——等价于「百万日活竟无人内部试用」。已修复，但暴露「最快、最赚钱」与银行级站点体验反差。 - OpenAI Agent Builder：宣传 6 周 1 人用 Codex 建成（对标 Uber 内部搭建器）；上线后 P0 bug 长期不修，论坛像被弃软件——「更快，但非更高质量」。 - Amazon AWS：内部 Cairo AI 编码工具允许代理删除并重建环境 → 大规模服务中断；Amazon.com 部分因 AI 生成代码宕机——Amazon 现规定 AI 生成变更须高级工程师审查，因初级工程师常「看起来没问题」就放行。

OpenCode 与 Dax 的诚实

- 开源 AI 编排层（harness）领导者，近 100 万 日活，4–5 个月 10 倍增长。 - Dax 上周播客：交出了太多本应先重构的草率补丁；判断失准；「我们告诉团队少用一些 AI」——在 AI 编码工具赛道，没有对手因「更会用 AI」而击败他们；OpenCode 反而因放慢而保持高质量编排层（harness）优势。 - Mario Zechner（OpenCode 底层 pi 作者，奥地利 AI 圈）：软件 everywhere 成「脆 mess」；98% 可用性成常态；UI 怪 bug 加速；Kent Beck（次日主题演讲嘉宾）总结：我们积累代码的速度快于积累信任的速度——代码需要理解与信任，现在没时间做。

审查倦怠（slob buries the craft）

- PR 量、AI 生成比例暴涨；多数人 AI 审查 + 人工点赞通过，未真读；少数仍认真审查者发现重复代码、推回，却被产出英雄淹没——倦怠、离职；OpenCode 专门招聘这类「最后守门人」；工程管理缺位，无人奖励守质量者。

经验放大

- 资深工程师获益最大；Hillel Wayne（次日演讲）：AI 生成 TLA+ 形式化规约，只有 TLA+ 专家在 prompt 里写清规格才成功——类比移动端：初级工程师可 prompt 出 iOS app，但不可维护。 - 领域驱动设计、Verbally guardrails 等「老派企业模式」在 OpenCode 复兴——代理像需护栏的「新初级工程师」。

基础设施脆弱

- GitHub 约 2 周前 8–12 小时 PR 全消失；第三方 uptime 追踪估 GitHub 整体可用性不到 99%（约 10% 时间部分故障）；官方称 2 年负载 3 倍，Gergely Orosz 不买账——「3 倍就扛不住，以后 5 倍 呢？」

给工程师与工程领导者的建议

核心洞察

标题即方法：慢下来才能加速——只 ship 你能验证的量；别外包学习。

深度解析

Gergely Orosz 核心建议：

1. 限制每日代理用量至可审查或可验证范围（不必读每一行，但要有验证体系）。Peter Steinberger（OpenClaw 创造者）ship 未读代码，但自建验证、看架构模块、让 AI 画架构图——验证，而非盲 ship。 2. 技术债现在便宜——让代理做「80/20」清债；不做首席清债官等于没用好 AI。 3. 实验不同用法：Mitchell Hashimoto 规则——编码时 1 个后台代理做规划，代理编码时人审查，永远只多 1 个代理；不用五路并行；他极高产且重质量。 4. 别外包学习（Google 旧同事 Gergely Orosz 引述）：bug 修了，心智模型没长；工具不会强迫你学——每次用 AI 须带走一点理解或系统能力。 5. 多思考、少敲键：Dax 曾 95% 思考 5% 编码；AI 后编码减 25% → 96% 思考 4% 编码——多数人做不到 96%，但方向是增加理解时间。

工程领导者

- 必须亲自动手，否则出局；可用 AI 补技术债、做集成；百强提交者里已有产品经理与工程领导——少做人管人，业务也期待如此；爱管人者会倦怠或被迫转型；普通工程师将少获职业辅导、涨薪放缓。

Closing 心态

- Martin Fowler、Grady Booch 等共识：自 60 年代以来软件业从未 12 个月内变这么快；感到应接不暇很正常；偶尔停步问：如何可持续、如何提高质量、如何自动化重复——然后循环。
