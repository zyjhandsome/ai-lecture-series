## 核心导读

> **全文论点**：**Engram** 的逆共识是——AI 当下真正的瓶颈不是原始智力，而是**记忆与持续学习**；与其把越来越大的提示词塞进上下文，不如像预训练/post-training 那样，把公司、团队、个人不断演化的语境**烘焙进权重**，让模型像工作数年的员工一样「懂你们」；代价是前期多烧训练算力，回报是推理侧可少耗**两个数量级乃至 100 倍**令牌，并在 RAG/缓存之外获得只有权重才能做的**联想与检索直觉**；他们想象的世界不是「一个越来越大、通向 AGI 的通用模型」，而是**人人、每团队各有一模型**——私有、持续学习、专精你在乎的事；记忆领域的「ChatGPT 时刻」，是某天你发现实习生一夜之间真的变聪明了。

**始终训练 vs 外挂记忆。** 官网 slogan「**models are always training**」意指：前沿模型已很聪明，但**新任务、新岗位、新组织语境**难以像「法国首都」或「写 Python」那样深植权重；**上下文工程**、工具调用仍有用，却被低估的是把**监督微调（SFT）**、**强化学习（RL）**、**同策略蒸馏（on-policy distillation）** 等同款管线用于**每家公司、每个工作区**。**Dan Biderman** 与 **Jessy Lin** 把**记忆**与**持续学习**视为同一枚硬币的两面。

**内化—外挂分工与产品形态。** 便签与笔记（外挂）永远有价值，但人上班第二天脑子里总留着**模糊直觉**；纯外挂记忆面临每人每天即将产生的**数千万级令牌**洪流——存、搜、反复读都贵且乱。**Engram** 在各工作区训练**按团队定制**的模型：大量**适配器微调（adapter fine-tuning）**——**低秩适配（LoRA）**、前缀、稀疏结构等——并把文档与 agent 交互转成**训练信号**；需**白盒权重**（开源 transformer 最易，闭源可合作）。客户关心的是品牌风格、工作流、该转派给谁——**事实与技能在方法上不分家**。

**苦涩教训、RAG 与 KV 缓存。** **Shaun Maguire** 追问 **Rich Sutton** 的**苦涩教训（bitter lesson）**：更长上下文架构常牺牲精度；**Engram** 同样「苦涩」——多烧算力去学**没见过的新语境**。纯 RAG 解决不了「不知道搜什么」；权重里的联想能补。**KV 缓存**把一篇维基文章撑到约 **80GB** HBM，而 **70B Llama** 全权重约 **100GB** 却「记住整个互联网」——梯度下降能把信息压进极少参数；若能离线把缓存**再压 1000 倍**，加载速度与表示保真度都会变天。

**未来：实习生一夜之间变聪明。** 持续学习概念验证仍是「可教的实习生真的越教越好」——再精的上下文工程也到不了。**Jessy Lin** 愿景：**五到十年**每人一模型，公司成为「**数据平面的神经接口**」——不像文件系统镜像，而是该组织的**脑状态**；**记忆钱包**、离职可带走**技能**（经消毒）与保密协议伦理，都是待定的产品形态。

## 始终训练：记忆与持续学习是一枚硬币的两面

### 核心洞察

> 模型够聪明了；瓶颈是**新语境**能否像预训练知识一样深植权重——这就是记忆与持续学习。

### 深度解析

**Sonya Huang** 以 **Engram** 官网开问：「我们不从预训练/post-training 的视角看世界，我们的模型**始终在训练**——这是什么意思？」

**Dan Biderman** 的回答框架很清晰：当今模型已掌握海量知识，但让它们**更有用**的瓶颈，日益不是原始智力，而是**理解新的、不断演化的语境**——新任务、新岗位、新组织背景。目标是把这些东西**烘焙进权重**，就像 pre-training 与 post-training 把「法国首都」「写 Python」写进去那样深。因此 **Engram** 自定位为做**记忆**与**持续学习**——「**同一枚硬币的两面**」：如何让模型学新东西，并深植于权重。

**Jessy Lin** 补充动机链：要搞懂**预训练/post-training 为何能以「魔法般」方式泛化**，并**控制**这一过程，使公司拥有私有数据时，模型也能学得跟公共知识一样好——「这是个很好玩的问题。」

### 对谈实录

**Sonya Huang**：「官网写 models are always training——什么意思？」

**Dan Biderman**：「瓶颈不是模型不够聪明，而是**新语境**怎么写进权重；记忆和持续学习是同一问题的两面。」

**Jessy Lin**：「我们要搞懂预训练/post-training 的泛化魔法，并控制它，让公司的私有数据也能学得像首都巴黎一样牢。」

## 超越上下文窗口：内化、外挂与上下文工程

### 核心洞察

> 上下文工程与工具仍会存在，但被低估的是把**前沿训练管线**用于每个组织域。

### 深度解析

**Sonya Huang** 追问：**记忆**若只是单独数据库、塞进上下文窗口，算不算真记忆、真持续学习？

**Dan Biderman** 认为各类工具会**汇合**：当下主流靠**上下文工程**——巨大提示词、多轮长对话、重组上下文。**工具调用**、上下文工程都会占一席。但被**低估**的是：用前沿实验室做数学/代码那套**训练管线**，应用到**每个公司、每个领域**的语境。

**Jessy Lin** 用人类类比：个人记笔记、贴便签非常有价值，不应抛弃；但第二天上班，脑子里总留着**痕迹与直觉**——「该往哪看」。**外挂记忆**与**内化记忆**应结合；现状偏向外挂，有两难：

1. **规模**：每人每天集体将产生**数千万 token** 量级（**Shaun Maguire** 自嘲「取决于日子，可能是**十亿**」）。
2. **成本与混乱**：存、搜、反复读会很贵；除非检索/理解有重大突破，否则模型也会困惑。

**Dan Biderman** 强调：并非赌工具不存在——模型始终假设**部分知识外挂**、**部分工具可用**；难的是判定**什么该内化、什么该外挂**。即便外挂，各公司也有**定制工具与做法**（不是人人同一套 bash CLI），让模型懂你的定制环境本身也是课题。

### 对谈实录

**Sonya Huang**：「把 memory 当数据库塞进 context，算不算真记忆？」

**Dan Biderman**：「都会用到。上下文工程是现状，但我们缺的是把**训练管线**用到每个领域。」

**Jessy Lin**：「便签永远有用，但人脑子里还有模糊直觉。纯外挂很快会到每天数千万 token——Sean 那天可能是十亿 [笑]——又贵又乱。」

## Engram 产品：按团队把上下文写进权重

### 核心洞察

> 在工作区里为每个团队训练**懂上下文**的模型——不是测试时读文件，而是像资深员工那样运作。

### 深度解析

**产品层**：接入 **Notion**、**Microsoft**、**Harvey** 等**长期协作工作区**——既有团队已写文档，也有越来越多的 agent 对话与反馈。**Engram** 要的是模型**深度理解**该语境：不只推理时读文件，而是像在公司干了几年的员工——知道公司在推什么战略、做事方式、招聘流程怎么跑，能跟任何同事一样干活。

**技术层**：

- **按团队训练**模型，随时间改进用户在乎的能力。
- **写进权重**：大量 **adapter fine-tuning**——**LoRA**、prefix、稀疏架构等（「研究了几十年的工具箱」）。
- **训练信号**：把原始文档、交互变成有用信号——**SFT**、**RL**、**on-policy distillation** 等组合成**持续学习**闭环。

**Sonya Huang** 确认场景：我在 **Notion** 的 agent 是否 = 带 **LoRA/adapter**、随 workspace 新内容**持续学习**的定制 agent？

**Dan Biderman**：方向如此。客户会用多种前沿模型、且会持续换模型；**Engram** 需要**白盒权重**——可与有闭源权重的公司合作，但**开源 transformer** 最易落地；「任何 transformer 我们都能做。」

**Jessy Lin**：内外分工是核心难题——什么内化、什么外挂；外挂层各公司工具也各不同。

### 对谈实录

**Sonya Huang**：「讲讲 Engram 架构/产品？」

**Dan Biderman**：「接 Notion、Microsoft、Harvey 等工作区，训练**懂你们公司**的团队模型——不是读文件，是像老员工。技术上 adapter + 把交互变成 SFT/RL 信号。」

**Sonya Huang**：「所以我 Notion agent 就是 LoRA 持续学新内容？」

**Dan Biderman**：「对。要白盒权重；开源最好，闭源可合作。任何 transformer 都行。」

## 算力换令牌：最高 100 倍推理节省

### 核心洞察

> 前期多烧训练，换推理少传上下文——组织优先级类知识可差 **100 倍** token。

### 深度解析

**Shaun Maguire** 问权衡：是否前期把「公司做法」学进权重，之后每次推理少发上下文？

**Dan Biderman** 确认，且不止于此：

- 不必反复**研究、重读**同一材料。
- 不必写**怪物级系统提示词**。
- 推理 token 可降**两个数量级**——不是 50%，而是可达 **100 倍**少。
- 与人、团队、组织、**优先级**相关的事，往往**不存在单一文档**（除非极端制度化全记录）；模型可通过训练**隐式学习**，用 **100 token** 答出前沿模型要 **10 万 token** 才能答的内容。

**质量维度**：不少任务对当前代模型并非「超自然难」；**Engram** 认为会长期存在 **3–6 个月**窗口——某些定制场景前沿模型尚未覆盖，若能**轻量自主学**，在那段窗口就有能力价值。

### 对谈实录

**Shaun Maguire**：「权衡是不是前期训练、之后少发上下文？」

**Dan Biderman**：「是。还能砍掉巨型 system prompt 和反复重读——**100 倍** token 不是夸张；组织优先级类问题，100 token 对 10 万 token。」

## 记忆与理解：什么事该写进权重

### 核心洞察

> 关键不是把「事实」与「技能」拆开，而是学会**什么值得记**——人类记忆本就是有损压缩。

### 深度解析

**Sonya Huang** 抛学术争议：LLM 是否**过度事实记忆**（如首都巴黎）——更好的是抽象「国家—首都」概念而非死记？如何拆分**记忆 vs 理解**？

**Dan Biderman**：

- 某种程度上**必须记住东西**才能组合成复杂概念。
- 缺的是判断**什么重要该记**；学术基准测试常测「某非洲国家某桥长度」——不该占容量。
- **人类记忆是有损的**——智能的一部分是压缩重要的、丢弃不重要的。
- **事实学习 vs 技能学习**无法干净分离；有人剥离事实只留「纯核心」，模型会很不自然、连基础都不知道——但**思考下一步**常需**即时回想基础事实**，走不远；故 **Jessy Lin** 侧强调**训练**重要：要思考更复杂抽象的事，须**内化**一些东西以便组合。

**Jessy Lin** 的历史视角：曾尝试可扩展地拆分、预训练成「检索不内化」——但当前集体**爬坡**的配方仍是事实密集的 pre-training。**深度学习魔法**在于传统 CS 里 **database（事实）** 与 **algorithm（操作）** 分两门课，如今**搅在一起**——需 **Anthropic** 等可解释性努力拆开。AI 进经济时又在**重新分离**：公司有私有语境精心工程化，通用模型像陌生人操作这些语境——但仍需**周期性收敛**，把事实与故事**混回模型**；缺点是可变事实（首都偶变）难烙印进权重。

**Dan Biderman** 收束：**假二分法**；真正问题是**什么重要该记 vs 不该记**。

### 对谈实录

**Sonya Huang**：「模型死记事实是不是 bug？该抽象还是该记？」

**Dan Biderman**：「得记一些东西才能组合概念。问题是**什么值得占容量**——人类记忆有损。剥离成纯核心很不自然。」

**Jessy Lin**：「数据库课和算法课在深度学习里被搅在一起；公司在把语境当数据库工程化，但仍需周期性写回权重。」

## 做梦与离线消化：测试时间之外的深度学习

### 核心洞察

> 缺的是像**做梦**一样的离线阶段——从交互中抽身，实验能力边界，再写回权重。

### 深度解析

**Sonya Huang** 问是否从**做梦**取灵感做 ranking/消化。

**Dan Biderman**（仅作松散类比）：当下**一切在推理时**——看用户给的语境，当场想；走不远、易错。需要把经历**消化回模型**，下次做对、再进步。

**Jessy Lin** 展开「梦」：听起来离谱，但有趣在于——梦里看见、自言自语、**实验在世界里能做什么不能做什么**，且明显偏**社交**。对他们构建的系统同样如此：给模型**时间**从真实交互中抽身，在环境里实验能力边界、测自己知道什么、处理尾部极端情境——「就像我们夜里梦的那些极端情境。」

### 对谈实录

**Sonya Huang**：「做梦对 ranking 有启发吗？」

**Dan Biderman**：「仅作松散类比——缺的是离线内化阶段，现在全在推理时硬想。」

**Jessy Lin**：「梦很怪，但在试能力边界——我们也让模型离线实验尾部情境。」

## 训练胜过策展：奥赛一周的思想实验

### 核心洞察

> 高风险领域人人默认**训练**整合能力；**Engram** 认为同样魔法应下沉到更多人与场景。

### 深度解析

**Shaun Maguire** 要典型赢例 / 动机例子。

**Dan Biderman** 思想实验：某 AI lab（如 **OpenAI**）要**一周内**赢数学奥赛——会建教材目录让人标注章节/graph，还是**合成训练数据、开训练任务、五六天评估**？对训过模型的人，后者显然更优——**训练**整合思想与能力。数学、代码、网络战等高风险域必走此路；**Engram** 认为大量魔法可以交到**更多人手里**、以更有趣方式发生。

**Jessy Lin** 对比**策展**：靠人挑章节不如让管线学。

## 每人一模型：与前沿实验室的不同世界观

### 核心洞察

> 前沿 lab 赌**一个越来越大**的通用模型；**Engram** 赌**每人、每组织一模型**——私有、冲突偏好、模糊奖励。

### 深度解析

**Shaun Maguire**：为何不是 foundation lab 做终局产品？如何在巨人之间穿行？

**Dan Biderman** 世界观对照：

| 前沿 lab | **Engram** |
|----------|------------|

## 苦涩教训、状态空间与架构取舍

### 核心洞察

> 长上下文架构常**精度换内存**；真正「苦涩教训」路径是多烧算力学**新语境**，而非无限重读文件。

## RAG 终结者、检索直觉与 KV 缓存的荒诞

### 核心洞察

> 客户要的是**本能式懂组织**；纯 RAG 缺「该搜什么」的直觉——权重里的联想补位；**KV 缓存**相对权重**荒谬地低效**。

### 深度解析

**客户学什么**（**Dan Biderman**）：应用层大量时间在让模型**为具体用例工作**——品牌风格做网站、跑工作流、个人写作方式等；方法与事实/技能区分**无关**。

**Jessy Lin**：几乎每层 app 都是前沿模型 + 搜索工具循环；合作方要**更快、更高效、更有语境感**地接数据——「机构知识编码进更高效的东西、不必反复检索」、**该转派给谁**；**轻量训练**即可让这些变**本能**，不必长 **agent 循环**。**RAG 终结者**说法：**仍可做 RAG/检索**，但人们想自动化重复劳动、接巨大数据平面——「某种意义上是 RAG 杀手。」

**Shaun Maguire** 追问：RAG vs 更新权重权衡？哪些进权重、哪些外挂？

**Dan Biderman**：**未解**；也是生物记忆根本问题——一年前酒店房号不必写进神经组织，家门密码未来几年有用可烙印。知识工作翻译仍在探索；**尽量少启发式**——人刷 **TikTok** 也接触大量垃圾，脑仍能学且不崩，模型也应如此。

**RAG 局限**（**Jessy Lin** + **Dan Biderman**）：

- 纯 RAG **无法建立关联**——同事做某研究，你抽象层回想相关事，用户没问你也该联想——这只能在**权重**里。
- 检索核心难在 **如何查询 / 知道找什么**——缺直觉；关键词搜索易扩展、RL 友好，但向量嵌入基础设施重。
- **缓存** 部分缓解「同一文件读一百遍」，但不解决「该内化什么」。

**KV 缓存**（**Jessy Lin** 深潜）：公司早期深挖缓存；**KV cache** 是当前做法的**怪胎**——单篇维基（如 **Taylor Swift**）对某配置可占 **~80GB** HBM；**70B Llama** 全权重 ~**100GB** 却压缩整个互联网——**梯度下降**存在证明：极少数字可压海量信息；几 **几十 KB** 文章变成 **80GB 脑状态**。可缓存、可磁盘→HBM，但……若离线用算力（**Fireworks** 等推理栈）把 **80GB 压到 1000 倍小**，加载、速度、保真度含义全变。

**Sonya Huang** 与 **Fireworks** 渊源被 **Shaun Maguire** 调侃——「高推理成本是好事吗？」「Sonia 爱推理 [笑]。」

### 对谈实录

**Shaun Maguire**：「RAG 杀手我还不搞懂——权重 vs RAG 怎么分？」

**Dan Biderman**：「没人答得出；生物也一样。酒店房号别内化，家门密码可以。少启发式——人刷垃圾视频脑也不崩。」

**Jessy Lin**：「RAG 建不了联想；难在**不知道搜什么**。KV cache 一篇维基 80GB，70B 权重 100GB 记住互联网——离线压 1000 倍就变了。」

**Dan Biderman**：「部分解决；但该内化才能联想。员工不会每次搜『我昨天干啥』——就是知道。」

## 记忆的 ChatGPT 时刻与未来世界

### 核心洞察

> 真正的时刻是模型**一夜之间变聪明**；未来每人一模型，公司是**数据平面的神经接口**。

### 深度解析

**Sonya Huang**：未来一两年 memory 的 **ChatGPT 时刻** 是什么？

**Dan Biderman**：难预测；持续学习的概念验证是**可教实习生、越教越好**——再精密的上下文工程**到不了**；需工具全开。时刻像「**跟昨天不一样，它真更聪明了**」。**ChatGPT** 本身未被预见——产品方向事前各说各话。

**Jessy Lin**：若今天辞职、唯一使命是为自己训更好模型、全天候做上下文工程——**个体几乎推不动分毫**，不如等下一版前沿模型；期望未来**你花时间 = 在你乎的领域性能提升**——难在扩展算力/训练**且不破坏数据**。

**快问快答摘录**：

- **最近 AI 惊喜**：**Dan**——读融资新闻；共同体感 **coding agents** 能力跃迁（非完全意外，因早已费力折腾）。**Jessy Lin**——**GitHub Copilot**、**ChatGPT**、agent 浪潮；**尚未见到**根本不同且**无害地即时学新东西**、个人与经济可行。
- **记忆钱包**（token 钱包类比）：**Dan Biderman**——**ChatGPT memory** 跨个人/工作常不想要（「你上周训 GPU 所以推荐床单」——完全无关）；记忆宜**分产品可控**。**Jessy Lin**——圣杯是上班烧 token 创造价值，**IP 留公司**但**技能/做法**经消毒可带去下家——像生物学 + 保密协议；数字版很有意思。
- **视觉 vs 语言**（**Shaun Maguire**「疯话理论」）：生物里视觉**比特率**远高于音频/语言；计算机里**全是电子**，视觉被削弱、语言被放大——到 LLM 层或许同场竞技，架构更适语言。**Dan Biderman**（神经科学）：知识工作非进化产物；人 **视觉偏好**，啮齿类 **嗅觉偏好**；**脑机接口** 应想大脑「地产」分配。**Jessy Lin**：从未爱语言——动物界交流形态各异；曾押视觉+行动，后见 **ChatGPT 时刻** 去 **Mosaic ML** 学 NLP——词 one-hot 高维、人为，却用比视觉大一个数量级模型仍奏效；视觉/视频仍有榨取空间，二者长期各有趣味。

**五到十年世界**（**Dan Biderman**）：每人**自己的模型**，与他人的、前沿的**各服务不同目的**——真「懂你」、按个人/团队方式充当助手；「**处处是不同的智能形态**。」

**Jessy Lin** 收束愿景：神经科学里 **记忆与导航** 共享回路——地标与情景记忆相关；公司可成每人 **LLM 对接数据平面**——像 **Databricks**、**Oracle** 但记忆是**神经化的、个性化、数亿实例**——不是文件系统镜像，而是该数据平面的 **脑状态**；更高效、更具联想性。
