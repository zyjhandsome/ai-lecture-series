## 核心导读

> **全文论点**：Claude Skills 只有在「先发现重复劳动、用 EADA 筛掉该删的、把流程逐步写清、再用 DBS 文件夹结构落地」时才真正省时间；多数人像 **Drake Surach** 16 岁在密歇根湖差点溺亡那样——不看警示就跳进水里——直接开建，Claude 会把错误方向越跑越远；工作本身也正从「雇人坐席」转向「写说明书交给 Claude 执行框架 24/7 跑流程」。

**组织层：席位变说明书。** 传统公司按部门雇人，每个岗位本质是「职位名 + 可重复步骤」。**Drake Surach** 认为重复性电脑劳动不必再占用人手：把触发条件、过程、输出字段写进 **CLAUDE.md** 与各部门 **skill.md**，Claude 按文件夹层级扮演自动化「C 套件」，营销/运营/财务 Skill 成簇运行——哪怕公司只有你一人，你已在手动跑这些流程。

**筛选层：EADA 顺序不可颠倒。** 第一步 **发现（Discovery）** 列出每周重复任务并标时薪（年收入÷2000）；学员 **Travis** 挖出年省 **1.2 万美元**、**173 小时**，**Greg** 约 **1.4 万美元**、**277 小时**。第二步 **消除—自动化—委派—加速（EADA）**：先删无人看的报告，再自动化同构任务，再以时薪 **1/3～1/4** 为界委派需人工判断者，最后才加速真正驱动结果的切片。

**落地层：映射 + DBS + 定时。** 第三步把「自动化堆」逐步写成触发/动作/输出（可用语音听写、**Mermaid** 鸟瞰图）；跳过映射的学员 **Eric** 把营销系统「缝合怪」化几近弃用 AI，重建后输出变「惊艳」。第四步用 **方向—蓝图—方案（DBS）** 建 Skill 文件夹：**skill.md** 定方向，**references/** 放风格与业务蓝图，**scripts/** 放 Python 可重复方案；在 **Claude Code** 里一句话生成 Newsletter 草稿 Skill，再用 **Routines** 挂 GitHub 远程每日巡检新视频。

## 从雇席位到写说明书：工作组织如何变

### 核心洞察

> 岗位即「说明书 + 执行者」；重复电脑劳动正被「说明书 + Claude」取代，你雇的是流程而非席位。

### 深度解析

**Drake Surach** 以社区学员 **Amy** 开场：她几乎不懂技术，却用同一套框架 **7 分钟**做完丈夫（律师）原本 **4 小时**的工作——今天他要交出的就是这套四步法。

他先画传统 **C 套件**结构：CEO 下挂市场、运营、财务、技术等部门，各部门再有员工。以运营客服为例：打开工单系统 → 找最旧工单 → 按规则/文档/公司数据作答 → 关单。**工作本质**就是「职位名 + 一套可重复指令 + 一个人来跑」。

变化在于：**不一定还需要那个人亲自做重复电脑任务**——你可能仍需审批代理产出，但亲手点同样按钮的时代在消退。做法是把指令写下来，交给 **Claude** 这类**执行框架**，由它按步骤执行。初期搭建费时，但你可以用 **CLAUDE.md** 写在文件夹层级里，告诉 Claude 有几个「部门」、各部门做什么、业务层级如何——Claude 据此管理一堆 **Skill**（工作流）。

Skill 在文件系统里就是 **skill.md** 工作流文件，可按营销、运营、财务、通用等成簇。你要写清：每个席位做什么、逐步怎么做、何时触发、过程中改哪些应用字段、最终输出什么——然后 **24/7** 运行且「不抱怨」。一人公司同样适用，因为你本就在独自跑这些流程。

**Drake Surach** 预告：多数人犯的错误，与他 **16 岁**在 **密歇根湖**差点溺亡是同一类——视频末尾回扣；在此之前先走完四步框架。

## 第一步发现：重复任务、时薪与机会排序

### 核心洞察

> 自动化前必须先盘点每周重复劳动并标时薪；多数人不愿慢这一步，但它往往决定整套 Skill 的优先级。

### 深度解析

**第一步：发现（Discovery）**——找**每周不假思索就会做**的重复事，不是一年一次的大项目：每次线索来了重写提案、每周一同一套报告、换了名字发了 **400 遍**的邮件。写下来，每项旁标注**时薪**。

时薪公式：**年收入 ÷ 2000**（约等于全年工时）。例：年薪 **10 万美元** → **50 美元/小时**；**20 万美元** → **100 美元/小时**。若这些重复活由你亲自做，就用你的时薪——用 **50 美元/小时**回邮件会显得荒谬，这正是要点。

**AI Foundations** 为学员做了「发现审计」工具：可调时薪、频率、痛苦指数，自动算年工时、年成本、周工时。更关键的是**加权排序**：AI 就绪度、节省时间、金钱影响、痛苦程度、搭建难度——先知道**自动化什么、按什么顺序**。

认真做第一步的学员数字惊人：

| 学员 | 年可收回成本 | 年可收回工时 |
|------|-------------|-------------|
| **Travis** | 约 **1.2 万美元** | **173 小时** |
| **Greg** | 约 **1.4 万美元** | **277 小时** |

皆非大公司，而是独立经营者，从未把日常手工加总。**Drake Surach** 强调：这是最没人愿意慢下来的步骤，却可能是整套 Claude Skill 开发里最重要的一环。

## 第二步 EADA：消除、自动化、委派、加速

### 核心洞察

> EADA 按难度递增排序：先删无用活，再自动化，再低价委派，最后才加速核心切片——多数人错在从「加速」起跑。

### 深度解析

**第二步：EADA 筛选**（口播偶作 EDA/EEDA，章节标题为 **EADA**）——四个字母依次是 **消除（Eliminate）、自动化（Automate）、委派（Delegate）、加速（Accelerate）**。对清单上每项自问：该删？该自动？该交给别人？该加码？

**消除（第一，最易被跳过）**：有些任务根本不该存在。没人看的报告直接砍掉，零成本买回时间。

**自动化（第二，本视频重心）**：Claude 让电脑上几乎任何同构任务都可自动化。在此步把完整工作流映射后交给 Claude，并可按日程在睡觉时运行——视频后半会演示定时。当时薪值钱且任务每次路径相同，就是标准自动化对象。学员 **Michael** 给父亲的零售店做审计，发现父亲每月分类开支等行政活可收回**逾两周**；这类活 AI 可在其睡眠时几分钟完成。

**委派（第三）**：仍需人类判断、但不必是你判断时，交给时薪 ≤ 你时薪 **1/3（×0.33）或 1/4（×0.25）** 的人。例：你 **100 美元/小时**，阈值约 **33 美元/小时**——对方每干 **3 小时**相当于买回你 **1 小时**，雇员变成「投资」。Claude 无法代你注入意图判断时，委派仍合理。**Drake Surach** 无法让 Claude 剪 YouTube 片，故委派给剪辑 **Daniel**。

**加速（最后）**：众人都想从这里开始——加人、加建、加量，却在系统仍有阻力时把要增长的东西弄坏。**干净系统上的速度才是资产**。消除垃圾、自动化重复、委派余下之后，真正驱动结果的那一小条工作才清晰；**加速**意为向该切片倾注更多（AI 或资金），少做其余一切。

**Drake Surach** 自举分类示例：

| 任务 | EADA 归类 | 理由 |
|------|-----------|------|
| 写 Newsletter | 自动化 | Claude 写得好时极适合 |
| 回邮件 | 自动化 | 同构重复 |
| 每周社群答疑 | 加速 | 必须本人在场、高价值 |
| YouTube 脚本 | 自动化或委派 | 视性价比 |
| 每周记账 | 委派 | 交给会计师，避 **IRS** 审查风险 |
| 剪 YouTube | 委派 | 需人工 |
| 零粉丝播客调研 | 消除 | 未产生结果就不该加码 |

## 第三步映射：训练新人的方式写流程

### 核心洞察

> 一个可自动化任务 = 一份按顺序写清的流程图；跳过映射就会「缝合怪」化，质量崩盘。

### 深度解析

**第三步：流程映射**——从「自动化堆」里取一项，在写任何代码或 Skill 之前，把从起点到终点的**每一步**写下来：每个想法、输入、决策、输出。像培训**从未见过你业务的新人**：第一步做什么？信息从哪来？检查什么？何谓「做对了」的成品？

有序步骤列表 = 可交给 Claude 的工作流；**一个工作流 = 一个 Claude Skill**。前期映射做完后，搭建反而简单。

**反面教材：Eric**——兴奋之下未映射就开建，在 Claude 里堆出整套营销系统（调研、报价、自动化等），文件成山无法理解，输出质量下滑；他自称把系统 **Frankenstein（缝合怪）** 化了，几近发誓不用 AI。慢下来后按 **AI Foundations** 结构重建：想法与文件未变，仅组织方式变了，输出立刻变「惊艳」——**工作内容没变，只是先映射再下水**。

**映射写法（无花哨要求）**：

- **触发（Trigger）**：任务如何开始？
- **动作（Action）**：过程中发生什么？
- **输出（Output）**：成品长什么样？

可用 **Google Docs**、纸笔，或 **Whisper Flow** 等语音听写边讲边记。勿过度纠结格式——按你脑子里的顺序写，Claude 后续会拆解。

**Drake Surach** 已有范例：

- **YouTube → 邮件列表**：新视频发布后发 Newsletter（本视频后半现场搭建）。
- **YouTube → LinkedIn 帖**：点视频 → 描述区「显示字幕」→ 复制字幕 → 提取可发帖要点 → 丢给 Claude 等模型处理。

**可视化技巧**：把流程贴给 Claude，请它生成 **Mermaid** 流程图代码（可要求可复制到 **mermaid.js** / **mermaid.live** 的代码）；Claude 也可在对话内直接渲染。多分支流程尤其有用，便于鸟瞰结构。

此步与开篇密歇根湖故事直接相连：**不探水域就下水**，Claude 亦会把你在错误方向上的设想越跑越远（见末节）。

## 第四步 DBS：Skill 文件夹的三层结构

### 核心洞察

> 成熟 Skill = 文件夹：**skill.md** 定方向，**references/** 放蓝图，**scripts/** 放可重复代码方案。

### 深度解析

**第四步：按结构构建 Skill**。学员常用 **DBS 框架**——**方向（Direction）、蓝图（Blueprints）、方案（Solutions）**——即良好 Skill 的形状。Skill 字面就是云端或本地的**文件夹**，三层如下：

| 层 | 文件夹/文件 | 作用 |
|----|------------|------|
| **D 方向** | **skill.md** | Claude 被触发时读取的主文件；含元数据、分步工作流，近乎模板化提示词；可挂日历、邮件、**Stripe**、银行账户等工具连接 |
| **B 蓝图** | **references/** | 风格指南、语气规则、FAQ、代码库片段等「怎么做才对」的静态参考；**skill.md** 某步可写「读 references/style.md，勿跳过」 |
| **S 方案** | **scripts/**（Python） | Claude 自建、每次同构执行的可重复脚本，负责解析、API 等硬逻辑 |

演示 Skill **Emoji Sparkle**：触发语 **「emojify this」** → 读工作流 → 读 **references** 风格（例：温暖、仅 **2 个** emoji 让文字透气）→ 输出带 emoji 的句子。展示 **DBS** 如何协同。

**Drake Surach** 还教授「**自动化 C 套件**」文件夹层级：根业务目录下分营销等部门，各部门有 **CLAUDE.md** 式说明 + 上下文（**voice.md** 语气、**drake pov.md** 观点、**stories.md** 故事、业务概览等），Claude 在该目录工作时自动带入人格与业务语境——缓解「桌面端怎么跑 Claude」的困惑。

社区案例：**Amy** 用 **DBS** 为律师丈夫建了 **7 个** Skill；**Eric** 重建后产出质变。入门者可下载 **Drake Surach** 另支视频中的免费 **DBS 框架** Skill 作母版。

## 现场搭建：YouTube 视频到 Newsletter 草稿

### 核心洞察

> 四步串起来：审计任务 → 判定自动化 → 贴流程图 → 在 Claude Code 用 DBS 母 Skill 一分钟出可测草稿。

### 深度解析

**Drake Surach** 从发现审计工具现场加任务 **「YT video email blast」**：

- 归属：内容创作
- 单次耗时：**45 分钟**（自述区间 30–60 分钟）
- 频率：每周 **3 次**（对应每周三更视频）
- 时薪：保守 **400 美元/小时**（自认实际更高）
- 工具测算：该任务年成本约 **5 万美元**；即使时薪仅 **125 美元** 也约 **1.5 万美元/年**，周省 **2.5 小时**

**EADA**：保留并**自动化**——Claude **Max** 档约 **200 美元/月**，按日工作 **8 小时粗算约 1.25 美元/小时**；花 **45 分钟**搭好自动化即划算。不委派（外包太贵），不加速（不想亲自做）。

**已映射流程要点**：

- **触发**：新 YouTube 视频发布
- **动作**：动笔前先读 **voice** 与业务概览；回顾已发 Newsletter 避免重复开头；用 **VidIQ** 拉字幕等
- **输出**：基于新视频字幕、符合本人语气的 Newsletter 草稿

进入 **Claude Code**（亦可用 **Claude Cowork** 或 **Claude Chat**；仅 **Chat** 不能像 **Code/Cowork** 那样按 **Routines** 调度 Skill——**Drake Surach** 已将包括非开发者在内的学员迁到 **Claude Code**，称当前最佳自动化执行环境）。提示词大意：

> 围绕 YouTube 上传与 **VidIQ** 字幕，建 Skill 在视频发布次日起草邮件；附工作流映射；**使用我的 DBS 框架 Skill 创建**。

Claude 会追问盲点（草稿落盘位置 → Skill 文件夹内 Markdown；历史邮件 → 从零建档案等），并复用已有 **voice.md**、**drake pov.md**、**stories.md**、业务概览。

生成 **Newsletter drafter** Skill：含 **skill.md** + **references/**，无需 **scripts/**（仅用 D 与 B）。对最新视频（口播标题含 **Hermes agent** 教程）实测：

- 主题行例：「I built an agent that works while I sleep」
- 预览与正文符合频道叙事风格
- 缺视频 URL 时一条跟进提示即可让 Skill 在「Watch it here」处插入 **YouTube** 链接或超链接

满意后可接邮件工具自动发送；Skill 本身可随时自然语言修订。

## 定时 Routine 与密歇根湖隐喻

### 核心洞察

> Routines 让 Skill 在 GitHub 远程 24/7 巡检；真正致命错误是不映射、不验证就跳进 Claude——如同无视湖岸警示下水。

### 深度解析

**定时（Routines）**：在 **Claude Code** 选 **Routines → New routine**，可本地或 **GitHub 远程**（学员常用远程以保持 **24/7**）。例：每日或每 **12 小时**检查「**Drake Surach** 是否有新视频」→ 无则跳过，有则跑 Newsletter Skill。

**密歇根湖回扣**（**16 岁**差点溺亡）：他没听水情、没勘察水域、无视沙滩上已有警示，直接下水游泳——表面感觉良好，暗流却把他越拉越远。**Claude 亦会验证你的想法**；若方向本就错了，它只会把你带得离目标更远。

按本框架先发现、EADA 筛选、映射、DBS 构建后再自动化，才会像 **Eric** 那样「突然好用」，像 **Michael**、**Amy**、**Enoch** 等社区成员那样以分钟计完成原需小时的工作。**Drake Surach** 称已给出配方——任何新建 Claude Skill 都应走此四步。

**课程入口**（视频描述）：完整 Claude 自动化课见 Skool **AI Foundations** 社区。

## 关键语录与交锋时刻
