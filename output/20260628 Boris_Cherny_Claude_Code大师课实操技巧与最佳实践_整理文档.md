## 核心导读

> **全文论点**：**Claude Code** 的价值不在「多补几行代码」，而在以**全代理（fully agentic）**方式理解、重构与交付整段功能；最高杠杆的入门不是立刻改代码，而是把代码库当同事来**问答**——再逐步接入计划确认、可验证迭代、团队工具与分层上下文，最终把同一套代理能力管道化进 **CI** 与并行会话。

**产品定位。** 与逐行补全的上一代助手不同，**Claude Code** 面向**整功能、整文件、整缺陷**的代理式交付；它嵌入现有 **IDE**（**VS Code**、**Xcode**、**JetBrains** 等）、**终端**、**SSH**、**tmux** 等环境，不要求整体换工作流。打开后只有提示栏——这是刻意的「强力工具」设计：用法由工程师自定，而非被产品锁死在某条路径上。

**上手顺序。** **Boris Cherny** 在 Anthropic 新员工技术入职第一天就教：**先问代码库**（架构、实例化方式、**Git** 历史、**GitHub Issue**），再碰花哨编辑。技术岗入职引导从约 **2–3 周** 缩至 **2–3 天**；代码不上传、不建远程索引、不用于训练——本地即用。进入编辑阶段后，核心模式是「探索 → 计划 → 请你确认 → 再写代码」，以及「给它能自检的手段（单测、**Puppeteer** 截图、模拟器截图）→ 自主迭代两三轮」。

**进阶与收束。** 上下文靠 **CLAUDE.md**、嵌套目录记忆、斜杠命令、**MCP** 与企业级策略分层共享；**Shift+Tab** 自动接受编辑、**#** 持久记忆、**!** 落 **bash** 进上下文、**Esc** 随时安全中断。**Claude Code SDK**（`claude -p`）把代理当 **Unix 管道** 嵌入 **CI**、事故响应与日志分析；重度用户则并行多份检出、**Git worktree** 与 **tmux** 远程挂会话。末段问答揭示：**bash** 安全与分级许可是最难工程之一；产品自始多模态；选 **CLI** 因 IDE 碎片化且模型进化可能使重 **UI** 投资很快过时；Anthropic 约 **80%** 技术人员每日使用 **Claude Code**，研究员亦用 notebook 工具做建模。

## Claude Code 是什么：代理式编程而非行补全

### 核心洞察

> **Claude Code** 是**全代理**编码助手：一次交付整功能/整文件/整缺陷，而非上一代「逐行补全」。

### 深度解析

**Boris Cherny** 开场先问现场谁用过 **Claude Code**——多数人已举手；未用过的可在笔记本执行安装命令（仅需 **Node.js**），以便跟完余下演讲。

编码 AI 助手已历数代：早期多是**补全下一行或几行**。**Claude Code** 不服务这一层，而是**全代理（fully agentic）**——用于**构建功能、写完整函数与文件、一次性修完整个缺陷**。

其差异化还在于**与现有工具共存**：不必换掉 **IDE** 或整套流程。**VS Code**、**Xcode**、**JetBrains** 乃至 **Vim/Emacs** 用户均可使用；可在本地、远程 **SSH**、**tmux** 等任意终端环境运行。产品定位是**通用**——对习惯自由形式编码助手的新用户，打开只见提示栏时容易困惑「该输入什么」；**Boris Cherny** 强调这是**强力工具**：能力面极广，因此产品**不**强行引导单一工作流，工程师应按自己的方式使用。

## 环境与入门：终端、GitHub App 与许可

### 核心洞察

> 首次打开先做终端初始化、主题与 GitHub App；常用工具写入许可列表，macOS 听写可大幅减轻打 prompt 负担。

### 深度解析

**Boris Cherny** 推荐的新手环境步骤：

| 步骤 | 命令/操作 | 作用 |
|------|-----------|------|
| 终端设置 | `terminal setup` | **Shift+Enter** 换行，免反斜杠续行 |
| 主题 | `/theme` | 浅色/深色/**Daltonize** 主题 |
| GitHub App | `/install GitHub app` | 当日发布：可在任意 **GitHub Issue/PR** @mention Claude |
| 工具许可 | 自定义允许工具集 | 高频操作免每次点批准 |
| 语音输入 | macOS 系统设置 → 辅助功能 → 听写 | 双击听写键口述 prompt |

他个人常把 prompt **说**出来而非手打——越具体越好，像对另一位工程师说话。**Boris Cherny** 称这「相当好用」：口述长 prompt 可显著减少打字量。

## 第一要诀：从代码库问答开始

### 核心洞察

> 新手与团队入职引导都应**先问代码库**——这是 Anthropic 新员工第一天的标准动作，比直接改代码更能建立能力边界感。

### 深度解析

**Claude Code** 自由度极高，**Boris Cherny** 的**第一建议**始终是：**代码库问答**——向仓库提问，而非一上来动刀。

Anthropic 技术入职第一天：安装 **Claude Code** 后**立刻**对代码库提问。传统入职流程大量消耗团队时间——问同事、翻代码、学工具链，往往要**2–3 周**；有了 **Claude Code** 后约 **2–3 天**。

**隐私与零配置**同样重要：

- **不做索引**：无远程数据库存代码，不上传，**不**用客户代码训练生成模型。
- **无等待**：下载启动即用，无需 indexing 阶段。

**可问的典型问题**（**Boris Cherny** 会演示具体 prompt）：

- 「这段代码如何被使用？」「如何实例化这个类？」——代理往往**不止文本搜索**，会追实例化与调用链，给出接近 wiki 的深度答案。
- **Git 历史**：「这个函数为什么有 15 个参数、命名为何这么怪？」——可追引入者、关联 Issue、提交语境；只需说「查 Git 历史」，**无需**写系统级指令；**Boris Cherny** 强调这**不是**系统 prompt 硬编码，而是模型足够强，知道如何用 **Git**。
- **GitHub Issue**：可用 web fetch 拉 Issue 上下文。
- **每周站会素材**：每周一问「我这周交付了什么？」——代理查 log、识别用户名，输出清单供粘贴到文档。

**给团队演示时的顺序**：**先问答，不要**先展示花哨工具或改代码——这同时教人们如何 prompt，以及「什么能一次成功、什么要两三轮、什么需要交互式解释器」的边界。

## 代码编辑：小工具集与先计划再动手

### 核心洞察

> 代理只需少量内置工具即可串链探索与编辑；大需求务必**先头脑风暴/出计划、请你批准再写代码**。

### 深度解析

熟悉问答后，下一步是**编辑代码**。代理式用法的关键是：**给工具，它会自己组合**。**Claude Code** 内置工具集**很小**——改文件、跑 **bash**、搜文件——代理会串起来：探索 → 头脑风暴 → 编辑。用户**不必**指定「先用 A 工具再用 B 工具」，只说「做这件事」即可。

**Boris Cherny** 常用模式：动手写代码前，先让 Claude **头脑风暴或出计划**。常见反模式是直接要求实现「3000 行巨型功能」——有时一次成功，有时产物完全不是你要的。**最省事的做法**：要求**先思考、列计划、跑给你看、批准后再写**；**不必**开特殊「计划模式」，口头说「写代码前先出计划」就够。

另一条高频「咒语」：**「think with this one commit push PR」**（口语化：想清楚 → commit → push → 开 PR）。无魔法，但 Claude 足够聪明：会建分支、按仓库历史推断 commit 格式、推远端、在 **GitHub** 开 PR——同样**不是**系统 prompt 硬教，而是模型能力。

## 接入团队工具：bash CLI 与 MCP

### 核心洞察

> 把团队已有 **bash CLI** 与 **MCP** 工具告诉 Claude，新仓库入职即可复用整套内部工具链。

### 深度解析

进阶阶段要**接入团队工具**——**Claude Code** 在此真正发光。

两类扩展：

1. **Bash 工具**：例：虚构的 `barley` CLI（**Boris Cherny** 现场举例，非真实产品）——可说「用该 CLI 做 X」，或让它 `--help` 自学；用多了可写入 **CLAUDE.md** 跨会话记忆。Anthropic 内外部客户均常见此模式。
2. **MCP（Model Context Protocol，模型上下文协议）**：除 bash 外可挂载 **MCP** 工具；告知用法后即可调用。

威力在于：进入新代码库时，把团队**已有**工具一并交给 **Claude Code**，它可代你操作——无需每人重配一遍。

## 三类工作流：探索确认、迭代验证与团队共享

### 核心洞察

> 「探索→计划→确认」与「给自检手段→自主迭代」是除问答外最值钱的两种模式；域内关键是让代理**看见**自己的结果。

### 深度解析

**Boris Cherny** 归纳的常见工作流：

| 工作流 | 要点 |
|--------|------|
| 探索 + 计划 + 确认 | 已述：写代码前请你点头 |
| **迭代验证** | 给 Claude **检查自身工作**的手段——单测、**Puppeteer** 截网页、**iOS** 模拟器截图等；有设计稿时先搭 UI 往往「差不多」，**两三轮迭代**后常接近完美 |
| 教工具 + 定流程 | 明确要头脑风暴、要迭代、还是直接改，以便 prompt 对齐预期 |

核心技巧：**无论你的领域**，给一种**能看见结果**的反馈通道（单测、集成测、截图……），代理会**自己迭代**，质量显著跃升。

下一步行动清单：**教 Claude 用你的工具**；**选定工作流**（直接改 / 先计划 / 要迭代）——知道如何 prompt 即可。

## 上下文工程：CLAUDE.md 与分层记忆

### 核心洞察

> **CLAUDE.md** 是跨会话自动注入的项目记忆；宜短、可入库共享，并与斜杠命令、企业策略、嵌套目录记忆组成层级体系。

### 深度解析

工程师脑中有大量系统史与架构上下文；给 Claude 的方式之一是 **CLAUDE.md**（固定文件名）：

- **项目根** `CLAUDE.md`：启动目录下自动读入，**首个用户轮次**即包含其内容。
- **本地 CLAUDE.md**：通常**不**入库，仅个人偏好。
- **团队 CLAUDE.md**：**应**入库共享——常见 bash 命令、**MCP** 工具、架构决策、关键文件、风格指南等；**宜短**，过长会占上下文且收益递减。Anthropic apps 仓库含常用 bash、风格指南、核心文件索引。
- **嵌套 CLAUDE.md**：子目录按需拉入；进入该目录工作时自动附加。
- **企业根 CLAUDE.md**：跨仓库统一策略，由平台团队代管。

**其它拉上下文方式**：

- **斜杠命令**（`.claude/commands`，可放 home 或入库）：例——**Claude Code** 仓库里给 Issue 自动打标签，由 **GitHub Action** 触发同名斜杠命令，省人工。
- **@ 文件/文件夹** 手动拉入。
- **`/memory`**：查看 enterprise policy、用户 memory、项目 **CLAUDE.md**、嵌套 memory 等层级；可编辑指定 memory；**#** 记住某事时可选择写入哪层 memory。

**配置层级**（信息密度高，**Boris Cherny** 原话称 slide 难做全）：

| 层级 | 范围 | 典型用途 |
|------|------|----------|
| 项目 | 单 Git 仓库 | 可入库或仅个人 |
| 全局 | 跨所有项目 | 个人默认 |
| 企业策略 | 全员统一部署 | 斜杠命令、**bash** 许可/拦截、**MCP** 等 |

企业策略例：全员共用的测试命令写入策略后**自动批准**；或**拦截**永不 fetch 的 URL，员工无法 override——既**放行**又**保安全**。**MCP** 的 `mcp.json` 入库后，任何人打开仓库会被提示安装共享 **MCP** 服务器。

**不确定从哪开始时**：**Boris Cherny** 建议**先做共享项目上下文**——写一次，全团队受益，形成网络效应。

Anthropic **apps** 仓库实例：入库 **Puppeteer MCP**，工程师可自动跑端到端测、截图迭代，无需各自安装。

**调优值得投入**：可过提示词优化流程；想清楚受众、是否每次拉入、是否按需、是否团队共享；做对可**大幅**提升表现。

## 快捷键与终端交互

### 核心洞察

> 终端 UI 极简导致快捷键难发现；**Shift+Tab** 自动接受编辑、**#** 记忆、**!** bash、**Esc** 安全中断是日常高频。

### 深度解析

**Boris Cherny** 插播「专业技巧」：终端产品设计像重新发现一门设计语言——极简导致快捷键难找。速查：

| 按键 | 行为 |
|------|------|
| **Shift+Tab** | 进入**自动接受编辑**模式：bash 仍要批准，文件改动自动过；可随时让 Claude undo |
| **#** | 让 Claude **记住**（如纠正工具误用）；自动并入 **CLAUDE.md** |
| **!** | 落 **bash 模式**跑本地命令；输出进入上下文，下轮可见 |
| **@** | 提及文件/文件夹 |
| **Esc** | **随时安全停止**当前动作，不 corrupt 会话；可改指令后重做部分编辑 |
| **Esc×2** | 回退历史 |
| **`claude --resume` / `--continue`** | 恢复会话 |
| **Ctrl+R** | 展开完整输出（与 Claude 所见上下文一致） |

**Boris Cherny** 在确信方向对或写单测迭代时，常开**自动接受编辑**，免逐条点 OK。

## Claude Code SDK：管道化智能工具

### 核心洞察

> **`claude -p`** 即 SDK 入口：与 Claude Code 本体同一套能力，适合 **CI**、事故响应与 Unix 管道组合——「超智能 Unix 工具」用法才刚起步。

### 深度解析

**Claude Code SDK**：同日上午隔壁 **Sid** 有深讲；若用过 **`claude -p`**，那就是 SDK。近几周功能持续增强——**与 Claude Code 用的是完全相同的 SDK**。

典型 CLI 用法：

```bash
claude -p "你的 prompt" --allowedTools "..." --output-format json
```

可指定允许工具（含特定 bash）、输出 **JSON** 或流式 **JSON** 供下游处理。

Anthropic **内部**用法：**CI**、**事故响应**、各类流水线——「给它 prompt，它还 JSON」；可 pipe in/out，像 **Unix 实用程序**。组合空间极大，**Boris Cherny** 认为行业**才刚起步**。

示例：

- `git status | claude -p "…" | jq …`
- 从 **GCP bucket** 读巨型 log pipe 进去，让 Claude 找值得关注的片段
- **Sentry CLI** 输出 pipe 进去做分析

## 并行进阶：多会话、worktree 与 tmux

### 核心洞察

> 重度用户并行多仓库检出、**Git worktree** 隔离与 **tmux** 远程挂会话；可同时跑任意多会话，团队正简化这一路径。

### 深度解析

**Boris Cherny** 自评「Claude 普通用户」：通常**一个** Claude 会话，偶尔多 tab 多仓库。内外部**重度用户**常见：

- **SSH** + **tmux** 隧道挂远程 Claude 会话
- 同一仓库**多份检出**，并行跑多个 Claude
- **Git worktree** 做隔离

产品侧**正在**让并行更易用；现阶段上述是可行思路。**可同时开任意多会话**，并行吞吐可观。

## 现场问答：安全、多模态、为何 CLI 与 ML 用途

### 核心洞察

> 最难工程之一是 **bash** 安全与许可平衡；产品自始多模态；选 CLI 因 IDE 碎片化且模型进化或使 IDE 层投资迅速贬值；Anthropic 约八成技术人员每日使用。

### 深度解析

**问：构建 Claude Code 最难的实现部分？**

**Boris Cherny**：**bash 安全**尤其棘手。**bash** 本质上危险、可意外改系统状态；若每条命令都手动批准又极烦、难以高效产出。要在**不同代码库形态**（并非人人 Docker 隔离）下规模化地做到安全，团队最终采用：**只读命令分类**、**静态分析**判断哪些命令可安全组合、以及**多层分级许可**（允许列表/拦截列表可在项目/全局/企业级配置）。

**问：给 Claude 图片是否有多模态能力？**

**Boris Cherny**：**Claude Code 自始全多模态**；终端 UI 使能力不易发现。可**拖放**图片、给文件路径、或**复制粘贴**图片。他常拖设计稿进去要求实现，配 **Puppeteer MCP** 对照迭代——全流程自动化。

**问：为何做 CLI 而非 IDE？**

**Boris Cherny** 两点：**Anthropic** 内部 **IDE** 极度碎片化（**VS Code**、**Zed**、**Xcode**、**Vim**、**Emacs**……），终端是**最大公约数**；其二，他们近距离看到模型进步极快——**年底前后人们可能不再用 IDE**；为避免在可能很快过时的 **UI** 层**过度投资**，选择 CLI 面向未来。

**问：用 Claude Code 做机器学习/建模（类 AutoML）体验如何？**

**Boris Cherny**：**用得很多**——工程师与研究员**每天**用 **Claude Code**；约 **80%** Anthropic 技术人员每日使用（希望体现在产品打磨与**内部试用**深度上）。研究员会用 notebook 相关工具编辑运行 notebook。
