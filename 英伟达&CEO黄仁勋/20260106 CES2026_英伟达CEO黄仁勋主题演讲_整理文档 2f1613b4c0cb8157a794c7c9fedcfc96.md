## AI平台变革与计算革命

### 计算行业的周期性变革

> 黄仁勋：“每十年到十五年，计算机行业会迎来新平台变革——从大型机到PC，PC到互联网，互联网到云计算，云到移动端。每次应用世界都会适配新平台。”
> 

黄仁勋指出，当前正在同时发生**两个平台变革**：

1. **AI成为新平台**：所有应用将建立在AI之上
2. **软件开发方式变革**：从编写软件转向训练软件，从CPU运行转向GPU运行

### 五层蛋糕架构重塑

> 黄仁勋：“AI重塑了整个计算栈，由于人工智能的发展，这五层蛋糕的每一层都在被重新发明。”
> 
> - 过去十年约**十万亿美元**的计算技术正在被现代化改造
> - 每年**数百亿美元**的风投资金投入AI现代化
> - **一百万亿美元**产业规模的研发预算正在转向AI

### 应用从预编译到实时生成

**传统应用**：
- 预录制、预编译
- 在设备上静态运行

**AI时代应用**：
- 理解上下文
- 每个像素、每个token**从零生成**
- 完全动态响应

## 开源AI模型与前沿发展

### AI发展里程碑时间线

| 年份 | 里程碑事件 |
| --- | --- |
| 2015年 | Scaling Law（规模定律）提出 |
| 2017年 | Transformer模型问世 |
| 2017年 | BERT语言模型发布 |
| 2022年 | ChatGPT时刻——唤醒世界对AI潜力的认知 |
| 2023年 | 首个推理模型问世，测试时扩展（Test-time Scaling）概念提出 |
| 2024年 | Agentic AI（智能体AI）系统崛起 |
| 2025年 | DeepSeek R1——首个开源推理系统模型 |

### DeepSeek R1的历史意义

> 黄仁勋：“去年我们见证了DeepSeek R1的突破——首个开源推理系统模型，震惊了整个世界，并彻底激活了开源AI运动。这真是令人振奋的工作！”
> 

黄仁勋对DeepSeek R1给予高度评价，认为这一突破彻底激活了开源AI运动。

### 开源模型的现状

- 开源模型已达到**前沿水平**，仅落后闭源模型约**六个月**
- 每六个月就会出现新的更智能模型
- 下载量激增：初创公司、大公司研究者、学生、各国政府都在积极参与
- 预计开源模型将成为**最大类别**

### 测试时扩展的三阶段

1. **预训练**：模型学习知识
2. **后训练（强化学习）**：模型学习技能
3. **测试时扩展**：模型实时思考

> 黄仁勋：“这是非常常识性的、合乎常理的做法——让AI思考。人工智能的每个发展阶段都需要大量计算资源。”
>

## 物理AI与机器人技术

### 物理AI的核心挑战

> 黄仁勋：“问题是如何将计算机内的智能，从通过声音和扬声器与人交互，转化为能与世界互动。”
> 

物理AI需要理解的常识：
- **物体恒存性**：转头再看，物体依然存在
- **因果性**：推它会倾倒
- **摩擦力和重力**
- **惯性**：重型卡车下坡需要更长时间刹车

> 黄仁勋：“这些常识连幼儿都明白，但AI完全不知。因此我们必须创建系统，让AI学习物理世界的常识。”
> 

### 三台计算机架构

物理AI需要三台专用计算机：

| 计算机 | 功能 |
| --- | --- |
| **训练计算机** | 训练AI模型（NVIDIA DGX） |
| **推理计算机** | 运行在汽车、机器人、工厂边缘设备中 |
| **模拟计算机** | 评估AI行为是否符合预期 |

> 黄仁勋：“模拟是NVIDIA几乎所有工作的核心，这是我们最擅长的领域。模拟确实是物理AI几乎所有工作的基础。”
> 

### 核心技术栈

- **Omniverse**：数字孪生，基于物理的模拟世界
- **Cosmos**：世界基础模型，与语言对齐
- **Isaac**：机器人模拟器（Isaac Sim实验室）

### Cosmos世界模型

> 黄仁勋：“物理AI的ChatGPT时刻即将到来，但挑战显而易见——物理世界多样且不可预测，收集真实训练数据缓慢且成本高昂，永远不够用。解决方案在于合成数据。”
> 

**Cosmos功能**：
- 为物理AI打造的开放前沿世界基础模型
- 基于互联网级视频、真实驾驶与机器人数据、3D模拟预训练
- 学习世界的统一表征
- 能够对齐语言、图像、3D与动作
- 执行物理AI技能：生成、推理、轨迹预测

**Cosmos应用场景**：
- 从单张图像生成逼真视频
- 从3D场景描述生成物理连贯运动
- 从驾驶场景遥测和传感器日志生成环视视频
- 从规划模拟器生成多摄像头环境
- 从场景提示将边缘案例生动呈现
- 交互式闭环模拟
- 分析边缘场景并推断后续情况

> 黄仁勋：“Cosmos将计算转化为数据，训练应对长尾问题的自动驾驶系统以及机器人如何适应各种场景。Cosmos已被下载数百万次，全球广泛使用。”
>

## 自动驾驶汽车

### Alpha Mayo系统发布

> 黄仁勋：“今天我们宣布Alpha Mayo——全球首个具备端到端推理的自主车辆AI。”
> 

**Alpha Mayo特点**：
- 直接从摄像头输入到执行输出
- 积累大量自行驾驶里程的人类示范
- 大量由Cosmos生成的合成数据
- 数十万个精细标注的示例

**革命性突破**：
- 不仅接收传感器输入并激活方向盘、刹车和加速
- 还会**推断**即将采取的动作
- 会**告知**即将采取的动作
- 会**解释**该动作产生的原因
- 行驶非常自然，因为直接学习自人类示范者

> 黄仁勋：“驾驶存在长尾问题，我们无法收集所有可能发生的场景。然而每个场景都可以分解为大量其他较小的常规场景，通过推理来应对。”
> 

### 完整技术栈

NVIDIA构建了完整的五层技术栈：

| 层级 | 组件 |
| --- | --- |
| **应用层** | 梅赛德斯-奔驰 |
| **模型层** | Alpha Mayo（今日已开源） |
| **基础设施层** | Omniverse + Cosmos |
| **芯片层** | 双Thor芯片（下一代双Orin） |
| **底层** | 汽车本身 |

> 黄仁勋：“这项卓越的工作成果需要数千人参与，我们的AV团队有数千人。”
> 

### 安全系统设计

> 黄仁勋：“这辆车刚刚被突击检查，梅赛德斯CLA刚被NCAP评为世界上最安全的汽车。这是我所知道的唯一系统，每行代码都通过安全认证。”
> 

**双栈安全架构**：
1. **Alpha Mayo栈**：端到端训练，技能卓越
2. **经典自动驾驶栈**：完全可追溯的备份系统

运作方式：
- 两个软件栈相互镜像
- 安全策略实时评估
- 确定情况下启用Alpha Mayo
- 不确定情况下切换回经典自动驾驶栈

> 黄仁勋：“我们是全球唯一同时运行这两套安全系统的汽车。所有安全系统都应具备多样性和冗余。”
>

## Vera Rubin架构发布

### 命名由来

**Vera Rubin**是美国天文学家，发现了暗物质：

> 黄仁勋：“她注意到星系的尾部运动速度与星系中心相当，这完全不合逻辑。牛顿力学会说，远离太阳的行星绕太阳运行速度比靠近太阳的行星慢。这种情况完全不合逻辑，除非存在不可见天体——我们称之为暗物质。薇拉·鲁宾发现了暗物质。”
> 

### 计算需求爆发式增长

| 增长因素 | 增长率 |
| --- | --- |
| 模型规模 | 每年10倍 |
| 推理token生成量 | 每年5倍 |
| 上一代AI成本下降 | 每年约10倍 |

> 黄仁勋：“我们必须每年推进计算技术的前沿，绝不落后一年。如今我们已实现每年交付GB200，现在我们正在大规模生产GB300。Vera Rubin已全面投产。”
> 

### 六芯片协同设计架构

Vera Rubin采用前所未有的六芯片协同设计：

| 芯片 | 功能描述 |
| --- | --- |
| **Vera CPU** | 定制设计CPU，性能是前代两倍，每瓦性能超越全球最先进CPU |
| **Rubin GPU** | 浮点性能是Blackwell的五倍，仅1.6倍晶体管数量 |
| **NVLink 6交换机** | 400Gbps交换芯片，传输全球互联网数据量两倍 |
| **BlueField 4 DPU** | 存储与安全卸载，区域隔离与虚拟化 |
| **ConnectX 9 NIC** | 全球最佳NIC，可编程RDMA和数据路径加速器 |
| **Spectrum X交换机** | 首款硅光子以太网交换机，512端口200Gbps |

> 黄仁勋：“我们在公司内部确立了一条原则：新一代产品芯片变更不得超过一到两处。但问题在于摩尔定律已大幅放缓，无法跟上十倍规模的模型需求。除非实施激进的跨层协同设计，在所有芯片层面全面创新。这就是我们决定本代产品不得不重新设计每颗芯片的原因。”
> 

### Vera CPU突破

- 88个CPU核心
- 采用空间多线程技术设计
- 每176个线程都能获得完整性能
- 本质相当于176个核心

### Rubin GPU突破

- 浮点性能是Blackwell的**五倍**
- 晶体管数量仅为**1.6倍**

> 黄仁勋：“这说明了当前半导体物理水平——如果不做协同设计，如果不进行极端协同设计，在系统每个芯片层面，这怎么可能？我们称之为极端协同设计。”
> 

### MVF P4张量核心

革命性的Transformer引擎创新：

> 黄仁勋：“我们芯片中的Transformer引擎不仅仅是一个四比特浮点数，它实际上是一个完整的处理器——能够动态调整精度和结构，以应对不同层级的Transformer模型。在可以牺牲精度的地方尽可能降低精度，再回到最高精度。自适应调整在处理器内部完成，软件无法实现这一点。”
> 

特点：
- 动态精度调整
- 自适应调整在处理器内部完成
- NVIDIA已发表相关论文，预计将成为行业标准

### 计算节点革新

**NVL72机架规格**：
- 18个计算模块
- 9个NVLink交换机
- 2.2万亿晶体管
- 重量近两吨（运输时更重，因忘记排水）

**制造突破**：
- 从**40根电缆**减少到**零根电缆**
- 从**6根管路**减少到**2根**
- 组装时间从**2小时**缩短到**5分钟**
- 从80%液冷升级到**100%液冷**

> 黄仁勋：“真的是一项重大突破。”
> 

### NVLink脊柱结构

- 两英里长的铜缆
- 五千根铜缆
- 世界历史上用于计算系统最多的铜
- Ceres从机架顶部驱动铜缆，以每秒400Gbps速度直达机架底部

### KV缓存存储创新

解决大规模AI部署的上下文内存挑战：

**问题**：
- AI的工作内存（KV缓存）存储在HBM内存中
- 长对话导致上下文内存大幅增长
- HBM容量不够用
- KV缓存迁移引发网络流量问题

**解决方案**：
- 使用BlueField 4实现高速KV缓存存储
- 直接放置在机架内
- 每个GPU额外获得**16TB**上下文内存
- 创建全新存储系统类别

> 黄仁勋：“行业对此极为兴奋，因为这是大量标记生成者的痛点。”
>

### 系统级创新

1. **能效提升一倍**：液体温度保持45°C，节省约6%数据中心电力
2. **可信计算安全**：所有传输过程、静态存储和计算期间均加密（每个PCIe接口、每个NVLink、CPU与GPU之间的连接、GPU到GPU之间）
3. **系统级功率平滑**：无需超额配置2-5%的能源容量
